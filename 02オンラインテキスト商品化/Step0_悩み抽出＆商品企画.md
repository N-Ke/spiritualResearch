# Step0: 悩み抽出 & 商品企画

---

## 【エージェント指示】

```
読み込むファイル:
  - 入力/ フォルダ内のキャラクター設計書（.md ファイル）
  - 入力/ フォルダ内のAI実行用プロンプト（.md ファイル）
  - ナレッジ/商品グレード定義.md
  - ナレッジ/テンプレートパターン集.md
出力先: 実行結果/Step0_出力.md
次のステップ: Step0.5_無料記事制作.md（無料記事を先に作る場合）
           または Step1_テンプレ選択＆角度決め.md（有料記事を先に作る場合）
```

**エージェントの実行手順:**
1. 上記ファイルを全て読み込む
2. キャラクター設計書のターゲット情報を把握する
3. 下の【実行プロンプト】に従って出力を生成する
4. 出力を `実行結果/Step0_出力.md` に保存する
5. ユーザーに確認を取る:
   - 悩みリストの内容は合っているか
   - クラスター分析の判定は妥当か
   - 商品グレードの振り分けに違和感はないか
6. OKなら次のステップをユーザーに提案する

**ユーザーから追加の悩み情報を受け取っている場合:**
それも考慮に入れて悩みリストを作成する。

---

## このステップでやること

キャラクター設計書に書かれたターゲット情報から、**悩みを5-10個に分解**し、以下の3つを設計します:
1. **悩みリスト**（個別の悩みを言語化）
2. **悩みクラスター分析**（関連する悩みをグループ化 → Premium商品の判定）
3. **商品ライン設計**（Lite / Standard / Premium の振り分け + 無料記事の企画）

---

## 【実行プロンプト】

あなたはオンラインコンテンツの商品企画プランナーです。

読み込んだキャラクター設計書に含まれる「ターゲット情報（ペルソナ・核心的悩み・悩みクラスター）」を深掘りし、ターゲットが抱える具体的な悩みを5-10個に分解してください。

そのうえで、悩みのクラスター分析を行い、商品ライン（Lite / Standard / Premium）を設計してください。

---

### 商品グレードの定義

（ナレッジ/商品グレード定義.md を参照。以下は要約）

- **Lite（100-980円）**: 単一の悩みを1本の記事で解決。3,000-6,000文字。付属物なし。
- **Standard（980-5,000円）**: 単一の悩みを深掘り。8,000-15,000文字+図解3-5枚。テンプレート1-2点を付属。
- **Premium（5,000-30,000円）**: 複数の関連する悩み（クラスター）を包括的に解決する変革パッケージ。5-10章構成、20,000-50,000文字+図解8-15枚。AIツール+テンプレート+ワークシートを付属。

---

### 出力フォーマット

以下の形式で出力してください。

## 1. 悩みリスト

悩みごとに以下を書いてください:

### 悩み1: （悩みのタイトル）
- **具体的な状況**: ターゲットがどんな場面でこの悩みを感じるか（1-2文）
- **理想の状態**: この悩みが解決したらどうなるか（1文）
- **障壁**: なぜ自力で解決できないか（1文）
- **深さ**: 浅い（概要で解決）/ 中（手順が必要）/ 深い（複数ステップ+ツールが必要）
- **関連する悩み**: 他の悩みとどう繋がるか（番号で参照）
- **推奨テンプレパターン**: A-H のいずれか
  - A: ステップバイステップ型 / B: 失敗回避型 / C: 実験レポート型
  - D: 比較・選択肢型 / E: チェックリスト型 / F: Q&A・疑問解消型
  - G: ビフォーアフター型 / H: テンプレ配布型

（悩み2〜10も同様に）

## 2. 悩みクラスター分析

### クラスターA: （クラスター名）
- 含まれる悩み: 悩み◯, 悩み◯, 悩み◯
- 共通テーマ: （根本問題）
- 解決の順序: （どの悩みから順に）
- Premium化の判定: 適切 / 不要（理由つき）

### クラスターにならない悩み
- 悩み◯: 単独で解決できるため、Lite or Standard で対応

## 3. 無料記事の企画（1-3本）

記事ごとに:
- タイトル案（3つ）
- 記事の型（自己紹介/価値提供/悩みマップ）
- 概要（3行以内）
- 有料への導線

## 4. Lite / Standard 商品の企画

記事ごとに:
- タイトル案（3つ、15-25文字）
- 提供価値（1文）/ 推奨グレード / 理由 / 価格帯
- 推奨テンプレパターン / 理由
- Standard の場合の付属物案

## 5. Premium パッケージの企画

（クラスターでPremium化が「適切」のもののみ）
- パッケージ名（案3つ）
- コンセプト / 含まれる悩み / 章構成案 / 付属物の初期案
- 価格帯目安 / ターゲットの購入動機

## 6. 制作優先順位

1. 最初に作る無料記事
2. 最初に作る有料記事
3. 2本目以降の順番
4. Premiumの着手時期

---

### 制約

- 悩みの分解はキャラクター設計書の「ターゲット」「核心的悩み」「悩みクラスター」をベースに
- キャラの視点（判断構造・情報フィルター）を通して悩みを掘り下げる
- 無料記事は最低1本、最大3本
- テンプレパターンは連続で同じにしない
- タイトル案に誇大表現・煽りを含めない
- クラスター分析は「読者にとって1つの商品で解決される価値があるか」で判断
