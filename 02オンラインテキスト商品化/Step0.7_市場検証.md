## リサーチプロンプト1: 競合商品調査

```
以下のテーマについて、2025-2026年の最新情報を中心に調査してください。

テーマ: 「（Step0で特定したターゲットのジャンル）」に関する有料テキスト商品の市場調査

調査してほしいこと:

1. note で「（ターゲットの悩みに関するキーワード3-5個）」に関連する有料記事を調査
   - 上位10-20件の記事タイトル、価格、スキ数
   - 価格帯の分布（100-500円、500-1000円、1000-5000円、5000円以上がそれぞれ何割か）
   - 最も売れている記事の共通点（タイトルの傾向、価格帯、内容の傾向）

2. Brain で同ジャンルの商品を調査
   - 上位10-20件の商品タイトル、価格、レビュー数、平均評価
   - 紹介料の設定（何%が多いか）
   - レビューに書かれている「良かった点」と「不満点」の傾向

3. 競合商品の構成パターン
   - 文字数の相場（どのくらいのボリュームが多いか）
   - テンプレート・付属物をつけている商品の割合
   - 無料部分の長さ（どこまで見せているか）

4. 価格と価値の関係
   - 1000円以上で売れている商品に共通する「含まれているもの」
   - 5000円以上の高単価商品の特徴
   - 「高い」「安い」とレビューで言及されている商品の境目

出力形式:
- 各調査項目について具体的なデータと事例を箇条書きで
- 情報源のURL（確認できる場合）
- 「確認できた事実」と「推測」を明確に区別すること
```

## リサーチプロンプト2: ターゲットの悩み実態調査

```
以下のテーマについて調査してください。

テーマ: 「（ターゲットのペルソナ）」が抱える悩みの実態とニーズ

調査してほしいこと:

1. X（Twitter）での悩みの語られ方
   - 「（悩みに関するキーワード）」で検索したときに出てくる投稿の傾向
   - いいね・RTが多い投稿の共通点
   - 悩みを語っている人のプロフィール傾向（フォロワー数、職業、発信テーマ）

2. Yahoo知恵袋・教えてgoo・Quoraでの質問
   - 「（悩みキーワード）」に関する質問の件数と傾向
   - 最も多い質問のパターン（「◯◯するにはどうすればいいですか」等）
   - 回答に対する「ベストアンサー」の傾向（どんな回答が支持されているか）

3. YouTube での関連コンテンツ
   - 「（悩みキーワード）」で検索した上位動画のタイトル、再生数、チャンネル登録者数
   - コメント欄でよく見られる悩み・質問
   - 視聴者が「もっと知りたい」と言っているテーマ

4. コミュニティ・オンラインサロンでの議論（確認できる範囲で）
   - 関連するオンラインコミュニティでどんな議論がされているか
   - 初心者がつまずきやすいポイント
   - 「お金を払ってでも解決したい」と語られているテーマ

出力形式:
- 各調査項目について具体的な事例とデータを箇条書きで
- 特に「お金を払ってでも解決したい」レベルの強いニーズを持つ悩みをハイライトすること
- 情報源のURL（確認できる場合）
```

## リサーチプロンプト3: 価格設定と商品設計の最適化調査

```
以下のテーマについて調査してください。

テーマ: デジタルコンテンツ（有料note/Brain）の価格設定と商品設計のベストプラクティス

調査してほしいこと:

1. 2025-2026年のnote/Brain市場のトレンド
   - 有料記事の購入者層の変化
   - 売れている価格帯の変化（以前と比べて高単価化しているか等）
   - 「パッケージ商品」（複数特典つき）の普及度

2. 価格設定の成功パターン
   - 初回割引→通常価格の段階戦略の事例
   - 「松竹梅」（複数グレード）で提供している成功事例
   - 返金保証やアフターサポートの有無と売上への影響

3. 商品設計のベストプラクティス
   - テンプレート・ワークシート等の付属物が売上に与える影響
   - 無料部分の最適な割合（全体の何%が効果的か）
   - サムネイル・タイトルの傾向（クリック率が高いパターン）

4. 購入の決め手
   - レビューやクチコミで「購入を決めた理由」として多いもの
   - 「買わなかった理由」として多いもの
   - リピート購入（同じ作者から2つ目を買う）の動機

出力形式:
- 各項目について具体的なデータと事例を箇条書きで
- 実行可能なアクションに落とし込める情報を優先すること
```

---

### プロンプト生成のルール

- 3つのプロンプト内の「（ ）」部分は、Step0の出力から具体的なキーワード・ペルソナ情報を自動で埋めること
- キーワードはStep0の悩みリストから、最も商品の核となる悩みに関連するものを選ぶ
- Premiumの場合は、クラスター全体をカバーするキーワードセットにする
- 各プロンプトは、ユーザーがそのままコピペして Deep Research に貼り付けられる完成形にすること

- **C: 要再設計** — リサーチ結果とStep0の企画に大きなズレがある。商品設計を見直すべき。

**今回の判定: ◯**

## 3. 差別化ポイントの発見

リサーチから見えた、**この商品だけの強み**:
- 競合がやっていないこと: ◯◯
- ターゲットが「あったらいいのに」と言っているのに市場にないもの: ◯◯
- キャラクターの強みを活かせる独自角度: ◯◯

## 4. リサーチで見つかった「使える情報」

本編執筆時に引用・参照できる情報:
- 数字データ: （市場規模、成功率等）
- ターゲットの生の声: （知恵袋やXの投稿から）
- 競合のレビューから見えた読者の本音: 

→ これらは Step1.5 のオリジナル化フェーズで活用する。

---

### 分析のルール

- リサーチ結果の「事実」と「推測」を混同しない
- 修正提案は「なぜそう変えるべきか」の根拠をリサーチデータから引用する
- 修正レベルAの場合も、差別化ポイントの発見は必ず出力する
- 「使える情報」セクションは、後のステップ（特にStep1.5）への橋渡し

## オプション: SNS検証（フォロワーがいる場合）

Deep Researchに加えて、すでにX/YouTubeのフォロワーがいる場合は、以下のSNS検証も実施できます。

### やり方
1. エージェントに「SNS検証もやりたい」と伝える
2. エージェントが投稿文を3パターン生成（共感/問いかけ/チラ見せ）
3. ユーザーが投稿し、3-7日間の反応を取得
4. 結果をエージェントに伝え、Phase Bの分析に追加情報として反映

### 判断基準
| 反応 | 判断 |
|------|------|
| 共感投稿のいいね率が通常の1.5倍以上 | ニーズあり |
| 問いかけにリプライが5件以上 | 強いニーズ |
| 「詳しく知りたい」系リプライが3件以上 | 有料で売れる可能性高い |
