## このステップでやること

有料記事に使う素材を**ディープリサーチで集め、キャラの視点でオリジナル化**します。

3フェーズに分かれています:
- **Phase A**: AIがリサーチ用のプロンプト/検索指示を生成
- **Phase B**: あなたがAIのディープリサーチ機能で実際にリサーチ
- **Phase C**: AIがリサーチ結果をキャラの視点でオリジナル素材に変換

商品グレードによる違い:
- **Lite/Standard**: 1つの悩みに対してリサーチ（現行通り）
- **Premium**: 悩みクラスター全体を横断してリサーチ。各章に必要な素材を一括で収集し、オリジナル化する。

## Phase B: あなたがリサーチを実行

### やること

Phase Aで生成されたリサーチプロンプトを使って、AIのディープリサーチ機能（ChatGPT, Perplexity, Gemini等）でリサーチを実行します。

### 手順

1. Phase Aの出力から「リサーチの優先順位」の順番で進める
2. 各リサーチプロンプトをディープリサーチに入力
3. 出力されたレポートを**そのままコピペで保存**（後でPhase Cに渡す）
4. 全プロンプトのリサーチが終わったらPhase Cへ

【オリジナル化の指示】

以下の4ステップでリサーチ結果をオリジナル素材に変換してください。

### ステップ1: 情報の仕分け

リサーチ結果から使える情報を抽出し、以下に分類:

| 分類 | 情報の概要 | 出典 | 使える場面 |
|-----|----------|------|----------|
| 事実・データ | | | |
| 一般論 | | | |
| 事例 | | | |
| 反論・例外 | | | |
| ターゲットの生の声 | | | |

### ステップ2: キャラの判断フィルターを通す

上記の情報に対して、キャラクター設計書の「判断構造（MBTI/ストレングスファインダー/エニアグラム）」を使い、以下を出力:

**キャラが共感する情報**:
- （情報）→ キャラの判断構造のどこに刺さるか

**キャラが異論を持つ情報**:
- （情報）→ なぜ異論を持つか（判断構造から）

**キャラが無視する情報**:
- （情報）→ なぜ無視するか

### ステップ3: オリジナル素材への変換

以下の4つの変換を行い、それぞれ箇条書きで出力:

**A. キャラの言葉に翻訳**
リサーチで見つけた事実や事例を、キャラの口調・比喩表現で言い換え:
- 元の情報: → キャラの言葉に変換:

**B. キャラの経験として再構築**
リサーチで見つけた事例を、キャラが「裏側で検証した結果」として語り直す:
- 元の事例: → キャラの検証エピソードとして:

**C. 独自フレームワーク化**
リサーチで見つけた散らばった情報を、キャラ独自のフレームワーク（命名つき）に整理:
- フレームワーク名: 
- 構成要素:
- なぜこの整理が有効か:

**D. 逆張りポイントの言語化**
一般論に対してキャラが「いや、実は…」と言える根拠:
- 一般論: → キャラの逆張り: → 根拠:

---

【制約】

- リサーチ結果を「そのまま」コピペで記事に使わないこと
- 全ての事例・データは、キャラの経験・判断を通して語り直すこと
