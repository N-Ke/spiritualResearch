# テンプレートパターン集（8種）

低単価の有料記事を複数出しても読者が飽きないよう、**悩みの性質に応じて異なる構成パターン**を選ぶ。
Step1 でAIが自動提案し、Step2-3 で構成・執筆に反映される。

---

## パターン早見表

| # | パターン名 | 一言で言うと | 向く悩み | タイトル型の例 |
|---|-----------|-----------|---------|-------------|
| A | ステップバイステップ型 | 手順を刻んで迷わせない | 「やり方がわからない」 | 「初心者が迷わない◯◯の3ステップ」 |
| B | 失敗回避型 | 地雷を先に見せる | 「失敗が怖い」 | 「◯◯で失敗する人の共通点5つ」 |
| C | 実験レポート型 | やってみた結果を見せる | 「本当に効果あるの？」 | 「◯◯を2週間試した結果」 |
| D | 比較・選択肢型 | 選び方を整理する | 「どれを選べばいい？」 | 「◯◯ vs △△｜初心者はどっち？」 |
| E | チェックリスト型 | 漏れなく確認できる | 「漏れなくやりたい」 | 「公開前に確認する10項目」 |

## A: ステップバイステップ型

### 構成の骨格

```
【無料部分】
H2: 結論（この記事で得られること）
  - 1行で結論
  - 読後状態（できるようになること）
  - 全ステップの概要図

H2: Step1の詳細
  - 具体的な手順
  - つまずきポイントと対処法
  - 実例1つ

---有料ライン---

【有料部分】
H2: Step2の詳細
  - 手順 + テンプレ
  - 失敗パターン

H2: Step3の詳細
  - 手順 + テンプレ
  - 応用パターン

H2: 注意点・よくある失敗
  - 3つに絞る
  - 各失敗に回避策

H2: まとめ + 次にやること
  - 要点の再掲
  - 今日から始める1アクション
```

### タイトル型
- 「初心者が迷わない◯◯の3ステップ」
- 「◯◯の始め方｜最初の30分でやること」
- 「【完全手順】◯◯を最短で立ち上げる方法」

### 無料部分の設計思想
Step1を完全公開することで「この人の手順解説は分かりやすい」と実感させる。全体像も見せるので、有料部分の価値（残りのステップ）が明確。

### 有料部分の価値
残りの全ステップ + 各ステップのテンプレ + 失敗パターン。「無料で全体像は分かった。あとは具体的にやるだけ」の状態を作る。

---

## B: 失敗回避型

### 構成の骨格

```
【無料部分】
H2: こんな失敗していませんか？（共感）
  - あるある失敗を3つ
  - 読者の「あ、自分のことだ」を引き出す

H2: 失敗の原因を分解する
  - 3-5つの原因（行動/環境/思考に分類）
  - 各原因に具体例

---有料ライン---

【有料部分】
H2: 正しい手順（失敗を避ける方法）
  - 原因ごとの対処法
  - 手順を番号で

H2: チェックリスト（コピペ用）
  - 実行前に確認する項目
  - Yes/No形式

H2: 「それでもうまくいかない」場合
  - 例外パターン1-2個
  - 別ルートの提案

H2: まとめ
  - 失敗パターンの再掲
  - 今日やめること1つ
```

### タイトル型
- 「◯◯で失敗する人の共通点5つ」
- 「やってはいけない◯◯｜初心者が陥る3つの罠」
- 「◯◯がうまくいかない本当の理由」

### 無料部分の設計思想
「失敗あるある」で強い共感を作り、「原因の分解」で知的満足を与える。ここだけ読んでも「何が問題か」が分かるので、単独で価値がある。

### 有料部分の価値
原因が分かった状態で「じゃあどうすればいい？」の正解手順 + チェックリスト。問題認識 → 解決策の流れが自然。

---

## C: 実験レポート型

### 構成の骨格

```
【無料部分】
H2: なぜこの実験をしたか（背景・動機）
  - きっかけの状況
  - 仮説（「◯◯すれば△△になるはず」）

H2: 結果サマリ（数字で見せる）
  - Before/After を数値で
  - 一言コメント

---有料ライン---

【有料部分】
H2: やったことの詳細（施策ログ）
  - 日付/行動/結果を時系列で
  - 判断の根拠も書く

H2: うまくいったポイント
  - 3つに絞る
  - 再現のコツ

H2: うまくいかなかったこと
  - 2-3個
  - 次にどう変えるか

H2: 再現する場合の手順
  - 読者がそのまま真似できる形
  - テンプレ/ツール紹介

H2: まとめ + 次の実験予告
  - 学びの要点
  - 「次は◯◯を試す」
```

### タイトル型
- 「◯◯を2週間試した結果｜正直レビュー」
- 「◯◯で月◯◯を目指す実験｜◯ヶ月目の報告」
- 「実際にやってみた◯◯のメリット・デメリット」

### 無料部分の設計思想
「なぜやったか」と「結果の数字」を先に見せる。結果サマリが刺さった人は「詳細が知りたい」と有料部分へ進む。数字があるので説得力が高い。

### 有料部分の価値
結果だけでなく「どうやって」「なぜうまくいった/いかなかった」の再現情報。読者が同じことを試せる状態にする。

---

## D: 比較・選択肢型

### 構成の骨格

```
【無料部分】
H2: そもそも何を選ぶ場面なのか（前提整理）
  - 読者の状況を言語化
  - 比較する対象を明示

H2: 比較の軸（選び方の基準）
  - 3-5個の比較ポイント
  - 「こういう人はA、こういう人はB」の方針

---有料ライン---

【有料部分】
H2: 選択肢Aの詳細
  - 特徴 / メリット / デメリット / 向く人

H2: 選択肢Bの詳細
  - 同上

H2: 選択肢Cの詳細（あれば）
  - 同上

H2: 結局どれ？（状況別おすすめ）
  - パターン分けで結論
  - 「迷ったらこれ」を1つ

H2: まとめ
  - 比較の再掲
  - 最初の1アクション
```

### タイトル型
- 「◯◯ vs △△｜初心者はどっち？」
- 「◯◯の選び方｜失敗しない3つの基準」
- 「【比較】◯◯を◯つの視点で整理してみた」

### 無料部分の設計思想
「何を選ぶべきか」の基準を先に提示。基準だけでも判断材料になるので、無料部分に価値がある。各選択肢の詳細深掘りは有料で。

### 有料部分の価値
各選択肢の詳細比較 + 状況別のおすすめ。「自分の場合はどれ？」に明確に答える。

---

## E: チェックリスト型

### 構成の骨格

```
【無料部分】
H2: なぜこのチェックリストが必要か
  - チェックなしで起きる失敗例
  - 全体像（何をチェックするか）

H2: 最重要の3項目（完全解説）
  - 項目ごとに「なぜ重要か」「判定基準」「よくある間違い」

---有料ライン---

【有料部分】
H2: 残りの全項目（4-10番）
  - 同じフォーマットで解説
  - 優先度つき

H2: コピペ用チェックシート
  - Markdown / テキスト形式
  - そのまま使える

H2: 使い方ガイド
  - いつ使うか
  - カスタマイズのコツ

H2: まとめ
  - チェックの運用方法
  - 定期見直しの推奨
```

### タイトル型
- 「公開前に必ず確認する10項目」
- 「◯◯のためのセルフチェックリスト」
- 「これを確認しないと◯◯で失敗します」

### 無料部分の設計思想
最重要3項目を完全に解説。これだけでも実用できるので「この人のチェックリストは使える」と実感してもらう。残りの全項目は有料で。

### 有料部分の価値
完全版チェックシート（コピペ可）+ 使い方ガイド。「毎回これを使えばOK」という安心感。

---

## F: Q&A・疑問解消型

### 構成の骨格

```
【無料部分】
H2: はじめに（こんな疑問ありませんか？）
  - 想定質問を一覧表示
  - 「全部答えます」宣言

H2: Q1〜Q3（上位3問の完全回答）
  - 1問1答 → 理由 → 補足 → 次のアクション
  - 具体例つき

---有料ライン---

【有料部分】
H2: Q4〜Q7（残りの質問）
  - 同じフォーマット

H2: 番外編（聞かれにくいけど大事な質問）
  - 1-2問
  - 裏話や本音

H2: まとめ + 質問の受付案内
  - 回答の要点リスト
  - 追加質問の導線
```

### タイトル型
- 「◯◯でよく聞かれる7つの質問」
- 「初心者がつまずく◯◯のQ&A」
- 「◯◯について正直に答えます」

### 無料部分の設計思想
最も多い質問3つに完全回答。「この人は質問にちゃんと答えてくれる」という信頼を作る。各回答が独立しているので途中離脱にも強い。

### 有料部分の価値
残り全問 + 番外編。特に「聞かれにくいけど大事な質問」に本音で答えることで、有料の希少価値を出す。

---

## G: ビフォーアフター型

### 構成の骨格

```
【無料部分】
H2: Before（変わる前の状況）
  - 具体的な状況描写
  - 読者が「自分のことだ」と感じる共感

H2: 転機（何がきっかけで変わったか）
  - きっかけの出来事・気づき
  - 「それまでの常識が崩れた瞬間」

H2: After概要（どう変わったか）
  - 数字で見せるビフォーアフター
  - 生活/仕事がどう変わったか

---有料ライン---

【有料部分】
H2: 再現の具体手順
  - 読者が同じ変化を起こすための手順
  - 番号つきステップ

H2: テンプレ/ツール
  - 実際に使ったもの
  - コピペ用テンプレ

H2: うまくいかないパターンと対処
  - 失敗例2-3個
  - 回避策

H2: まとめ
  - 変化の要点
  - 最初の一歩
```

### タイトル型
- 「月◯万→◯万に変わった◯◯の話」
- 「◯◯を始めて3ヶ月で変わったこと」
- 「◯◯ができなかった僕が、◯◯できるようになるまで」

### 無料部分の設計思想
Before → 転機 → After の「ストーリー」で感情を動かす。After の概要で「自分もこうなれるかも」と思わせ、再現手順（有料）への動機を作る。

### 有料部分の価値
ストーリーで動機づけされた読者に、具体的な「再現手順 + テンプレ + 失敗回避」を提供。感情 → 行動の流れが自然。

---

## H: テンプレ配布型

### 構成の骨格

```
【無料部分】
H2: こんな課題ありませんか？
  - テンプレがない状態の不便さ
  - 自力で作ると何が大変か

H2: テンプレの全体構造（プレビュー）
  - テンプレの項目一覧
  - 「何が手に入るか」を視覚的に見せる
  - サンプル1項目だけ公開

---有料ライン---

【有料部分】
H2: テンプレ本体（コピペ用）
  - 完全版テンプレ
  - Markdown / テキスト形式

H2: 使い方の詳細
  - 項目ごとの書き方ガイド
  - 良い例 / 悪い例

H2: カスタマイズ例（3パターン）
  - 状況別に変えるポイント
  - 応用の考え方

H2: FAQ
  - よくある質問3-5個

H2: まとめ
  - テンプレの使いどころ
  - 定期的に見直す推奨
```

### タイトル型
- 「コピペで使える◯◯テンプレ」
- 「◯◯の型を作ったので配布します」
- 「毎回ゼロから作らなくていい◯◯テンプレート」

### 無料部分の設計思想
テンプレの「全体構造」と「サンプル1項目」を見せる。構造が見えるので「これ欲しい」と判断しやすい。サンプルで品質も確認できる。

### 有料部分の価値
完全版テンプレ + 使い方ガイド + カスタマイズ例。「もらってすぐ使える」が最大の価値。

## テンプレパターンの選び方（Step1で使う判断基準）

### 悩みの性質 → 推奨パターン

| 悩みのキーワード | 推奨パターン | 理由 |
|---------------|------------|------|
| やり方、始め方、手順 | A (ステップ) | 手順を刻むのが最適 |
| 怖い、不安、失敗 | B (失敗回避) | 恐怖を先に取り除く |
| 効果、本当に？、検証 | C (実験レポ) | 実証で納得させる |
| どっち、選び方、比較 | D (比較) | 判断基準を整理 |
| 漏れ、確認、チェック | E (チェックリスト) | 網羅性が価値 |
| 質問、疑問、分からない | F (Q&A) | 細かい疑問を一気に解消 |
| 変わりたい、自信がない | G (ビフォーアフター) | ストーリーで動機づけ |
| 自分で作れない、型がほしい | H (テンプレ配布) | 使える資産を渡す |

### 連続する商品でパターンを分散させるルール

同じパターンが**2本連続しない**ようにする。例:

- 1本目: A (ステップ) → 2本目: B (失敗回避) → 3本目: H (テンプレ) → 4本目: C (実験レポ)

迷ったときの優先順:
1. 悩みの性質に最も合うパターン
2. 直前の記事と被らないパターン
3. キャラの強みが活きるパターン（実験好き→C、構造分析→D/E、共感→G）

# グレード別の構成拡張

各パターン(A-H)の**Lite版**は上記の骨格がそのまま該当する。
以下ではStandard版・Premium版で骨格がどう拡張されるかを定義する。

---

## Standard版の拡張ルール（全パターン共通）

Liteの骨格をベースに、以下の3要素を追加する。

### 追加要素1: 深掘りセクション
有料部分の各H2に、H3レベルの深掘りを1-2個追加:
- 「具体例をもう1つ」（事例の追加）
- 「よくある質問」（そのセクションに対するFAQ 2-3個）
- 「応用パターン」（基本の手順を応用する場合の変形）

### 追加要素2: 付属テンプレートセクション
記事末（まとめの前）に独立したH2を追加:
```
H2: 付属テンプレート
  H3: テンプレ①の使い方
  H3: テンプレ①本体（コピペ用）
  H3: テンプレ②の使い方（あれば）
  H3: テンプレ②本体（あれば）
```

### 追加要素3: 図解ポイント
以下の箇所に図解を配置:
- フレームワーク紹介時 → フレームワーク図
- 手順の全体像 → フロー図
- ビフォーアフター → 比較図

---

## Standard版の骨格（パターン別）

### A-Standard（ステップバイステップ型）
```
【無料部分】
H2: 結論 + 全ステップの概要図 【図解: フロー図】
H2: Step1の詳細（実例つき）
H2: Step2の詳細（実例つき）

---有料ライン---

【有料部分】
H2: Step3の詳細 + 応用パターン
H2: Step4の詳細（あれば）
H2: よくある失敗と対処法
  H3: 失敗パターン1-3
  H3: 各失敗のQ&A
H2: 付属テンプレート
  H3: ステップ実行チェックリスト
  H3: 計画テンプレート
H2: まとめ + 次にやること
```

### B-Standard（失敗回避型）
```
【無料部分】
H2: こんな失敗していませんか？（共感）
H2: 失敗の原因分解 【図解: 原因マップ】
H2: 最も多い失敗の正しい対処法（1つだけ詳しく）

---有料ライン---

【有料部分】
H2: 残りの失敗の正しい手順
  H3: 失敗ごとの対処法 + 応用パターン
H2: 「それでもうまくいかない」Q&A
H2: 付属テンプレート
  H3: 失敗回避チェックリスト
H2: まとめ
```

### C-Standard（実験レポート型）
```
【無料部分】
H2: なぜこの実験をしたか
H2: 結果サマリ 【図解: ビフォーアフター図】
H2: 最も重要な発見（1つ深掘り）

---有料ライン---

【有料部分】
H2: やったことの全詳細（時系列ログ）
H2: うまくいった3ポイント + 応用パターン
H2: うまくいかなかったこと + Q&A
H2: 再現手順
H2: 付属テンプレート
  H3: 実験記録テンプレート
  H3: 再現チェックリスト
H2: まとめ + 次の実験予告
```

### D〜H のStandard版
（共通ルール: Liteの骨格 + 深掘りセクション + 付属テンプレ + 図解3-5枚）
パターン固有の拡張は、悩みの性質に応じてStep2でAIが自動設計する。

## Premium版の構成ルール（全パターン共通）

Premiumは「1パターンの拡大版」ではなく、**複数の悩みを章ごとに解決する構成**。
各章にLiteレベルのパターンを割り当て、全体を1つの商品にまとめる。

### 章構成テンプレート

```
【無料部分（導入章）】
導入章: この商品で何が変わるか
  H2: ターゲットの悩みの全体像 【図解: 悩みマップ】
  H2: このガイドのゴール
  H2: 全体のロードマップ 【図解: ロードマップ図】
  H2: 各章のプレビュー（1章ずつ2-3行で紹介）

---有料ライン---

【有料部分】
第1章: 悩みAの解決（パターンX）
  H2-H3: Liteレベルの骨格
  末尾: この章のまとめ + 次章への橋渡し
  【図解: 1-2枚】

第2章: 悩みBの解決（パターンY）
  H2-H3: Liteレベルの骨格
  末尾: この章のまとめ + 次章への橋渡し
  【図解: 1-2枚】

第3章: 悩みCの解決（パターンZ）
  （同様の構成）

... 悩みクラスターの数だけ繰り返し ...

統合章: 全体を繋げる実践法
  H2: 各章の解決策をどう組み合わせるか
  H2: 優先順位の決め方 【図解: 優先順位マトリクス】

  H2: 各テンプレートの使い方
  H2: 各ワークシートの使い方
```

### Premium版の各章でのパターン割り当て

各章には悩みの性質に合ったパターン（A-H）を割り当てる。
同じパターンが連続しないよう分散させるのはLiteと同じルール。

例（ウラアドの場合）:
- 第1章: 悩み「アドセンス依存の脱却」→ パターンA（ステップ）
- 第2章: 悩み「非属人の信頼構築」→ パターンC（実験レポ）
- 第3章: 悩み「商品を作れない壁」→ パターンG（ビフォーアフター）
- 第4章: 悩み「概要欄の導線設計」→ パターンB（失敗回避）
- 統合章: 全体を繋げる → パターンA（ステップ）的な構成
- ロードマップ章: パターンE（チェックリスト）的な構成

### Premium版の付属物設計ガイド

| 章の位置 | 推奨する付属物 |
|---------|-------------|
| 導入章 | 現状診断ワークシート、現状診断AIツール |
| 各本編章 | その章の実行テンプレート |
| 統合章 | 優先順位決定テンプレート |
| ロードマップ章 | アクションプランワークシート、進捗管理テンプレート |
