# note記事チェックリスト

Step4（品質チェック）で使用するチェック項目の全一覧。
各チェック項目にはIDがついており、修正リストで参照できます。

---

## 1. プラットフォーム規約準拠（C01-C08）

| ID | チェック項目 | 判定基準 |
|----|-----------|---------|
| C01 | 誇大広告・虚偽の効果保証 | 「確実に」「絶対に」「100%」「必ず成功」等の断定表現がない |
| C02 | 薬機法・健康系の断定 | 「◯◯が治る」「◯◯に効く」等の医学的断定がない |
| C03 | 金融商品の断定的判断 | 「必ず儲かる」「損しない」等の投資成果の保証がない |
| C04 | 差別・誹謗中傷 | 特定の個人・団体・属性への攻撃的表現がない |
| C05 | 著作権侵害の引用 | 他者の文章を引用する場合、出典明記＋引用の範囲が適切 |
| C06 | 個人情報の露出 | 第三者の本名、連絡先、所属先が許可なく記載されていない |
| C07 | 公序良俗違反 | 反社会的行為の推奨、違法行為の助長がない |
| C08 | AI利用の表記 | AI生成を活用した場合、記事末に表記がある（note推奨事項） |

---

## 2. キャラクター一貫性（C09-C14）

| ID | チェック項目 | 判定基準 |
|----|-----------|---------|
| C09 | 一人称 | キャラクター設計書で指定された一人称が全文で統一されている |
| C10 | よく使う表現 | 設計書の「よく使う表現」が記事内に自然に含まれている |
| C11 | 禁止表現 | 設計書の「絶対に使わない表現」が記事内に含まれていない |
| C12 | 判断構造との整合 | MBTI/SF/エニアグラムの特性と矛盾する主張がない |
| C13 | トーンの統一 | 記事の前半と後半で口調・テンションが乖離していない |
| C14 | 情報フィルターの整合 | キャラが「重視しない」情報が記事の主軸になっていない |

---

## 3. 可読性（C15-C20）

| ID | チェック項目 | 判定基準 |
|----|-----------|---------|
| C15 | 1文の長さ | 40文字を大幅に超える文が連続していない |
| C16 | 改行 | 2-3文ごとに改行があり、スマホ表示で読みやすい |
| C17 | 漢字率 | 目視で漢字が密集していない（目安30%以下） |
| C18 | 専門用語の説明 | 専門用語の初出時に1行以内の説明がある |
| C19 | 見出しの魅力 | H2/H3が目次で見て「読みたい」と思える表現になっている |
| C20 | リード文の長さ | 150-300文字で収まっている |

---

## 4. オリジナル性（C21-C24）

| ID | チェック項目 | 判定基準 |
|----|-----------|---------|
| C21 | コピペ感 | リサーチ結果をそのまま転載した感のある箇所がない |
| C22 | 事例の変換 | 事例がキャラの経験として語り直されている |
| C23 | キャラ視点の付加 | 一般論だけで終わる見出しがない（キャラの独自解釈がある） |
| C24 | フレームワークの活用 | Step1.5で作った独自フレームワークが本文で使われている |

---

## 5. 無料部分の品質（C25-C28）

| ID | チェック項目 | 判定基準 |
|----|-----------|---------|
| C25 | 単独価値 | 無料部分だけ読んでも「読んでよかった」と思える |
| C26 | 具体性 | 結論の概要、手順の一部、事例のいずれかが無料部分に含まれている |
| C27 | ティーザー化 | 無料部分が「チラ見せだけ」になっていない |
| C28 | 有料ライン前のセリフ | 押し売り感がなく、自然な流れで有料部分への期待を作っている |

---

## 6. 有料部分の品質（C29-C32）

| ID | チェック項目 | 判定基準 |
|----|-----------|---------|
| C29 | 追加価値 | 無料部分にない具体的情報（詳細手順、テンプレ、事例）がある |
| C30 | テンプレの実用性 | テンプレ/チェックリストがコピペで使える形式になっている |
| C31 | 購入満足度 | 読了後に「買ってよかった」と思えるレベルの具体性がある |
| C32 | CTAの具体性 | 「次にやること」が具体的な1アクションとして提示されている |

---

## 7. 図解の品質（C33-C36）※ Standard/Premium

| ID | チェック項目 | 判定基準 |
|----|-----------|---------|
| C33 | 本文との整合 | 図解プレースホルダーの内容が前後の本文と一致している |
| C34 | 説明テキスト | 図解の alt text 相当の説明が適切に書かれている |
| C35 | 種類の適切性 | 図解の種類（フロー/比較表/マップ等）が内容に合っている |
| C36 | 数量基準 | Standard: 3枚以上 / Premium: 8枚以上 |

---

## 8. 付属物の品質（C37-C42）※ Standard/Premium

| ID | チェック項目 | 判定基準 |
|----|-----------|---------|
| C37 | 本編との対応 | 付属物が本編の具体的な章/見出しと対応している |
| C38 | 使い方ガイド | 各付属物に使い方ガイドが添付されている |
| C39 | 記入例/使用例 | 各付属物に具体的な記入例/使用例がある |
| C40 | テンプレの空欄 | テンプレの記入箇所が【　　　】形式で明確 |
| C41 | AIツールの動作 | AIツールのプロンプトがコピペで動作する形式 |
| C42 | WSの問いかけ | ワークシートの問いが具体的で答えやすい形 |

---

## 9. 章間一貫性（C43-C47）※ Premium のみ

| ID | チェック項目 | 判定基準 |
|----|-----------|---------|
| C43 | 章間接続 | 各章の冒頭が前章の末尾と自然に繋がっている |
| C44 | 用語統一 | 章をまたいで同じ概念に同じ用語が使われている |
| C45 | 悩みカバー | 悩みクラスターの全ての悩みが本編でカバーされている |
| C46 | 統合章の整合 | 統合章で各章の内容が適切にまとめ・横断されている |
| C47 | タイムライン整合 | ロードマップ章のタイムラインが本編の内容と矛盾しない |

---

## 10. 商品としての価値（C48-C50）※ Premium のみ

| ID | チェック項目 | 判定基準 |
|----|-----------|---------|
| C48 | ボリューム | 価格に見合う文字数・章数・図解数がある |
| C49 | 付属物の本質性 | 付属物が「おまけ」ではなく本質的な価値を持っている |
| C50 | 変革実感 | 読者がこの商品で「変われる」と感じられる構成になっている |

---

## 使い方

1. Step4のプロンプトにこのチェックリストのID体系が組み込まれています
2. AIが出力する修正リストは `C01`, `C15` のようにIDで参照します
3. 修正漏れがないよう、チェック結果のサマリで「要修正」になった観点を優先的に確認してください
4. Lite は C01-C32 のみ、Standard は C01-C42、Premium は C01-C50 が対象です
