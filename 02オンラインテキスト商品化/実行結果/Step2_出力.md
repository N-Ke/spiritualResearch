# Step2 出力: 構成設計

## 1. 記事全体の設計

- **文字数の目安**: 全体 約5,500文字（無料: 約2,200文字 / 有料: 約3,300文字）
- **見出し数**: H2 7個 / H3 9個
- **読了時間の目安**: 約12分

## 2. 無料部分のアウトライン

### H2: なぜ「非属人×リストマーケ」を実験しようと思ったか

#### H3: アドセンス12年目の焦り
- 内容の要約: アドセンスで安定して稼いできたが、アルゴリズム変動で再生数が半減した月に「このままじゃまずい」と感じた。収益源が1本だけの恐怖。でもターゲットと同じ「非属人」だから、顔出しせずにどうにかしたい。
- 使うオリジナル素材: ターゲットの声①③
- 想定文字数: 350文字

#### H3: 「非属人で信頼構築なんて無理」という思い込み
- 内容の要約: 最初は自分も「顔出しなしでリスト取れるわけない」と思っていた。属人チャンネルのコンサル生が次々リスト取得に成功していくのを横目に、「非属人は仕組みが違う」と諦めかけた。
- 使うオリジナル素材: ターゲットの声①④、逆張りポイント①の前振り
- 想定文字数: 300文字

### H2: 結果サマリ — 2週間で47件のリスト登録

#### H3: ビフォー・アフターの数字
- 内容の要約: 変更前: 概要欄クリック率0.3%、リスト登録数 2週間で5件。変更後: クリック率0.5%、リスト登録数 2週間で47件。顔も名前も出していない。
- 使うオリジナル素材: 事例①（概要欄の文言変更実験）
- 想定文字数: 250文字

#### H3: 「面白い発見があったんだけど」
- 内容の要約: 数字の変化だけでなく、登録者の「質」にも変化があった。登録後のメール開封率が高い。「無料プレゼント目当て」ではなく「この人の情報をもっと知りたい」で登録した人が多い印象。
- 使うオリジナル素材: 主張⑤（リスト取得の本質は"次も会いたい"）
- 想定文字数: 300文字

### 無料部分の価値チェック
- **無料だけ読んだ人が得られるもの**: 「非属人でもリスト取得ができる」という実証データと心理的なブレイクスルー。概要欄の文言変更という具体的なヒント。
- **「有料も読みたい」と思う理由**: 「何を変えたか」の概要は分かったが、「なぜそれで登録率が変わるのか」の構造と「自分も再現するための手順」が有料部分にある。
- **無料部分が「薄い」と感じないポイント**: 具体的な数字（47件、1.8倍）が出ている。概要欄の文言変更という実行可能なヒントも含まれている。

**境界の直前に書くセリフ**:

## 4. 有料部分のアウトライン

### H2: やったことの詳細 — 何を変えたか、なぜ変えたか

#### H3: 仮説「信頼の正体は顔じゃなくて体験密度」
- 内容の要約: 実験の出発点になった仮説を説明。信頼の3層モデル（能力の信頼→一貫性の信頼→体験の信頼）を紹介し、非属人でも全層を構築できる理屈を提示。12年のコンサル経験で、属人でも信頼構築に苦戦する人と非属人でも信頼される人の違いに気づいた話。
- 使うオリジナル素材: フレームワーク①（信頼の3層モデル）、事例③（属人vs非属人の比較）、主張①
- 想定文字数: 500文字

#### H3: 概要欄で変えた3つのポイント
- 内容の要約: 概要欄の3要素チェック（具体的な体験 / 読者メリット / ハードル下げ）を紹介。変更前の文言と変更後の文言を具体的に並べて比較。なぜ「具体的な体験」を足すだけで結果が変わるのかを解説。
- 使うオリジナル素材: フレームワーク②（概要欄の3要素チェック）、主張②、事例①
- 想定文字数: 500文字

### H2: うまくいったポイント — 再現のコツ3つ

#### H3: コツ1「情報の具体性で信頼を稼ぐ」
- 内容の要約: 非属人の武器は「情報の質がダイレクトに伝わる」こと。顔がないからノイズがなく、コンテンツの価値そのものが評価される。これを活かすには、曖昧な表現を排除して具体的な数字・手順を書く。
- 使うオリジナル素材: 逆張りポイント①、主張③
- 想定文字数: 300文字

#### H3: コツ2「最小ルートで結果を出す」
- 内容の要約: ブランディングの統一（ロゴ、声、世界観）は後回しでいい。まず「1つの動画→概要欄→登録」の最短ルートだけ整える。概要欄の3要素を満たすだけでまず結果を出す。
- 使うオリジナル素材: 逆張りポイント②、主張④
- 想定文字数: 250文字

#### H3: コツ3「"次も会いたい"を設計する」
- 内容の要約: リスト取得の本質は「次の情報も知りたい」という感情。フリーギフトは「もらえるもの」ではなく「体験の延長」として設計する。登録後の最初のメールが最も重要。
- 使うオリジナル素材: 主張⑤、ターゲットの声②
- 想定文字数: 250文字

### H2: うまくいかなかったこと — 正直に書く

- 内容の要約: 最初に試した「無料プレゼント」形式は失敗した。登録率は低く、登録してもメール開封率が低かった。「無料だから登録した人」はエンゲージが低い。この失敗から「体験の密度」という仮説にたどり着いた。もう1つの失敗: CTA位置を動画の冒頭にしたら離脱が増えた。
- 使うオリジナル素材: 事例②（質問コメントのエピソード）の対比として使用
- 想定文字数: 350文字

### H2: 再現する場合の手順 — あなたのチャンネルで試すなら

- 内容の要約: 読者が自分のチャンネルで再現するための5ステップ手順。①概要欄の現状を3要素チェックで診断 → ②「具体的な体験」要素を追加 → ③フリーギフトの内容を見直す → ④2週間計測 → ⑤結果をもとに調整。チャンネルテーマ別の概要欄テンプレ例も。
- 使うオリジナル素材: フレームワーク②を実行手順に変換
- 想定文字数: 500文字

### H2: まとめ + 次の実験予告

- 内容の要約: この実験で分かったことの要約（信頼の正体、概要欄の3要素、"次も会いたい"の設計）。次は「リスト登録後のステップメールで、非属人のまま商品販売まで持っていけるか」を実験予定。「まず小さく、概要欄の文言を変えることから試してみませんか？」
- 使うオリジナル素材: ターゲットの声④への応答
- 想定文字数: 300文字

## 5. リード文の設計

「非属人チャンネルでリストマーケなんて無理でしょ」——僕もそう思ってた。顔も名前も出してないのに、誰がメールアドレスを預けてくれるんだと。でも、ちょっと実験してみたんです。概要欄の文言を1箇所変えただけで、2週間で47件のリスト登録。この記事では、何を変えて、なぜそれで結果が出たのかを、正直に全部書きます。前半は結果サマリ（無料で読めます）。後半は具体的な手順と失敗記録です。

想定文字数: 約200文字
