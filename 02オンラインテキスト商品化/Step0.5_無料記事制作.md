# Step0.5: 無料記事制作（信頼構築用）

---

## 【エージェント指示】

```
読み込むファイル:
  - 入力/ フォルダ内のキャラクター設計書
  - 入力/ フォルダ内のAI実行用プロンプト
  - 実行結果/Step0_出力.md
  - ナレッジ/テンプレートパターン集.md
出力先: 実行結果/Step0.5_無料記事_出力.md
次のステップ: Step1_テンプレ選択＆角度決め.md
```

**エージェントの実行手順:**
1. 上記ファイルを全て読み込む
2. Step0の出力から無料記事の企画（セクション3）を取得する
3. 下の【実行プロンプト】に従って無料記事を生成する
4. 出力を `実行結果/Step0.5_無料記事_出力.md` に保存する
5. ユーザーに確認を取る
6. OKなら次のステップ（Step1）を案内する

---

## このステップでやること

有料記事を出す前に、キャラの人柄と専門性を伝える**無料記事を1-2本**作ります。この無料記事が「この人の記事は価値がある」と思ってもらう入口になります。

このステップはエージェントが自動実行します。

---

## 【実行プロンプト】

```
あなたは以下のキャラクターとして、note/Brain向けの無料記事を執筆します。

---

【AI実行用プロンプト（キャラクターの振る舞い定義）】
（エージェントが 入力/AI実行用プロンプト から自動読み込み）

---

【キャラクター設計書】
（エージェントが 入力/キャラクター設計書 から自動読み込み）

---

【無料記事の企画（Step0の出力から）】
（エージェントが 実行結果/Step0_出力.md のセクション3から自動読み込み。記事の型・タイトル・概要・導線先の有料記事を含む）

---

【執筆の指示】

上記のキャラクターとして、以下のルールに従って無料記事を書いてください。

■ 記事の構成ルール（型に応じて）

【自己紹介型の場合】
1. フック: ターゲットの悩みに共感する1-2文
2. 自分の紹介: 何をしている人か、なぜこの発信をしているか
3. 過去の自分: ターゲットと同じ悩みを持っていた時期
4. 転機: 何がきっかけで変わったか
5. 今やっていること: 裏で実験していること、共有したいこと
6. 読者への約束: この場で何を発信していくか
7. 導線: 「もっと具体的に知りたい方は…」（自然に有料記事へ）

【価値提供型の場合】
1. フック: ターゲットの悩みを具体的に描写
2. 結論: この記事で得られること（1文）
3. 本題: 悩みの概要レベルの解決策（全体像 + 最初の一歩）
4. 具体例: キャラの経験ベースで1-2個
5. まとめ: 要点の再掲
6. 導線: 「もっと詳しい手順/テンプレは…」（自然に有料記事へ）

【悩みマップ型の場合】
1. フック: 「こんな悩み、ありませんか？」
2. 悩みの全体像: ターゲットの悩みを整理・分類
3. 各悩みの概要: それぞれ3-5行で「何が問題か」「どうすればいいか」のヒント
4. 自分のスタンス: キャラとしてどの順番で解決していくか
5. 導線: 各悩みに対応する有料記事への案内

■ 文体ルール
- キャラクターの「よく使う表現」を自然に使う
- キャラクターの「絶対に使わない表現」は使わない
- 一人称はキャラ設計書の指定に従う
- 2-3文ごとに改行（スマホで読みやすく）
- 1文は短く（句読点で区切る）
- 文字数: 1500-3000文字

■ 導線ルール
- 記事の末尾に、有料記事への導線を**さりげなく**置く
- 「買ってください」ではなく「もっと詳しく知りたい方は」のトーン
- キャラの「まず一歩だけ示す」スタイルを活かす
- 押し売り感が出ないこと

■ 出力フォーマット
- 記事タイトル
- 本文（Markdown形式、H2見出しつき）
- 末尾の導線テキスト
```

## セルフチェック（公開前に確認）

- [ ] キャラの口調が一貫しているか（設計書の表現パターンと照合）
- [ ] 禁止表現（煽り、命令形、無責任な保証）が含まれていないか
- [ ] 無料記事だけ読んでも「この人の記事また読みたい」と思えるか
- [ ] 有料記事への導線が自然か（押し売り感がないか）
- [ ] スマホで読みやすい改行になっているか
- [ ] 文字数が1500-3000文字の範囲か
