# Step3.5: 付属物制作（Standard / Premium 専用）

---

## 【エージェント指示】

```
読み込むファイル:
  - 入力/ フォルダ内のキャラクター設計書
  - 入力/ フォルダ内のAI実行用プロンプト
  - 実行結果/Step2_出力.md（付属物仕様セクション）
  - 実行結果/Step3_*.md（本編の全パート）
出力先:
  - Phase A: 実行結果/Step3.5_PhaseA_提案.md
  - Phase B: 実行結果/Step3.5_PhaseB_設計_（付属物名）.md（付属物ごと）
  - Phase C: 実行結果/Step3.5_PhaseC_（付属物名）.md（付属物ごと）
次のステップ: Step4_編集＆品質チェック.md
```

**エージェントの実行手順:**
1. 上記ファイルを全て読み込む
2. Phase Aを実行: 付属物の提案を生成 → `実行結果/Step3.5_PhaseA_提案.md` に保存
3. ユーザーに確認: どの付属物を作るか選んでもらう（提案内容を見せて確認）
4. ユーザーが選んだ付属物それぞれについて:
   a. Phase Bを実行: 設計を生成 → `実行結果/Step3.5_PhaseB_設計_（名前）.md` に保存 → 確認
   b. Phase Cを実行: 完成品を生成 → `実行結果/Step3.5_PhaseC_（名前）.md` に保存 → 確認
5. 全付属物の制作完了後、次のステップ（Step4）を案内

---

## このステップでやること

本編が完成した後、読者の行動を変える**付属物（AIツール・テンプレート・ワークシート）** を制作します。

初心者でも迷わないよう、**3フェーズ**に分かれています:

| フェーズ | あなたがやること | AIがやること |
|---------|---------------|------------|
| **A: 提案** | 本編の完成原稿を貼るだけ | 「こういう付属物があると読者の行動が変わります」を提案 |
| **B: 設計** | 作るものを選ぶだけ | 各付属物の詳細設計 + 制作手順を番号つきで出力 |
| **C: 制作** | プロンプトをコピペ実行 | 完成品 + 使い方ガイドを出力 |

## Phase A: 付属物の提案

### このフェーズでやること
本編の完成原稿を渡すと、AIが「この内容なら、こういう付属物があると読者が変わる」と**具体的に提案**します。

何を作ればいいか自分で考える必要はありません。

このフェーズはエージェントが自動実行します。

【付属物の種類カタログ】

以下の種類から、本編に合うものを選んで提案してください。

■ AIツール
読者がAIに貼って繰り返し使えるプロンプト。
- 診断ツール: 読者の現在地を把握する（「あなたの◯◯レベルは？」）
- 生成ツール: 繰り返し使う作業を自動化する（「◯◯を自動生成」）
- 分析ツール: データや状況を構造化する（「◯◯を分析して整理」）
- 改善ツール: 読者の成果物をAIが改善提案する（「◯◯を添削」）

■ テンプレート
空欄を埋めるだけで使える型。
- 設計テンプレ: 何かを作るときの設計書（「◯◯設計シート」）
- 計画テンプレ: 行動計画を型に沿って作る（「◯◯計画書」）
- チェックリスト: Yes/Noで確認する項目リスト（「◯◯確認リスト」）
- 文言テンプレ: コピペで使える文章の型（「◯◯テンプレート」）

■ ワークシート
問いに答えながら自分の状況を整理するシート。
- 現状診断WS: 今の状態を客観視する（「◯◯セルフチェック」）
- 目標設定WS: 目指す状態を具体化する（「◯◯ゴール設計」）
- 振り返りWS: 実行後の改善点を洗い出す（「◯◯振り返りシート」）
- アクションWS: 具体的な行動計画を作る（「◯日アクションプラン」）

■ チェックリスト
作業の前後で確認するYes/Noリスト。

■ スプレッドシート設計書
数値管理・進捗追跡用（列の定義・計算式・使い方を設計書として出力）。
