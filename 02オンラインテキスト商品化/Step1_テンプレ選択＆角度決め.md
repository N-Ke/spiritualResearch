# Step1: テンプレ選択 & 角度決め

---

## 【エージェント指示】

```
読み込むファイル:
  - 入力/ フォルダ内のキャラクター設計書
  - 入力/ フォルダ内のAI実行用プロンプト
  - 実行結果/Step0_出力.md
  - ナレッジ/テンプレートパターン集.md
出力先: 実行結果/Step1_出力.md
      （Premiumの場合: 実行結果/Step1_第N章_出力.md を章ごとに出力）
次のステップ: Step1.5_ディープリサーチ＆オリジナル化.md
```

**エージェントの実行手順:**
1. 上記ファイルを全て読み込む
2. Step0の出力から、制作対象の商品を特定する
3. グレードを確認:
   - Lite/Standard: 1回実行。出力を `実行結果/Step1_出力.md` に保存
   - Premium: 章ごとに繰り返し実行。各章の出力を `実行結果/Step1_第N章_出力.md` に保存。各章で確認を取る
4. ユーザーに確認を取る
5. OKなら次のステップ（Step1.5）へ

---

## このステップでやること

Step0の商品企画から1本を選び、以下の3つを確定させます:
※ **Premium** 商品の場合は、章ごとにこのステップを実行します（各章ごとにテンプレパターン・差別化角度・タイトルを決定）。
1. **テンプレパターン**（A-Hのどれを使うか）
2. **差別化角度**（この記事ならではの切り口）
3. **タイトル**（確定版）

## 使い方

このステップはエージェントが自動実行します。

【テンプレパターン一覧】

以下の8種から1つ選んでください。

A: ステップバイステップ型 -- 手順を刻んで迷わせない
   向く悩み: やり方がわからない
   
B: 失敗回避型 -- 地雷を先に見せる
   向く悩み: 失敗が怖い

C: 実験レポート型 -- やってみた結果を見せる
   向く悩み: 本当に効果あるの？

D: 比較・選択肢型 -- 選び方を整理する
   向く悩み: どれを選べばいい？

E: チェックリスト型 -- 漏れなく確認できる
   向く悩み: 漏れなくやりたい

F: Q&A・疑問解消型 -- 細かい疑問を一気に解消
   向く悩み: 質問が多い

G: ビフォーアフター型 -- 変化のストーリーで動かす
   向く悩み: 変われるか不安

H: テンプレ配布型 -- コピペで使える資産を渡す
   向く悩み: 自分で考えるのが苦手

【出力フォーマット】

## 1. テンプレパターンの決定

- **選択パターン**: （A-H の記号と名前）
- **選択理由**: 悩みの性質とこのパターンが合う理由（2文以内）
- **Step0の推奨と異なる場合の理由**: （同じなら「推奨通り」）
- **直前の記事との差別化**: （初回なら「初回のため不要」）

## 2. 差別化角度

同じ悩みを扱う他の記事と「何が違うか」を明確にしてください。

- **一般的な解決策**: この悩みに対して世間で言われる一般論（1-2個）
- **キャラだけの切り口**: キャラクター設計書の「判断構造」「情報フィルター」を使って、このキャラならではの視点を提案してください（2-3個）
  - 各切り口について「なぜこのキャラがこの視点を持つか」を1文で説明
- **採用する角度**: 上記から1つ選び、記事の軸にする
- **ワンフレーズ化**: 採用した角度を1フレーズ（15文字以内）で言語化

## 3. タイトル案

以下の基準で5案出してください:
- 15-25文字
- 差別化角度が伝わる
- テンプレパターンのタイトル型に準拠
- 煽り・誇大表現を避ける
- キャラの口調が反映されている

タイトル案:
1. 
2. 
3. 
4. 
5. 

**推奨タイトル**: （5案から最も適切なものを1つ選び、理由を1文で）

## 4. 記事の方向性サマリ

- テンプレパターン: 
- 差別化角度（ワンフレーズ）: 
- 確定タイトル: 
- 商品グレード: 
- ターゲットの悩み: 
- 提供価値: 
- 記事を読んだ人の理想状態:

【制約】

- テンプレパターンは直前の記事と同じものを選ばないこと
- 差別化角度は「キャラの判断構造」から導くこと（一般論にしない）
- タイトルはキャラの表現パターン（設計書の「よく使う表現」「禁止表現」）に準拠すること
- 出力の「4. 記事の方向性サマリ」は次のステップにそのまま渡すので、必ず出力すること
