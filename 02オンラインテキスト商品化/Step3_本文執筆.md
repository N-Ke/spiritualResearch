# Step3: 本文執筆

---

## 【エージェント指示】

```
読み込むファイル:
  - 入力/ フォルダ内のキャラクター設計書
  - 入力/ フォルダ内のAI実行用プロンプト
  - 実行結果/Step1_出力.md（Premiumの場合: 全章分）
  - 実行結果/Step1.5_PhaseC_出力.md
  - 実行結果/Step2_出力.md
出力先:
  - Lite: 実行結果/Step3_無料部分.md → 実行結果/Step3_有料部分.md
  - Standard: 実行結果/Step3_無料部分.md → Step3_有料前半.md → Step3_有料後半.md
  - Premium: 実行結果/Step3_導入章.md → Step3_第1章.md → Step3_第2章.md → ...
次のステップ:
  - Lite → Step4_編集＆品質チェック.md
  - Standard/Premium → Step3.5_付属物制作.md
```

**エージェントの実行手順:**
1. 上記ファイルを全て読み込む
2. Step0の出力から商品グレードを確認する
3. グレードに応じた分割方法で、1パートずつ執筆する:
   - 各パートの出力をファイルに保存
   - 各パートごとにユーザーに確認を取る
   - 前パートの出力を次パートに引き継ぐ（エージェントが自動読み込み）
4. Premiumの場合: 各章の冒頭で前章の末尾（まとめ+橋渡し）を自動参照し、文脈を繋げる
5. 全パート完了後、次のステップを案内

---

## このステップでやること

Step2のアウトラインに従って、**キャラクターの声で本文を執筆**します。
商品グレードによって執筆の分割方法が異なります。

| グレード | 分割方法 | 実行回数 |
|---------|---------|---------|
| Lite | 無料部分 → 有料部分 | 2回 |
| Standard | 無料部分 → 有料前半 → 有料後半 | 3回 |
| Premium | 導入章 → 各章ごと（5-10回） | 7-12回 |

---

## 【実行プロンプト】Lite / Standard 共通 — 無料部分

```
あなたは以下のキャラクターとして、有料記事の「無料部分」を執筆します。

---

【AI実行用プロンプト（キャラクターの振る舞い定義）】
（エージェントが 入力/AI実行用プロンプト から自動読み込み）

---

【キャラクター設計書】
（エージェントが 入力/キャラクター設計書 から自動読み込み）

---

【記事のアウトライン（Step2の出力）】
（エージェントが 実行結果/Step2_出力.md から自動読み込み）

---

【オリジナル素材集（Step1.5の出力）】
（エージェントが 実行結果/Step1.5_PhaseC_出力.md から自動読み込み）

---

【執筆の指示】

Step2のアウトラインの「無料部分」を執筆してください。

■ 文体ルール
- キャラクター設計書の「表現パターン」に従う
- 2-3文ごとに改行（スマホ最適化）
- 1文は40文字以内を目安
- 漢字率30%以下
- 専門用語は初出時に1行で説明

■ 構成ルール
- リード文から始める
- 各H2/H3はアウトラインに忠実に
- 指定された「使うオリジナル素材」を必ず使う
- 想定文字数に近づける

■ オリジナル素材の使い方
- 「主張」→ キャラの持論として語る
- 「事例」→ キャラの経験として語る
- 「フレームワーク」→ 名前を出して紹介
- 「ターゲットの声」→ 「よく聞くのが〜」として使う
- 「逆張りポイント」→ 記事の山場で使う

■ 図解の挿入
アウトラインで「図解: あり」と指定された箇所に、以下の形式でプレースホルダーを配置:

【図解: （Step2の図解設計から名前をコピー）】
（図解の内容を1-2文で説明するテキスト）

■ 無料部分の品質基準
- 無料だけで「良い記事だった」と思える内容
- 結論の一部、手順の一部、実例が含まれている
- 有料を買わなくても何か持ち帰れる

■ 有料ラインの直前
- Step2で設計した「境界の直前に書くセリフ」を自然に配置

■ 出力フォーマット
- Markdown形式（H2は ## 、H3は ###）
```

【キャラクター設計書】
（エージェントが 入力/キャラクター設計書 から自動読み込み）

【記事のアウトライン（Step2の出力）】
（エージェントが 実行結果/Step2_出力.md から自動読み込み）

（エージェントが 実行結果/Step1.5_PhaseC_出力.md から自動読み込み）

【すでに書いた無料部分の本文】
（エージェントが 実行結果/Step3_無料部分.md から自動読み込み）

【執筆の指示】

有料部分を執筆してください。無料部分の流れを引き継ぎ、違和感なく繋げてください。

■ 文体ルール・構成ルール・図解ルール
（無料部分と同じルールに従う）

■ 有料部分の品質基準

- 無料にない手順/テンプレ/事例を提供
- 「買ってよかった」と思えるレベルの具体性
- テンプレ/チェックリストはコピペで使える形

■ CTA
- Step2で設計したCTAを末尾に配置

■ 出力
- 「--- ここから有料 ---」を冒頭に明記
- 有料部分本文 → CTA の順
```

## 【実行プロンプト】Standard — 有料前半 / 有料後半

Standard は有料部分を2回に分けて執筆します。

### 有料前半
```
（上記Liteの有料部分プロンプトと同じ構成。ただし以下を変更:）

【執筆の指示】

有料部分の「前半」（Step2のアウトラインで前半に配置されたH2群）を執筆してください。

対象のH2: （Step2のアウトラインから前半のH2名を列挙）

■ 追加ルール（Standard）
- 各H2に「深掘りH3」が指定されている場合、応用パターンやFAQとして書く
- 図解プレースホルダーを指定箇所に配置
```

### 有料後半
```
（前半と同じ構成。ただし:）

【すでに書いた部分】
（エージェントが 実行結果/Step3_無料部分.md と 実行結果/Step3_有料前半.md から自動読み込み）

【執筆の指示】
有料部分の「後半」（残りのH2群 + 付属テンプレートセクション + CTA）を執筆してください。

対象のH2: （残りのH2名を列挙）

■ 付属テンプレートセクションの書き方
- Step2の「付属物仕様」に従い、テンプレの使い方を説明
- テンプレ本体の配置場所に「【付属物: ◯◯テンプレート】（Step3.5で制作）」と記載
  ※ テンプレの実物はStep3.5で別途制作するため、ここでは使い方の説明のみ
```

## 【実行プロンプト】Premium — 導入章

```
あなたは以下のキャラクターとして、Premium商品の「導入章」を執筆します。
導入章は無料公開されるため、これだけ読んでも価値がある内容にしてください。

---

【AI実行用プロンプト】
（エージェントが 入力/AI実行用プロンプト から自動読み込み）

【キャラクター設計書】
（エージェントが 入力/キャラクター設計書 から自動読み込み）

【導入章のアウトライン（Step2の出力から導入章部分）】
（エージェントが 実行結果/Step2_出力.md の導入章部分から自動読み込み）

【オリジナル素材集（Step1.5の出力）】
（エージェントが 実行結果/Step1.5_PhaseC_出力.md から自動読み込み）

---

【執筆の指示】

■ 導入章の役割
- 「まず最初にやること3つ」を冒頭に太字で配置する（Step2で設計した内容をそのまま使う）
- ターゲットの悩みの全体像を描く
- このガイドで何が変わるかを伝える
- 全章のプレビューで「読みたい」を作る
- 無料公開に耐える品質にする

■ 「まず最初にやること3つ」の配置ルール
- 導入章のタイトル直後、本文の最初に配置する
- Step2の出力「まず最初にやること3つ」セクションの内容を使う
- 読者が「全部読みきれるかな」と思う前に安心させる
- 3つのアクションは全て「◯分でできる」レベルに絞る

■ 文体・図解ルール
（Lite/Standardと同じ）

■ 有料ラインの直前
- Step2で設計したセリフを配置

■ 出力
- Markdown形式
- 導入章のみ出力
```

## 【実行プロンプト】Premium — 各章（第N章）

各章ごとにこのプロンプトを繰り返し実行します。

```
あなたは以下のキャラクターとして、Premium商品の「第◯章」を執筆します。

---

【AI実行用プロンプト】
（エージェントが 入力/AI実行用プロンプト から自動読み込み）

【キャラクター設計書】
（エージェントが 入力/キャラクター設計書 から自動読み込み）

【この章のアウトライン（Step2の出力から第◯章部分）】
（エージェントが 実行結果/Step2_出力.md の該当章部分から自動読み込み）

【オリジナル素材集（Step1.5の出力）】
（エージェントが 実行結果/Step1.5_PhaseC_出力.md の該当章用素材から自動読み込み）

【前章のまとめ部分（文脈の引き継ぎ用）】
（エージェントが前章の出力ファイルの末尾から自動読み込み。第1章の場合は 実行結果/Step3_導入章.md の末尾）

---

【執筆の指示】

■ この章の情報
- 章番号: 第◯章
- 対応する悩み: （Step2の該当章設定から）
- テンプレパターン: （Step2の該当章設定から）
- この章のゴール: （Step2の該当章設定から）

■ 章の構成ルール
- 前章の末尾から自然に繋がる冒頭にする
- アウトラインの各H2/H3に忠実に執筆
- 図解プレースホルダーを指定箇所に配置
- 章末に「この章のまとめ」+「次章への橋渡し」を書く

■ 文体ルール
（他のプロンプトと同じ）

■ 付属物への言及
この章に対応する付属物がある場合、本文中で自然に言及:
- 「この手順をテンプレにまとめたので、付属の◯◯を使ってみてください」
- 「自分の状況を整理したい方は、付属の◯◯ワークシートが役立ちます」
※ 付属物の実物はStep3.5で制作するため、ここでは言及のみ

■ 出力
- Markdown形式
- この章のみ出力（他の章は含めない）
```

## 【実行プロンプト】Premium — 統合章・ロードマップ章

```
（各章プロンプトと同じ構成。以下を追加:）

■ 統合章の場合

## 結合手順

### Lite
無料部分 → 有料ライン → 有料部分 → CTA

### Standard
無料部分 → 有料ライン → 有料前半 → 有料後半（付属テンプレセクション含む）→ CTA

### Premium
導入章 → 有料ライン → 第1章 → 第2章 → ... → 統合章 → ロードマップ章 → 付属物使い方ガイド章 → CTA

## セルフチェック

### 全グレード共通
- [ ] キャラの口調が全体で一貫しているか
- [ ] 禁止表現が含まれていないか
- [ ] 図解プレースホルダーが指定箇所に配置されているか
- [ ] オリジナル素材が本文に反映されているか

### Standard 追加
- [ ] 深掘りH3（応用パターン/FAQ）が書かれているか
- [ ] 付属テンプレートセクションに使い方の説明があるか

### Premium 追加
- [ ] 各章の冒頭が前章の末尾と自然に繋がっているか
- [ ] 各章末に「まとめ+橋渡し」があるか
- [ ] 付属物への言及が本文に自然に含まれているか
- [ ] 統合章で各章の解決策が横断的に整理されているか
- [ ] ロードマップ章に具体的なタイムラインがあるか
