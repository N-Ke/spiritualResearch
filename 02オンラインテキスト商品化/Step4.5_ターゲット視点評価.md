## Phase 2: 隙の設計チェック

ここからはAIの分析視点に戻ります。

### 2-1. 「出し切り」チェック

この商品が扱う悩みに対して、解決策を出し惜しみしていないかを検証。

| チェック項目 | 判定 |
|------------|------|
| 悩みに対する解決策が具体的で実行可能か | OK / 不足 |
| 「続きは次の商品で」的な中断がないか | OK / 違反 |
| 価格に見合う情報量と具体性があるか | OK / 不足 |
| 読者がこの商品だけで「次の一歩」を踏み出せるか | OK / 不足 |

**判定:** 出し切れている / 出し惜しみ箇所あり

出し惜しみ箇所がある場合:
- 箇所:
- なぜ出し惜しみと判断したか:
- 追加すべき内容:

### 2-2. 「隙」の設計チェック

「隙」＝次の商品への自然な導線。出し惜しみではなく、成長ステップとしての余白。

**原則: 縦（深さ）は出し切る。横（別の悩み）は触れない。**

| チェック項目 | 判定 |
|------------|------|
| この商品が扱う悩みの「縦」は十分に深いか | OK / 浅い |
| 別の悩み（横）に踏み込みすぎていないか | OK / 踏み込みすぎ |
| 「次にこの壁が来る」という自然な成長ステップが見えるか | OK / 見えない |
| 「もっと深くやりたい人は」の選択肢が自然に示唆されているか | OK / 不自然 / なし |

**隙の状態:**
- 健全な隙がある（次の商品を買う動機が自然に生まれる）
- 隙がなさすぎる（この1商品で完結しすぎて次の購買が発生しにくい）
- 隙が悪質（出し惜しみで不満が残る）

**隙が悪質な場合の修正案:**
（どこに情報を追加して「出し切り」状態にするかの具体的提案）
