# キャラクター連携設計

> フェイスレス動画自動生成システムで設計したキャラクターを、台本生成に連動させるための設計書。
> キャラクター設計書の情報をどのように台本に反映するかを定義します。

---

## 連携の全体像

```
フェイスレス動画自動生成システム              フェイスレスストーリー台本システム
━━━━━━━━━━━━━━━━━━━━━━━              ━━━━━━━━━━━━━━━━━━━━━━━━━
                                             
SKILL_ターゲットリサーチ.md                   
  → 悩みデータベース ─────────────→ {{TARGET_PAIN_POINTS}} に使用
                                             
SKILL_コンセプト設計.md                       
  → コンセプト文 ─────────────────→ ナレーターの立ち位置を決定
                                             
SKILL.md（キャラクター設計）                    
  → キャラクター設計書 ───────────→ {{CHARACTER_PROFILE}} に使用
  → 判断構造 ─────────────────────→ ナレーターの語り口を決定
  → 口癖・表現リスト ─────────────→ 台本の言い回しに反映
```

---

## キャラクター設計書から台本に反映する要素

### 1. 基本プロフィール → ナレーターの自己紹介

キャラクター設計書の以下を台本のナレーターに反映する：

| 設計書の項目 | 台本での反映先 |
|------------|-------------|
| 名前 | ナレーターの名前（冒頭の自己紹介） |
| 経歴ストーリー | ナレーターの立ち位置（専門家 or 当事者 or 仲間） |
| 専門領域 | カバーするお悩みジャンル |
| ターゲット層 | 主人公やサブキャラの年齢・属性設計 |

### 2. 性格パラメータ → ナレーションのトーン

| パラメータ | 台本への影響 |
|-----------|-----------|
| MBTI | 情報の処理順序・提示の仕方 |
| エニアグラム | 根本動機・ストレス時の反応・共感のしかた |
| SF Top5 | 強調するポイント・分析の切り口 |
| コミュニケーションスタイル | 断定度・親近感・専門性のバランス |

**反映方法:**

```
【MBTI別の情報提示順序の例】

ISFJ（杉山誠タイプ）:
  ①自分の体験 → ②相手の気持ち → ③数字で確認 → ④対処法
  
INTJ（分析型キャラ）:
  ①構造的な原因分析 → ②データ提示 → ③戦略的解決策 → ④実行計画

ENFP（共感型キャラ）:
  ①感情的な共感 → ②可能性の提示 → ③多角的な視点 → ④希望の訴求

ESTJ（指導型キャラ）:
  ①問題の明確化 → ②ルールの提示 → ③成功事例 → ④行動指示
```

### 3. トーン設定 → 語り口の調整

キャラクター設計書のトーンスコアを台本のナレーションに反映する：

| トーン項目 | スコア例 | 台本での表現例 |
|-----------|---------|-------------|
| 断定度 3/10 | 低い | 「〜かもしれませんね」「〜という考え方もあります」 |
| 断定度 8/10 | 高い | 「これは間違いありません」「断言します」 |
| 親近感 9/10 | 高い | 「私も同じでした」「一緒に考えましょう」 |
| 親近感 3/10 | 低い | 「客観的に見ると」「データが示しています」 |
| 専門性 4/10 | 低い | 「ざっくり言うと」「難しい話は省きますね」 |
| 専門性 9/10 | 高い | 「学術的に言えば」「研究によると」 |
| 感情度 6/10 | 中程度 | 適度に感情を見せつつ冷静さも保つ |
| 挑発度 1/10 | 低い | 対立や煽りは一切しない |
| 挑発度 7/10 | 高い | 「ぶっちゃけ言いますけど」「みんな分かってるくせに」 |

### 4. 口癖・表現リスト → 台本の言い回し

**「よく使う表現」リストから台本に組み込む:**
- ナレーション内に自然に散りばめる
- 1分あたり1〜2回の頻度が目安
- 特にフック、クライマックス、結びで使用する

**「絶対に使わない表現」リストでNGチェック:**
- 台本完成後に全文検索でNGワードがないか確認
- AIに台本を生成させる際、プロンプトにNG表現リストを含める

### 5. 判断構造 → ストーリーへの独自視点の注入

キャラクターの判断構造は、ナレーターが物語にコメントを入れる際の「フィルター」になる：

```
【判断構造の台本への反映例】

判断構造: 「新しい情報を見たとき → まず自分の経験と照らし合わせる」

台本への反映:
ナレーション「この話を聞いた時、私は自分の経験を思い出しました。
あの時の私も、まったく同じ間違いをしていたんです」
```

## ストーリー型ごとのキャラクターの位置づけ

| 型 | ナレーターの推奨ポジション |
|----|----------------------|
| type_A 転落と気づき | ナレーター（物語の外から語る） |
| type_B ハイブリッドN選 | ナレーター（物語の外から語る） |
| type_C 比較群像劇 | キャラクターの一人 or ナレーター |
| type_D 対比構造 | ナレーター（失敗者視点で語ることも可） |
| type_E メンター型 | メンター役 or ナレーター |
| type_F スカッと逆転 | ナレーター（物語の外から語る） |
| type_G 再起・成長型 | ナレーター or 主人公と同じ体験者 |
| type_H 選択の岐路型 | ナレーター（選択の重さを伝える語り手） |
| type_I 連鎖崩壊型 | ナレーター（警告者として語る） |
| type_J 救いの手型 | ナレーター（温かく見守る語り手） |

## 台本プロンプトへの組み込み方法

### キャラクター情報の貼り付け

台本生成プロンプト（04_台本生成プロンプト.md）の `{{CHARACTER_PROFILE}}` に、以下の情報を貼り付ける：

```
【キャラクター情報（以下をコピー＆ペースト）】

■ 名前: [キャラクター名]
■ 年齢: [年齢]
■ 経歴: [簡潔な経歴]
■ ポジション: [専門家 / 当事者 / 仲間 / 先輩 / 挑戦者]
■ 一人称: [私 / 僕 / 俺 / etc.]
■ 視聴者の呼び方: [皆さん / みんな / あなた / etc.]

■ トーン:
  - 断定度: [1〜10]
  - 親近感: [1〜10]
  - 専門性: [1〜10]
  - 感情度: [1〜10]
  - 挑発度: [1〜10]

■ よく使う表現:
  - 「[表現1]」
  - 「[表現2]」
  - 「[表現3]」
  - （以下すべて列挙）

■ 絶対に使わない表現:
  - 「[NG表現1]」
  - 「[NG表現2]」
  - （以下すべて列挙）

■ 情報提示の順序:
  ①[最初に何をするか]
  ②[次に何をするか]
  ③[その次に何をするか]
  ④[最後に何をするか]

■ 口癖・思考の癖:
  - [癖1]
  - [癖2]
  - [癖3]

■ 基本構造（台本の黄金ルール）:
  [例: 「厳しい現実 + でも大丈夫」]
```

### フェイスレスシステムからの自動転記

フェイスレス動画自動生成システムの出力「キャラクター完全設計書」から、上記の各項目を転記する対応表：

| 設計書のセクション | 台本プロンプトの変数 |
|-----------------|-------------------|
| キャラクター名 | ■ 名前 |
| 基本パラメータ → MBTI | トーン、情報提示の順序に反映 |
| 基本パラメータ → エニアグラム | 基本構造に反映 |
| 基本パラメータ → SF Top5 | 口癖・思考の癖に反映 |
| 媒体別表現 → YouTube ロング動画 | よく使う表現、トーンに反映 |
| 判断構造 | 情報提示の順序に反映 |

## キャラクター未設定の場合のデフォルト

フェイスレスシステムでキャラクターを設計していない場合は、以下のデフォルト設定を使用する：

```
【デフォルトナレーター】

■ 名前: （テーマに合わせてAIが自動設定）
■ ポジション: 当事者（同じ悩みを経験した仲間）
■ 一人称: 「私」
■ 視聴者の呼び方: 「皆さん」

■ トーン:
  - 断定度: 3/10（押しつけず提案する）
  - 親近感: 8/10（仲間として語る）
  - 専門性: 4/10（噛み砕いて分かりやすく）
  - 感情度: 6/10（適度に感情を見せる）
  - 挑発度: 1/10（対立や煽りはしない）

■ よく使う表現:
  - 「正直に言いますね」
  - 「私も同じでした」
  - 「ここ、大事なところなのでゆっくりいきますね」
  - 「一緒に見ていきましょう」
  - 「大丈夫ですよ、まだ間に合います」
  - 「怖がらなくていいんです。でも、知っておいてほしいんです」

■ 絶対に使わない表現:
  - 「○○しないとヤバいです！」（煽り）
  - 「誰でも簡単に○○」（詐欺的）
  - 「プロが教える」（ポジション矛盾）
  - 「情弱」「勉強不足」（見下し）
  - 「今すぐやらないと手遅れ」（恐怖訴求）
  - 「絶対に○○です」（過度な断定）

■ 基本構造:
  「厳しい現実 + でも大丈夫」
```

## 台本の基本フレーム（キャラクター連携版）

```
【フック】（10〜15秒）
→ 視聴者の悩みに共感 or 結末先見せ
→ キャラクターの口調で開始

【自己紹介】（5秒）
→ 「こんにちは、[キャラクター名]です」

【導入】（30〜60秒）
→ テーマ提示 + キャラクターの体験に紐づけ
→ 「実は私も〜」「今日はこの話をしたいんです」

【本編】
→ ストーリー型に応じた構成
→ 各シーンでキャラクターのフィルターを適用
→ キャラクター固有の口癖で緩急

【まとめ】（60〜90秒）
→ ポイントを3〜5個に整理
→ 「今日お伝えしたかったのは〜」

【クロージング】（30秒）
→ 「コメントで教えてくださいね」
→ 「チャンネル登録が励みになります」
→ 次回予告
```
