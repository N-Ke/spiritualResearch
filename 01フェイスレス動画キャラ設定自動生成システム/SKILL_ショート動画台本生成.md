# ショート動画台本生成スキル

## このスキルの目的

キャラクター設計書を元に、**TikTok / Instagram リール / Facebook リール / YouTube ショート** 向けのフェイスレス動画台本を生成する。

4媒体共通のベース台本を生成した後、媒体ごとの微調整（尺・CTA・ハッシュタグ・トーン）を自動適用する。

---

## 前提条件

### 必須ファイル

| ファイル | 用途 | 必須度 |
|---------|------|--------|
| キャラクター設計書（完全版） | ナレーショントーン・判断構造の参照 | 必須 |
| `04_SNS出力フォーマット.md` | 各媒体の特性 | 必須 |
| `記事生成テンプレート.md` | 台本テンプレート | 参照 |
| ターゲット悩みデータベース | 悩みの参照 | 推奨 |

### 呼び出し元

- `SKILL_コンテンツ生成.md`（Step 5からの委譲）
- ユーザーの直接指示

---

## 起動条件

ユーザーが以下のいずれかの意図を示したとき：
- 「TikTok台本を作って」
- 「リール台本を作って」
- 「YouTubeショート台本を作って」
- 「ショート動画の台本が欲しい」
- 特定の媒体を指定してショート動画台本を求めた場合

---

## 実行フロー

```
Step 1: 入力の確認（テーマ・媒体・核心メッセージ）
    ↓
Step 2: フック構築（最初の1〜3秒）
    ↓
Step 3: ナレーション原稿の生成
    ↓
Step 4: テキストオーバーレイ指示の生成
    ↓
Step 5: 映像素材・BGM指示の生成
    ↓
Step 6: 媒体別カスタマイズ
    ↓
Step 7: 品質チェック → ユーザー確認
```

---

## Step 1: 入力の確認

### SKILL_コンテンツ生成.mdからの委譲の場合

以下が既に決まっている：
- テーマ
- 核心メッセージ
- 解決策の構造
- ターゲットの悩み

→ そのまま Step 2 に進む。

### ユーザーの直接指示の場合

```
ショート動画の台本を生成します。

【必須】
1. テーマ：何についての動画ですか？
2. 媒体（複数選択可）：
   □ TikTok
   □ Instagram リール
   □ Facebook リール
   □ YouTube ショート
   □ 全部

【任意】
3. 伝えたい核心メッセージ（あれば）
4. 推奨尺（あれば）
   → 未指定の場合は媒体最適尺を自動設定
```

## Step 2: フック構築（最初の1〜3秒）

**フェイスレス動画で最も重要な要素。冒頭で勝負が決まる。**

### フックの5パターン

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
フック設計
━━━━━━━━━━━━━━━━━━━━━━━━━━

パターン1：衝撃型（Te/タイプ8向け）
テロップ：「[衝撃の事実/数字]」（大きいフォント、画面中央）
ナレーション：「○○がやってるのは全部間違い」
映像：[衝撃を視覚化する素材]

パターン2：共感型（Fe/タイプ2向け）
テロップ：「[ターゲットの悩みワード]」
ナレーション：「○○で悩んでる人、これだけ見て」
映像：[悩みを感じさせるシーン]

パターン3：好奇心型（Ni/タイプ5向け）
テロップ：「知らないとヤバい」
ナレーション：「○○の本当の意味、知ってる？」
映像：[ミステリアスな素材]

パターン4：結論先出し型（Ti/タイプ1向け）
テロップ：「答え：○○」
ナレーション：「結論から言うと、○○です」
映像：[答えを視覚化する素材]

パターン5：Before-After型（Se/タイプ3向け）
テロップ：「Before → After」
ナレーション：「○○だったのが、○○になった方法」
映像：[ビフォーアフターの視覚的対比]

→ キャラクターのMBTI主機能 × エニアグラムから最適なパターンを自動選定
━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Step 3: ナレーション原稿の生成

### ショート動画の基本構造

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
ナレーション原稿
━━━━━━━━━━━━━━━━━━━━━━━━━━

[0〜3秒] フック
「[衝撃/共感/好奇心の一文]」
→ 音声 + テロップ同時表示
→ 効果音：[インパクト効果音]

[3〜10秒] 問題提起
「[ターゲットの悩みを代弁]」
「[なぜそうなるのか、一言で核心]」
→ テンポ：速め
→ 映像：[悩みの視覚化]

[10〜35秒] 解決策（メイン部分）
「[解決策1の要点]」
「[解決策2の要点]」
「[解決策3の要点]」
→ テンポ：やや速め〜普通
→ 映像：[各ステップを視覚化]
→ テロップ：[キーワードを大きく表示]

[35〜50秒] 結論
「[核心メッセージの再確認]」
「[今日からできる具体アクション]」
→ テンポ：普通
→ 映像：[ポジティブな結末の素材]

[50〜60秒] CTA
「[媒体別のCTA文]」
→ テロップ：[フォロー/保存/チャンネル登録 等]

━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### ナレーションの品質ルール

```
【AI音声で自然に聞こえるためのルール】
1. 一文は短く（20文字以内を推奨）
2. 句読点で間を取る（「。」=0.5秒、「、」=0.3秒）
3. 体言止めを効果的に使う
4. 「えっと」「まあ」等のフィラーは入れない
5. 専門用語は直後に簡単な説明を入れる
6. 強調したい単語は【】で囲んで指示する
   例：「これが【最大の間違い】なんです」

【キャラクターの声として聞こえるためのルール】
1. キャラクター設計書の「表現の癖」を必ず反映
2. 語尾のパターンを統一する
3. 判断構造に矛盾する表現を使わない
4. 「誰が読んでも同じ」にならないよう個性を出す
```

[35〜50秒]
まとめテロップ：「[核心メッセージ]」 → 大きなフォント
ナレーション字幕：表示する

[50〜60秒]

## Step 6: 媒体別カスタマイズ

### 共通のベース台本を生成した後、以下を媒体ごとに調整する：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
媒体別カスタマイズ
━━━━━━━━━━━━━━━━━━━━━━━━━━

■ TikTok カスタマイズ
  尺調整：30〜60秒に最適化
  CTA：「フォローして次のPart見て」
  トレンド音源：使用推奨（あれば指定）
  ハッシュタグ：3〜5個（トレンド + ニッチ）
  追加要素：ループ構造（最後→最初がつながるよう調整）

■ Instagram リール カスタマイズ
  尺調整：15〜30秒に最適化（Explore表示有利）
  CTA：「保存して後で実践してね」
  ハッシュタグ：5〜15個（関連 + ニッチ）
  キャプション：300〜500文字の解説テキスト（SEO効果）
  追加要素：フィードの世界観に合うビジュアルスタイル指示

■ Facebook リール カスタマイズ
  尺調整：30〜60秒に最適化
  CTA：「シェアして周りの人にも教えてあげて」
  テキストサイズ：大きめ（30代以上ユーザー対応）
  追加要素：共感→シェアの流れを強化する構成

■ YouTube ショート カスタマイズ
  尺調整：30〜50秒に最適化
  CTA：「チャンネル登録して、詳しくはメイン動画で」
  追加要素：ロング動画への導線（「続きは概要欄から」等）
  サムネイル不要（自動生成）

━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Step 7: 品質チェック

### チェックリスト

```
【フック品質】
- [ ] 最初の1秒で視覚的なインパクトがあるか
- [ ] テロップとナレーションが同時に機能するか
- [ ] 音なしでも内容が伝わるか

【ナレーション品質】
- [ ] キャラクターの「声」として自然か
- [ ] AI音声で読み上げたとき不自然にならないか
- [ ] 一文が短く、テンポが良いか

【構成品質】
- [ ] 核心メッセージが30秒以内に伝わるか
- [ ] 無駄な部分がないか（全カットに意味があるか）
- [ ] CTAが自然に挿入されているか

【フェイスレス品質】
- [ ] 映像素材の指示が具体的で調達可能か
- [ ] テロップが読みやすいサイズ・配置か
- [ ] BGMのムードが内容と合っているか

【キャラクター一貫性】
- [ ] 判断構造に矛盾する表現がないか
- [ ] 他の媒体の台本と核心メッセージが一致するか
- [ ] キャラクターの口癖・表現パターンが反映されているか
```

## ショートカットコマンド

| コマンド | 動作 |
|---------|------|
| 「TikTok台本を作って」 | TikTok指定で実行 |
| 「リール台本を作って」 | IG Reels + FB Reels指定で実行 |
| 「YouTubeショート台本を作って」 | YT Shorts指定で実行 |
| 「ショート動画全部作って」 | 4媒体一括実行 |
| 「フックを変えて」 | フック部分のみ再生成 |
| 「もっと短くして」 | 尺を短縮（要素の削除・圧縮） |
| 「もっと長くして」 | 尺を延長（要素の追加） |
| 「別の切り口で」 | フック + 導入部分を変更 |
