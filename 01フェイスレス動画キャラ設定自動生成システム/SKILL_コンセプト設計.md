# コンセプト設計スキル

## このスキルの目的

ターゲットリサーチで得た悩みクラスターを元に、**「自分はどの悩みを、どう解決する人か」**を1つのコンセプトとして定義する。

コンセプトが決まらないと、キャラクター設計がブレる。
キャラクター設計がブレると、コンテンツがブレる。
だから、**ターゲットリサーチの後、キャラクター設計の前**にこのスキルを実行する。

---

## このスキルの位置づけ

```
ターゲットリサーチ（SKILL_ターゲットリサーチ.md）
  → 悩みクラスター12個 + Top20 + ペルソナ
      ↓
コンセプト設計（このスキル）    ← 今ここ
  → ポジショニング + コンセプト文 + 差別化軸
      ↓
キャラクター設計（SKILL.md）
  → コンセプトに最適なキャラクターを自動設計
```

---

## 前提条件

| ファイル | 用途 | 必須度 |
|---------|------|--------|
| ターゲットリサーチ結果（悩みクラスター） | コンセプトの材料 | 必須 |
| ターゲットペルソナ | 誰に届けるかの確認 | 必須 |
| キャラクター設計書 | 既にあれば参照（なくてもOK） | 任意 |

---

## 起動条件

ユーザーが以下のいずれかの意図を示したとき：
- 「コンセプトを設計して」
- 「ポジショニングを決めたい」
- 「どんな発信者になるか決めたい」
- 「差別化を考えたい」
- ターゲットリサーチ完了後に「次は？」と聞かれた場合

---

## 実行フロー（全Step必須）

```
Step 1: ターゲットリサーチ結果の読み込み
    |
Step 2: 解決する悩みの選定（どの悩みクラスターを軸にするか）
    |
Step 3: ポジショニングの設計（どの立場から解決するか）
    |
Step 4: コンセプト文の作成（一言で何の専門家か）
    |
Step 5: 差別化軸の明確化（他の発信者と何が違うか）
    |
Step 6: コンセプトシートの出力 + ユーザー確認
```

---

## Step 1: ターゲットリサーチ結果の読み込み

### 読み込むもの

ターゲットリサーチの出力から以下を抽出する：

```
【内部メモ】
- ジャンル: [ジャンル]
- ターゲットペルソナ: [要約]
- 悩みクラスター数: 12個
- 特に刺さる悩み Top20: [一覧]
- ターゲットの核心的悩み Top3: [ペルソナから]
```

---

## Step 2: 解決する悩みの選定

### このStepの目的

12個の悩みクラスターの中から、**自分が解決する悩みを1〜3個に絞る**。
全部の悩みを解決しようとすると、何でも屋になってコンセプトがボヤける。

### ユーザーに聞くこと

```
ターゲットの悩みクラスターは12個あります。

[12クラスターの一覧を提示]

この中で、あなたが「これなら自信を持って解決できる」と思うものはどれですか？

選び方のコツ：
- 1〜3個に絞る（多くても3個まで）
- 自分の経験・スキル・実績と一致するもの
- 「これについて語り続けても飽きない」もの
- 迷ったら「自分が過去に苦しんだ悩み」を選ぶと強い

分からない場合は「お任せ」で、AIが最適な組み合わせを提案します。
```

### AIが提案する場合の選定基準

```
以下の3軸で評価し、最適な1〜3クラスターを選出：

1. 市場性（その悩みを抱えている人が多いか）
   → Top20に多く含まれるクラスターが高い

2. 差別化しやすさ（他の発信者がカバーしていない切り口があるか）
   → ありがちなアドバイスで終わらないクラスターが高い

3. 深さ（表層だけでなく中層・深層まで掘れるか）
   → クラスター内に多層の悩みがあるほど高い
```

---

## Step 3: ポジショニングの設計

### このStepの目的

選んだ悩みクラスターに対して、**どの立場から解決するか**を決める。
同じ悩みでも、立場が違えばコンセプトが全く変わる。

### ポジショニングの5パターン

```
以下の5パターンから、最も合うものを選ぶ：

パターン1: 先輩型（経験者ポジション）
  「私も同じ悩みを経験した。こうやって解決した」
  → 説得力：実体験  
  → 向いている人：自分が乗り越えた経験がある

パターン2: 専門家型（プロフェッショナルポジション）
  「この分野のプロとして、正しい方法を教える」
  → 説得力：知識・資格・実績  
  → 向いている人：専門知識や資格がある

パターン3: 伴走型（コーチポジション）
  「一緒に考えながら、あなたに合った方法を見つける」
  → 説得力：寄り添い・個別対応  
  → 向いている人：人の話を聞くのが得意

パターン4: キュレーター型（情報整理ポジション）
  「大量の情報を整理して、本当に必要なものだけ届ける」
  → 説得力：リサーチ力・要約力  
  → 向いている人：情報収集・分析が得意

パターン5: 挑戦者型（リアルタイム共有ポジション）
  「今まさに同じ挑戦をしている。リアルな過程を共有する」
  → 説得力：共感・ライブ感  
  → 向いている人：初心者でも始めやすい
```

### ユーザーに聞くこと

```
どのポジションが自分に合いそうですか？

1. 先輩型（経験で語る）
2. 専門家型（知識で語る）
3. 伴走型（一緒に考える）
4. キュレーター型（情報を整理する）
5. 挑戦者型（リアルタイムで共有する）

分からない場合は「お任せ」で、AIがあなたの状況から最適なものを提案します。
```

## Step 4: コンセプト文の作成

### このStepの目的

Step 2（どの悩み）+ Step 3（どの立場）を合体させて、**一言で「何の専門家か」を表現する**。

### コンセプト文のフォーマット

```
基本形：
「[ターゲット]の[悩み]を、[ポジション]として[方法/価値]で解決する人」

=== 具体例 ===

NG: 広すぎる
  「子育てについて発信します」
  「副業を教えます」
  「恋愛の相談に乗ります」

OK: 具体的で一発で分かる
  「3歳児の夜泣きに悩む共働き夫婦の、"罪悪感なし"の子育て術」
  「会社員のまま月5万を安定させる、週末2時間の副業設計」
  「出会いはあるのに2回目に繋がらない人の、距離感リセット法」
```

### AIの生成プロセス

```
1. ターゲット: [ペルソナの要約]
2. 解決する悩み: [選んだクラスターの代表的な本人語]
3. ポジション: [選んだパターン]
4. 方法/価値: [ポジションから導出される独自の価値]

→ これらを組み合わせて、コンセプト文を3案生成する

コンセプト文案1: [案1]
コンセプト文案2: [案2]
コンセプト文案3: [案3]

→ ユーザーに選んでもらう or 修正指示をもらう
```

---

## Step 5: 差別化軸の明確化

### このStepの目的

同じジャンルの他の発信者と**何が違うか**を明確にする。
コンセプトだけでは差別化が弱い場合、ここで補強する。

### 差別化の7軸

```
以下の軸から、自分の強みに合うものを1〜3個選ぶ：

1. 経験の差別化
   「実際に[具体的な経験]を乗り越えた」
   例：「自分自身が3年間の不妊治療を経験した」

2. 方法の差別化
   「一般的な方法とは違う[独自の方法]を使う」
   例：「心理学ベースの会話術で、テクニックに頼らない恋愛」

3. 切り口の差別化
   「同じテーマでも[独自の角度]から見る」
   例：「子育てを"子どもの発達段階"ではなく"親の心理"から語る」

4. ターゲットの差別化
   「他の人がカバーしていない[ニッチなターゲット]に特化」
   例：「シングルファザー専門の子育て発信」

5. 表現の差別化
   「[独自のトーン・スタイル]で発信する」
   例：「毒舌だけど的確、笑えるけど本質をつく」

6. プロセスの差別化
   「[独自のフレームワーク・ステップ]を持っている」
   例：「3ステップの"距離感リセット法"」

7. 実績の差別化
   「[具体的な数字・成果]がある」
   例：「相談者の80%が3ヶ月以内に交際開始」
```

### ユーザーに聞くこと

```
他の発信者との違いを明確にしたいです。
上の7軸の中で、自分に当てはまるものはありますか？

なくても大丈夫です。
「お任せ」なら、あなたのジャンルとポジションから最適な差別化軸をAIが提案します。
```


## ショートカットコマンド

| コマンド | 動作 |
|---------|------|
| 「コンセプトを設計して」 | Step 1から順番に実行 |
| 「ポジションを変えて」 | Step 3を再実行 |
| 「コンセプト文を変えて」 | Step 4を再実行 |
| 「差別化を強化して」 | Step 5を再実行 |
| 「コンセプトシートを見せて」 | 現在の設計シートを表示 |
| 「キャラクター設計に進む」 | 設計シートを確定し、SKILL.mdに引き渡し |

## エラーハンドリング

### ターゲットリサーチが未実行の場合

```
コンセプト設計にはターゲットリサーチの結果が必要です。
先に「ターゲットをリサーチして」と指示してください。

ターゲットリサーチが完了したら、自動的にコンセプト設計に進めます。
```

### 悩みクラスターが選べない場合

```
迷っている場合は、以下の質問に答えてみてください：

1. 過去に自分が一番苦しんだのは、どのクラスターの悩みに近いですか？
2. 友人に相談されたとき、一番自信を持って答えられるのはどれですか？
3. 1年間毎日話しても飽きないのはどのテーマですか？

どれも分からなければ「お任せ」で、AIが市場性 × 差別化 × 深さから最適な組み合わせを選びます。
```

### コンセプトが「広すぎる」場合

```
コンセプトが広すぎる可能性があります。
チェック：「このコンセプトを聞いて、具体的な1人の人物が頭に浮かぶか？」

浮かばない場合は：
- ターゲットをもっと絞る
- 解決する悩みを1つに減らす
- 「〇〇専門」「〇〇限定」のように条件を追加する
```
