# フェイスレス動画台本一括生成スキル

## このスキルの目的

作成済みのキャラクター設計書を元に、ユーザーが入力した情報を「そのキャラクターのフィルター」で咀嚼し、複数の動画媒体向けの台本を一括生成する。

フェイスレス動画（顔出しなし・AI音声ナレーション＋映像素材）に最適化された台本を出力する。

---

## 前提条件

### 必須ファイル

| ファイル | 用途 | 必須度 |
|---------|------|--------|
| キャラクター設計書（完全版） | 判断構造・表現ルールの参照 | 必須 |
| `ライティングナレッジ統合参照ガイド.md` | **固有ナレッジの参照ルール（最優先）** | **必須** |
| ターゲット悩みデータベース | ターゲットの悩み・不安の参照 | 推奨 |
| `ターゲット設計テンプレート.md` | データベース未作成時のフォーマット | 参照 |
| `記事生成テンプレート.md` | 動画台本の構成テンプレート（汎用・補助用） | 参照 |
| `04_SNS出力フォーマット.md` | 動画媒体の基本特性 | 参照 |

### 関連スキル

| スキル | 役割 | 連携 |
|--------|------|------|
| `SKILL_ターゲットリサーチ.md` | ターゲット悩みデータベースの生成 | 本スキルのStep 2で参照 |
| `SKILL_ネタ出し.md` | コンテンツテーマの選定・計画 | 本スキルのStep 1に入力を提供 |
| `SKILL_ショート動画台本生成.md` | **TikTok/IG Reels/FB Reels/YT Shorts台本の専用生成** | ショート動画台本はこのスキルに委譲 |
| `SKILL_YouTube長尺台本生成.md` | **YouTube ロング動画台本の専用生成** | YouTubeロング台本はこのスキルに委譲 |

ネタ出しスキルから引き渡された場合、Step 1のユーザー入力は不要（テーマ・切り口・悩みが既に決定済み）。

### 台本生成の分担（重要）

```
ショート動画台本（TikTok/IG Reels/FB Reels/YT Shorts）
  → SKILL_ショート動画台本生成.md に委譲

YouTube ロング動画台本
  → SKILL_YouTube長尺台本生成.md に委譲

本スキルの役割：
  → 情報の咀嚼（Step 0〜4）を一元的に実行
  → 核心メッセージ・解決策を各専用スキルに引き渡す
  → 全媒体の一貫性を管理する
```

### 重要：ナレッジの優先順位

```
1. 固有ナレッジ（ライティングナレッジ統合参照ガイド経由） ← 最優先
2. キャラクターの判断構造（設計書）
3. 汎用テンプレート（記事生成テンプレート.md） ← 固有ナレッジがない媒体のみ使用
```

### キャラクター設計書が未作成の場合

「SKILL.md（フェイスレス動画自動生成システム）」を先に実行し、キャラクターを完成させてから本スキルを使用すること。

## 起動条件

ユーザーが以下のいずれかの意図を示したとき：
- 「この情報で動画台本を作って」
- 「動画コンテンツを生成して」
- 「台本を一括で作りたい」
- テーマや情報素材を提示して、台本の生成を求めた場合

---

## 実行フロー全体像

```
Step 0: キャラクター設計書の読み込み
    ↓
Step 1: 入力の受け取り
    ↓
Step 2: ターゲットの悩み・不安の特定
    ↓
Step 3: キャラクターのフィルターで情報を咀嚼
    ↓
Step 4: 解決策の構造化
    ↓
Step 5: 媒体別台本の一括生成（専用スキルに委譲）
    ↓
Step 6: ユーザー確認・修正
    ↓
Step 7: 最終出力 + 成長データの記録
```

---

## Step 0: キャラクター設計書の読み込み

### 読み込む情報

キャラクター設計書から以下を抽出する：

1. **判断構造（情報フィルター10項目）** → 情報の咀嚼に使用
2. **ビジネス判断構造** → 解決策の方向性に使用
3. **媒体別表現設計** → 各媒体の出力スタイルに使用
4. **表現の癖** → ナレーションの口調再現に使用
5. **ビジネスポジショニング** → ターゲットとの関係性に使用

### キャラクターの「声」の確認

読み込み後、内部的にキャラクターの「声」を確認する：

```
【内部チェック】
このキャラクターは：
- 情報をまず[主機能]で処理する
- 根本的に[エニアグラムの動機]で動いている
- [SF Top資質]が行動に現れる
- [トーン設定]でナレーションする
→ OK、このキャラクターの声で台本を生成する
```

## Step 1: 入力の受け取り

### ユーザーに聞くこと

```
動画台本を生成します。以下を教えてください。

【必須】
1. テーマ・情報素材
   何について動画を作りますか？ URL、テキスト、ニュース、自分の考えなど、
   どんな形式でも構いません。

2. 出力する媒体（複数選択可）
   □ TikTok
   □ Instagram リール
   □ Facebook リール
   □ YouTube ショート
   □ YouTube ロング動画
   □ ショート動画全部（TikTok/IG/FB/YT Shorts）
   □ 全部

【任意】
3. ターゲットの具体的な悩み（分かっていれば）
   → 未入力の場合は、ターゲット設計テンプレートから自動選定します

4. 動画の方向性（あれば）
   例：「問題提起系」「ハウツー系」「体験談系」「Before-After系」「リスト型」
   → 未入力の場合は、キャラクターのパラメータから最適な方向を自動選定
```

### 入力パターンの処理

**パターンA：テーマのみ**
→ ターゲットの悩みは自動選定、方向性はキャラクターから導出

**パターンB：テーマ + ターゲットの悩み**
→ 悩みをそのまま使用、方向性はキャラクターから導出

**パターンC：テーマ + 悩み + 方向性**
→ すべてユーザー指定通りに進行

**パターンD：URLや記事の貼り付け**
→ まず内容を要約し、テーマを抽出してからパターンA〜Cと同じフローに入る

## Step 2: ターゲットの悩み・不安の特定

### 2-1. ターゲット設計テンプレートが存在する場合

`ターゲット設計テンプレート.md` のデータベースから、入力テーマに最も関連する悩み・不安を自動抽出する。

**抽出ロジック：**
1. テーマのキーワードとデータベースの悩みをマッチング
2. 関連度の高い悩みを上位3〜5個抽出
3. 最も核心的な悩みを1つ「メイン悩み」として選定
4. 残りを「サブ悩み」として保持

### 2-2. ターゲット設計テンプレートが未作成の場合

AIが即座に以下のフレームワークでターゲットの悩みを推定する：

```
【テーマ】[入力されたテーマ]
【キャラクターの専門領域】[設計書から]
【ターゲット像】[設計書から]

このテーマに関して、ターゲットが抱えている可能性が高い悩み：

レベル1：表層の悩み（本人が自覚している）
- [悩み1]
- [悩み2]
- [悩み3]

レベル2：中層の悩み（漠然と感じているが言語化できていない）
- [悩み1]
- [悩み2]

レベル3：深層の悩み（本人も気づいていない根本課題）
- [悩み1]

→ メイン悩み：[レベル2から1つ選定]
  （レベル2が最も動画としてのインパクトが高い）
```

### 2-3. ユーザーが悩みを明示した場合

入力された悩みをメイン悩みとして採用し、AIが追加のサブ悩みを補完する。

## Step 3: キャラクターのフィルターで情報を咀嚼

### 3-1. 情報咀嚼プロセス

入力された情報を、キャラクターの判断構造を通して処理する。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
情報咀嚼レポート（内部処理）
━━━━━━━━━━━━━━━━━━━━━━━━━━

【元の情報】
[ユーザーが入力した情報の要約]

【このキャラクターの第一反応】（MBTI主機能）
「[キャラクターが最初に感じること/考えること]」

【フィルター適用】（エニアグラム × MBTI）
- この情報の本質：[キャラクターの解釈]
- 重要だと感じるポイント：[キャラクターが注目する点]
- 疑問・違和感：[キャラクターが感じる引っかかり]
- 見落とされがちな視点：[キャラクターならではの切り口]

【ターゲットの悩みとの接続】
- メイン悩み「[悩み]」に対して、この情報が持つ意味：
  [キャラクターの解釈]
- ターゲットが陥りがちな誤解：
  [キャラクターの視点]

【発信意図の形成】（SF Top資質）
- 伝えたいこと：[核心メッセージ]
- 切り口：[どの角度から語るか]
- ゴール：[視聴者にどうなってほしいか]

━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 3-2. 核心メッセージの確定

咀嚼レポートから、1つの核心メッセージを抽出する：

```
【核心メッセージ】
[1文で表現できる、この動画の中心的な主張]

【なぜこのメッセージか】
キャラクターの[判断基準N]に基づき、ターゲットの[メイン悩み]に対して、
最も価値のある示唆は[核心メッセージ]であると判断した。
```

この核心メッセージが、全媒体の台本の「軸」になる。

## Step 4: 解決策の構造化

### 4-1. 解決策フレームワーク

キャラクターの判断構造に基づいて、解決策を構造化する：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
解決策構造
━━━━━━━━━━━━━━━━━━━━━━━━━━

【問題の再定義】
ターゲットは「[表層の悩み]」で困っていると思っている。
しかし、本当の問題は「[深層の悩み / 再定義された問題]」である。
→ このキャラクターの[判断基準]による再定義

【解決の方向性】
[キャラクターのビジネスポジショニングから導出される、このキャラクターならではの解決アプローチ]

【具体的な解決策】（3〜5個）

解決策1：[最も重要な解決策]
- なぜこれが有効か：[キャラクターの根拠]
- 具体的にどうするか：[アクションステップ]
- ターゲットへの効果：[期待される変化]

解決策2：[二番目に重要な解決策]
- なぜこれが有効か：[キャラクターの根拠]
- 具体的にどうするか：[アクションステップ]
- ターゲットへの効果：[期待される変化]

解決策3：[三番目の解決策]
...

【このキャラクターが絶対に言わないこと】
- [判断構造と矛盾する解決策]
- [このキャラクターのスタンスに反するアドバイス]

【落とし穴の警告】
- [ターゲットがやりがちな間違い]
- [このキャラクターの経験から来る注意点]

━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 4-2. 解決策のキャラクター化

生成された解決策が「このキャラクターらしい」かを検証する：

- [ ] 解決策がキャラクターのビジネスポジショニングと一致するか
- [ ] 表現がキャラクターのトーンと一致するか
- [ ] 判断構造に矛盾しないか
- [ ] 「誰が言っても同じ」にならず、「このキャラクターだから言える」内容になっているか

## Step 5: 媒体別台本の一括生成

### 5-1. 生成の順序

**重要：生成前に `ライティングナレッジ統合参照ガイド.md` を必ず読み込むこと。**

1. **YouTube ロング動画台本を最初に生成**（最も情報量が多い）
   → `SKILL_YouTube長尺台本生成.md` を実行する
   → 本スキルのStep 3〜4で生成した核心メッセージ・解決策をそのまま渡す
2. **ショート動画台本をロング動画から変換**
   → `SKILL_ショート動画台本生成.md` を実行する
   → ロング動画の最もインパクトのある部分を抽出・圧縮
   → TikTok/IG Reels/FB Reels/YT Shorts の4媒体向けに微調整

### 5-2. 各媒体の生成仕様

#### ショート動画（TikTok/IG Reels/FB Reels/YT Shorts）→ 専用スキルで生成

```
ショート動画台本は SKILL_ショート動画台本生成.md で生成する。

本スキルから呼び出す場合：
- テーマ・核心メッセージ・解決策を専用スキルに引き渡す
- フック構築・ナレーション・テロップ・映像指示は専用スキルが実行
- 媒体ごとの微調整（尺・CTA・ハッシュタグ）も専用スキル内で実行
- 生成結果を本スキルの出力に統合する
```

#### YouTube ロング動画 → 専用スキルで生成

```
YouTube ロング動画台本は SKILL_YouTube長尺台本生成.md で生成する。

本スキルから呼び出す場合：
- テーマ・核心メッセージ・解決策を専用スキルに引き渡す
- チャプター構成・フック・視聴維持率設計は専用スキルが実行
- タイトル・サムネコンセプト案の生成も専用スキル内で実行
- 生成結果を本スキルの出力に統合する
```

### 5-3. 媒体間の一貫性チェック

全媒体の生成が完了したら、以下を確認：

- [ ] 核心メッセージが全媒体で一致しているか
- [ ] キャラクターの「声」が全媒体で統一されているか（テンポは媒体で変わるが、判断は変わらない）
- [ ] 解決策の方向性が矛盾しないか
- [ ] 媒体ごとに表現が最適化されているか（全部同じ台本のコピペになっていないか）
- [ ] ショート動画のフックがロング動画と連動しているか

### 出力フォーマット

```
━━━━━━━━━━━━━━━━━━━━━━━━━━

■ テーマ：[テーマ]
■ 核心メッセージ：[1文]
■ ターゲットの悩み：[メイン悩み]
■ キャラクター：[キャラクター名]

────────────────────────────
TikTok台本（[N]秒）
────────────────────────────
[生成されたTikTok台本]

────────────────────────────
Instagram リール台本（[N]秒）
────────────────────────────
[生成されたIG Reels台本]

────────────────────────────
Facebook リール台本（[N]秒）
────────────────────────────
[生成されたFB Reels台本]

────────────────────────────
YouTube ショート台本（[N]秒）
────────────────────────────
[生成されたYT Shorts台本]

────────────────────────────
YouTube ロング動画台本（[N]分）
────────────────────────────
[生成されたYT Long台本]

━━━━━━━━━━━━━━━━━━━━━━━━━━

修正が必要な場合は指示してください。
- 特定の媒体だけ修正したい → 「TikTokだけ修正して」
- トーンを変えたい → 「もっと攻撃的に」「もっと優しく」
- フックを変えたい → 「フックを変えて」
- 方向性を変えたい → 「ハウツー寄りにして」
- 全体を作り直したい → 「やり直し」
```

### 修正の処理

- 特定媒体の修正 → その媒体のみ再生成（他は維持）
- トーン変更 → キャラクターの表現設計書のスペクトラムを一時調整
- フック変更 → フック部分のみ再構築
- 方向性変更 → Step 4に戻って解決策を再構造化
- 全体やり直し → Step 1に戻る

## Step 7: 最終出力 + 成長データの記録

### 最終出力

ユーザーが承認したら、各媒体の台本をそのまま使える形で最終出力する。

### 成長データの記録（推奨）

生成した台本の情報を、キャラクターの成長システムに記録する：

```
【動画台本生成ログ】
日付：[YYYY-MM-DD]
テーマ：[テーマ]
核心メッセージ：[1文]
ターゲットの悩み：[メイン悩み]
出力媒体：[使用した媒体一覧]
推定尺：[各媒体の秒数/分数]
ユーザー修正：[あり/なし、修正内容]
```

この記録は、後日「反応分析トリガー」（成長・進化システム）で活用される。

## エラーハンドリング

### キャラクター設計書が見つからない場合
```
キャラクター設計書が見つかりませんでした。
先に「フェイスレス動画自動生成システム」でキャラクターを作成してください。
→ 「キャラクターを作って」と指示すれば開始できます。
```

### 入力情報が不足している場合
```
テーマが不明確です。以下のいずれかの形式で入力してください：
- テーマを一言で（例：「AIの活用法」）
- 参考URLの貼り付け
- 自分の考えをそのまま入力
```

### 生成結果がキャラクターらしくない場合
内部的に以下をチェックし、自動修正する：
1. 判断基準との一致度を検証
2. 表現の癖との一致度を検証
3. 乖離が大きい箇所を再生成
