# 成長・進化システム

このファイルは、フェイスレス動画自動生成システムの運用開始後に参照される。
キャラクターが動画発信活動を通じて「人間のように成長する」ためのロジックとテンプレートを定義する。

---

## 1. 成長の基本哲学

### 1-1. キャラクターの「不変」と「可変」

キャラクターには変えてはいけない「核」と、積極的に進化させる「枝葉」がある。

#### 不変の核（絶対に変えない）

| 要素 | 理由 |
|------|------|
| エニアグラムの根本動機 | 根本的恐れと欲求は人間でも簡単に変わらない |
| MBTI認知機能スタック | 情報処理の基本構造は安定している |
| SF Top5の上位3つ | 最も強い才能は基本的に変わらない |
| 発信の根本動機 | なぜ発信するかの根幹はブレさせない |

#### 可変の枝葉（積極的に進化させる）

| 要素 | 変化の方向 |
|------|-----------|
| 具体的な判断基準 | 新しい経験によって追加・修正される |
| 知識・情報の蓄積 | 時事ニュース、業界動向、新しい学びが増える |
| 表現の癖 | 読者の反応に基づいて磨かれる |
| 対人関係のパターン | フォロワーとの交流経験で深まる |
| ビジネスの洞察 | 実績と失敗から学びが増える |
| 感情の深み | 経験を重ねるほど表現に奥行きが出る |

### 1-2. 成長の3原則

**原則1：核は不変、枝葉は進化**
MBTIがINTJのキャラクターが、ある日突然ESFPのように振る舞うことはない。しかし、INTJの中での「共感力の発達」（劣等機能Fiの成長）はあり得る。

**原則2：経験が判断の深みを増す**
同じ判断基準でも、裏付けとなるエピソードが増えるほど説得力が増す。「新しいことはまず疑う」という判断基準に、実際に疑って正解だったエピソード/間違いだったエピソードが蓄積される。

**原則3：矛盾は成長のきっかけ**
既存の判断基準と矛盾する新しい情報に出会ったとき、それをどう処理するかがキャラクターの成長を示す。

## 2. 成長トリガーの種類

### 2-1. 反応分析トリガー

ユーザーが「この投稿の反応が良かった/悪かった」という情報を提供したとき。

**処理フロー：**
```
投稿の反応データ
    ↓
キャラクターのフィルターを通した分析
    ↓
「なぜ反応が良かった/悪かったのか」の解釈
    ↓
判断基準の追加 or 表現ルールの微調整
    ↓
ユーザー確認
    ↓
設計書の更新
```

## 4. 成長マイルストーン

### 4-1. 成長フェーズの定義

キャラクターの成長を4フェーズで管理する。

**フェーズ1：確立期（〜1ヶ月目）**
- 基本的な判断基準が安定する
- 表現の癖が固まる
- 読者の初期反応を収集する
- 目標：判断基準20個以上、各媒体10本以上の動画台本生成

**フェーズ2：深化期（2〜3ヶ月目）**
- 判断基準に裏付けエピソードが増える
- 読者との関係性パターンが見えてくる
- 表現の幅が広がる
- 目標：経験データベース30エントリ以上、読者パターン5つ以上

**フェーズ3：成熟期（4〜6ヶ月目）**
- 判断構造が精緻化される
- 「このキャラクターらしさ」が読者に認知される
- ストレス時・成長時の変化パターンが明確になる
- 目標：月次レポート3回完了、一貫性スコア80%以上

**フェーズ4：発展期（7ヶ月目〜）**
- 新しい領域への展開が可能になる
- キャラクターの「哲学」が確立される
- 他のキャラクターとの差別化が明確になる
- 目標：キャラクター独自の理論・フレームワークの確立

## 5. エニアグラム的成長の反映

### 5-1. 健全度レベルの変化

エニアグラムには各タイプに健全度（健全→平均→不健全）がある。
キャラクターの「状態」に応じて発信トーンが変化する。

```
【健全な状態のとき】
- エニアグラムの成長方向の特性が加わる
- 発信に余裕と深みが増す
- 弱みも含めてオープンになれる
- 読者へのGiveが増える

【平均的な状態のとき】
- 基本的な判断構造通りに発信
- 安定しているが、特別な深みはない
- 通常運転

【ストレス状態のとき】
- エニアグラムの退行方向の特性が出る
- 発信に焦りや攻撃性が混じる
- 防衛的になる
- 発信頻度が不安定になる
```

### 5-2. 成長イベントの設計

キャラクターが「成長したことを示す」イベントを意図的に作る。

**例：タイプ8のキャラクターの成長イベント**
- 通常：強い主張、本音、業界への挑戦
- 成長イベント：弱さの開示、感謝の表現、他者への寄り添い（タイプ2方向への成長）
- 具体的な投稿：「正直に言うと、最近少し疲れた。でもそんなとき、読者からのメッセージに救われた。いつも強くいる必要はないかもしれない」

このような投稿はキャラクターに**人間的な深み**を加え、読者との絆を深める。

## 6. 設計書の更新手順

### 6-1. 更新の基本ルール

1. **変更は常にユーザー確認を経る**
2. **変更理由を必ず記録する**
3. **変更前の状態をバックアップする（変更履歴に記載）**
4. **核に触れる変更は特に慎重に判断する**

## 7. 複数キャラクター運用時の成長管理

### 7-1. キャラクター間の差分維持

複数のキャラクターを運用する場合、成長によって個性が薄れないよう注意する。

**ルール：**
- 同じニュースに対する反応は、キャラクターごとに必ず異なる解釈にする
- 成長の方向は、パラメータの差異を維持する形で行う
- 定期的にキャラクター間の差分をチェックする

### 7-2. キャラクター間差分チェックテンプレート

```
【差分チェック】

テスト質問：「[業界の最新ニュース]についてどう思うか？」

キャラクターA の回答：
[回答]

キャラクターB の回答：
[回答]

差分スコア：[1-10]（10=完全に異なる、1=ほぼ同じ）

差分が不足している場合のアクション：
[どちらのキャラクターの、どのパラメータを強化するか]
```
