# 初心者向けマニュアル

顧客の悩みを大量に棚卸しするプロンプト群の使い方

（恋愛コンサルでも、副業コンサルでも、健康でも、キャリアでも、全部この手順で回せます）

---

## このプロンプト群でできること

- 見込み顧客が抱える悩みを **100〜150個レベルで一気に列挙**
- 取りこぼしを減らす（偏り防止の軸が入ってる）
- その悩みを **12〜20のクラスターに整理** して、商品設計・発信ネタに転用
- 必要なら Deep Research で **「本人語の引用＋出典付き」** にもできる

---

## まず覚える全体の流れ

初心者はこの順番だけ守ればOKです。

```
1. 入力情報を埋める
2. 汎用マスターで発散（100〜150個）
3. 追加モジュールで増量と偏り修正（必要なものだけ）
4. 整形プロンプトで整理（クラスター化）
5. Deep Research版で根拠補強（必要なときだけ）
```

---

## 準備するもの

最小限これだけあれば回せます。

- ジャンル（例：恋愛、副業、転職、営業、ダイエット）
- 顧客の状態（例：初心者、伸び悩み、失敗経験あり）
- 顧客のゴール（例：恋人を作る、月5万稼ぐ、内定を取る）
- 顧客の制約（例：時間がない、お金がない、家族が反対）

わからない項目があっても大丈夫。
その場合は「不明」と書いて進めるのがコツです（止まるより出す）。

---

## 入力欄テンプレ

まずこれを埋めてください。

```text
ジャンル：
顧客の状態：
顧客の望むゴール：
顧客の制約：
（任意）想定する代表セグメント：
（任意）避けたい表現やNG：
```

### 埋めるときのコツ

6項目ぜんぶ埋めなくてOK。初心者はこれだけで動きます。

- ジャンル：何の相談か
- 顧客の望むゴール：どうなりたいか
- 顧客の制約：何が邪魔してるか（時間・金・自信だけでOK）

それ以外（状態、セグメント、NG）は「不明」でも回せます。
むしろ、**不明のまま一回出して、出力を見てから埋め直す** のが正攻法。

### 10分ルール

```
1. 6項目は10分だけ悩む
2. 埋まらないところは「不明」or「仮」で埋める
3. そのまま悩みを100個出す
4. 出てきた悩みを見て「あ、こういう人だな」が分かったら入力欄を修正

入力欄は最初から当てにいくものじゃなく、生成結果で"逆に分かる"もの。
```

## 各項目が埋められない場合のAI補助

テンプレートを埋められない場合、AIに手伝ってもらえます。
以下を項目ごとにコピペして使ってください。

### ジャンルが決まらない場合

ジャンルは「テーマ名」じゃなく「誰が／何を／どの方向に」の形にするとブレません。

例：
- 恋愛 → 「恋人を作りたい人の恋愛」
- 副業 → 「会社員が在宅で月5万作る副業」
- 転職 → 「未経験で職種チェンジしたい転職」

```text
私はジャンルを決めたい初心者です。
「コンサルのジャンル」を、ブレにくい定義（誰が/何を/どの方向に）で
候補を10個出してください。

私が今考えているテーマ：
{{思いつく単語を1〜3個でOK 例：副業、SNS、在宅}}
```

### 顧客の状態が分からない場合

状態は「ステージ + 経験」の2軸。迷ったら「初期で停滞」に置くと悩みが一番出やすいです。

```text
ジャンルは{{ジャンル}}です。
見込み顧客の「状態」を、ステージ×経験で整理したいです。

ステージ（開始前/情報収集/初期行動/継続/停滞/失敗後/再挑戦/長期運用）
×
経験（初心者/中級/経験者）
で、代表的な状態を12個提案してください。

その中から「悩みが最も多そう」な上位3つも理由つきで選んでください。
```

### ゴールが思いつかない場合

```text
ジャンルは{{ジャンル}}。
見込み顧客が望みそうなゴールを「具体的」に20個出してください。

条件：
- 数字/期限/状態のいずれかを入れる
- 初心者が言いそうな口調
- 短期ゴール10個＋中期ゴール10個
```

### 制約が分からない場合

制約は「時間・金・スキル・環境・心理」の5つから選ぶだけです。
初心者なら「時間がない／何が正解か分からない／失敗が怖い」で十分。

```text
ジャンルは{{ジャンル}}。
見込み顧客が持ちがちな制約を、
「時間/お金/スキル/環境/心理」の5カテゴリでそれぞれ5個ずつ出してください。

さらに、そのジャンルで特に致命的になりやすい制約Top5を理由つきで選んでください。
```

### セグメントが分からない場合

```text
ジャンルは{{ジャンル}}です。
見込み顧客のセグメントを「2軸」だけで切りたいです。

汎用性の高い2軸を3案提案し、
それぞれで4象限（4タイプ）の特徴と典型的な悩みを5つずつ出してください。
```

### NG表現が分からない場合

```text
ジャンルは{{ジャンル}}。
見込み顧客に対して「避けたい表現・NG」を作りたいです。

- トーン面のNG（言い方）
- 倫理/誇大/断定のNG（内容）
に分けて、それぞれ10個ずつ提案してください。

さらに、代替の言い換え例も付けてください。
```

### 全部まとめてAIに埋めてもらう場合

6項目を悩んで止まるくらいなら、AIにこう頼むのが一番早いです。

```text
私は初心者です。以下の情報だけで、
テンプレ入力（ジャンル/顧客の状態/ゴール/制約/代表セグメント/NG）を
「仮置きで」3パターン作ってください。

私が分かっていること：
- ジャンルの候補：{{例：副業、恋愛、転職のどれか}}
- こういう人を助けたい：{{一言でOK}}
- 絶対にやりたくないこと：{{あれば}}

条件：
- 仮置きでよいが、なぜそう置いたか理由も一言添える
- 3パターンはターゲットの切り口を変える
```

→ 出てきた3パターンのうち「しっくりくる」やつを選んで、次のステップへ。

# ステップ1

## 汎用マスタープロンプトで悩みを100〜150個出す

### 使う場面

- とにかく悩みを大量に出したい
- まだターゲット理解が浅い
- ネタ切れしている

### やること

下のプロンプトを貼って、`{{ }}` を埋めて実行します。
（出力が少なかったら「150個まで増やして」と追い打ちするだけでOK）

### 汎用マスタープロンプト

```text
あなたは「顧客の悩みを大量に棚卸しするブレスト担当」です。
目的は、{{ジャンル}}コンサルの見込み顧客が抱える悩みを、具体的な本人語で大量に列挙し、カテゴリ別に整理することです。

【対象（ターゲット）】
- 相談領域（ジャンル）：{{例：恋愛 / 副業 / 転職 / 資格 / 起業 / 営業 / ダイエット など}}
- 顧客の状態（ざっくりでOK）：{{例：初心者〜中級 / 伸び悩み / 失敗経験あり / 自信がない など}}
- 顧客の望むゴール：{{例：交際したい / 月5万稼ぐ / 3ヶ月で転職内定 など}}
- 顧客の制約：{{例：時間がない / お金がない / 家族が反対 / スキル不足 など}}
※不明なら推測してよいが、その場合は「推測」と明記する。

【悩みの定義】
「本人が日常で口にする不満・不安・迷い・つまずき・恥・恐れ・面倒・焦り」を悩みとする。
抽象語だけで終わらせず、「何が」「どの場面で」「何に困るか」が分かる文にする。

【生成ルール】
- 1行＝悩み1つ（本人の一人称）
- "抽象語"禁止（例：不安、つらい、モヤモヤ → 何が不安かまで書く）
- 重複OK。ただし「状況・言い回し」を変えて粒度を増やす
- 最低100個。可能なら150個
- 以下の軸を必ず満遍なくカバーする（偏り防止）：
  1) ステージ：開始前/情報収集/初期行動/継続中/停滞/失敗後/再挑戦/長期運用
  2) 種類：行動（何をすれば）/判断（どれが正解）/感情（怖さ焦り）/対人（相手・周囲）/環境（時間・金）/能力（スキル不足）/リスク（失敗・損）
  3) セグメント：初心者/経験者、短期志向/長期志向、慎重派/行動派、自己流派/指導希望、コスト敏感/投資できる

【出力形式】
(1) まずカテゴリ一覧を提示（あなたが作る）
(2) 次にカテゴリごとに箇条書きで悩みを列挙
(3) 最後に「特に刺さりが強い悩み Top20」を選び、各悩みに1行で典型シーンも付ける

【カテゴリ例（必要なら改変OK）】
開始前の心理 / 情報収集・比較 / 初期のつまずき / 継続・習慣化 / 停滞・伸び悩み / 失敗・挫折 / 対人・コミュニケーション / お金・時間・環境 / 自信・自己肯定感 / 将来・長期不安
```

### 成功している出力の目安

- 悩みが「不安です」みたいな抽象で終わってない
- 「いつ・どこで・何が起きて困る」が想像できる
- 出会い系の悩みばかり、などの偏りが少ない
- Top20が「確かにこれ刺さる」と思える

# ステップ2

## 必要に応じて追加モジュールを足す

マスターの出力が「弱い」と感じたら、ここで補強します。
全部やる必要はありません。困っている症状で選べばOKです。

### 症状別おすすめ

#### 抽象的すぎる

→ 抽象→具体モジュールを追加

```text
抽象語が出たら、必ず具体化して言い直すこと。
具体化テンプレ：
「（いつ/どこで）〜している時に、（何が起きて）、（何に困る/何が怖い）」
例：
×「将来が不安」
○「このまま続けても成果が出る気がせず、時間だけ失って終わりそうで怖い」
```

#### 本人の言い方っぽくない

→ 独り言モジュールを追加

```text
悩みはできるだけ「独り言」っぽく書く。
語尾例：
「…だったらどうしよう」「自分だけ置いてかれてる気がする」「また失敗しそう」「結局どれが正解なの？」
```

#### 表に出ない悩みが足りない

→ 地雷・恥モジュールを追加

```text
次に「人に言いにくい悩み」も30個追加して。
例：恥、劣等感、嫉妬、見栄、依存、過去の失敗、家族に言えない、周りにバレたくない、など。
```

#### B2Bや組織が絡むジャンル

→ B2B対応モジュールを追加（副業、経営、営業、転職などで強い）

```text
もし対象がB2B/BtoC混在なら、悩みを以下の立場別にも出す：
- 本人（実行する人）の悩み
- 意思決定者（支払う/許可する人）の悩み
- 周囲（上司/家族/チーム/顧客）の反応による悩み
```

# ステップ3

## 整形プロンプトで「使える形」にまとめる

### 使う場面

- 悩みは出たけど、商品や発信に落とし込みづらい
- 大量のリストが散らかって見える
- 訴求の軸を決めたい

### やること

マスターの出力（悩みリスト）を貼って、これを実行。

```text
上で出した悩みリストを整理して、重複を統合しつつ「悩みクラスター」を12個作ってください。
各クラスターごとに以下を出す：

- クラスター名（短く）
- 代表的な本人語（3つ）
- 典型シーン（1つ）
- 背景要因（評価/制約/スキル/環境のどれが強いか）
- その悩みが解決すると得られる状態（ベネフィットを1文）
- 検索で使われそうなキーワード（5つ）
```

### ここまでできると何が嬉しいか

- クラスター＝そのまま「商品コンセプト」「講座の章立て」「発信カテゴリ」になる
- 代表的本人語＝LPや広告の見出しに使える
- 検索キーワード＝SEOやYouTubeのネタになる

# ステップ4（任意）

## Deep Researchを使うときの手順

Deep Researchは「悩みを出す」より、**悩みの裏付け（本人語の引用）** に強いです。
なので初心者はこう使うのがおすすめ。

### おすすめ運用

- まずマスターで **大量に出す**
- クラスター化して **重要テーマを絞る**
- 絞ったテーマに対して Deep Research で **本人語・出典収集**

### Deep Research用プロンプト

```text
目的：{{ジャンル}}領域の見込み顧客が抱える悩みを、公開情報から「本人語」で収集し、頻出パターンを作る。

やること：
1) Q&Aサイト、掲示板、SNS、ブログ等から「悩みが書かれた文章」を収集
2) 悩みをパターン分類（統合してクラスター化）
3) 各クラスターに「代表的な本人語（短い引用）」と出典リンクを付ける
4) オンライン情報の偏り（愚痴が集まりやすい等）も注記する

出力：
- クラスター名（12〜20個）
- 代表的な本人語（引用2〜3個、短く）
- 典型シーン
- 背景要因の仮説（仮説は仮説と明記）
- よくある誤解/やりがちな失敗
- 出典リンク
```

### Deep Researchでテンプレを自動で埋める方法

入力情報が不十分なとき、Deep Researchで逆算することもできます。

```text
目的：{{ジャンル}}領域の見込み顧客について、公開情報から
(1) よくある状態（ステージ）
(2) よく言われるゴール
(3) 代表的な制約
を抽出し、テンプレ入力欄を埋める。

やること：
- Q&A、掲示板、SNS、ブログなどから「悩み/目標/制約」が含まれる投稿を収集
- それらを頻出パターンにまとめる
- 可能なら本人語を短く引用し、出典リンクを添える

出力：
- ジャンル（定義）
- 顧客の状態：代表3タイプ（根拠つき）
- 顧客の望むゴール：代表5個（根拠つき）
- 顧客の制約：代表10個（根拠つき）
- 代表セグメント：2軸で提案
- 避けたい表現やNG：この領域で炎上しやすい言い方・誇大表現の例
```

# よくある失敗と対処法

## 悩みが抽象的になる

原因：条件がふわっとしてる or ルールが弱い
対処：抽象→具体モジュールを足す + 「シーン必須」にする
追い打ち例：

- 「全部の悩みに"いつ・どこで・何が起きて困る"を1文で付けて」

## 似た悩みばかり出る

原因：ステージや種類の軸が埋まってない
対処：マスターの「ステージ × 種類」を明示して増やす
追い打ち例：

- 「"停滞""失敗後""再挑戦"のステージだけで30個追加して」

## 悩みが綺麗すぎて刺さらない

原因：本人語になってない
対処：独り言モジュールを追加
追い打ち例：

- 「敬語禁止。独り言の口調で言い直して」

## 出力が多すぎて使えない

原因：整理工程を飛ばしてる
対処：整形プロンプトでクラスター化する
追い打ち例：

- 「重複を統合して、上位12クラスターに圧縮して」

# 最後に

## 初心者が最短で成果を出すコツ

- 最初から完璧な条件を埋めようとしない（不明で進めてOK）
- まず100個出す（量が質を連れてくる）
- 刺さるのはだいたい **Top20** にいる
- 使うためには必ず **クラスター化** までやる

## この後の流れ

ターゲットリサーチが完了したら、次は以下のステップに進みます：

```
1. ターゲットリサーチ完了（このマニュアル）
    ↓
2. コンセプト設計（SKILL_コンセプト設計.md）
   - 悩みクラスターを元に「自分はどの悩みを解決する人か」を決める
    ↓
3. キャラクター設計（SKILL.md）
   - コンセプトに合うキャラクター（ペルソナ）を自動設計
    ↓
4. コンテンツ生成（SKILL_コンテンツ生成.md / X専用 / Threads専用）
   - キャラクターの声で、ターゲットの悩みに刺さるコンテンツを生成
```
