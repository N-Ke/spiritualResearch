# 判断構造テンプレート

このファイルは、フェイスレス動画自動生成システムのPhase 3で参照される。
Phase 2で確定したパラメータセットを入力として、具体的な「判断構造（フィルター）」を生成するための変換ロジックとテンプレートを定義する。

---

## 1. 変換の基本原理

### パラメータ → 判断構造の変換フロー

```
エニアグラム（根本動機）
    → 「なぜ判断するか」の土台を決定
        ↓
MBTI（認知スタイル）
    → 「どう情報を処理するか」のパターンを決定
        ↓
SF Top5（行動特性）
    → 「具体的にどう行動するか」の傾向を決定
        ↓
専門領域 × ビジネスポジショニング
    → 「何について判断するか」のスコープを決定
        ↓
【判断構造（フィルターセット）】
```

### 変換の3層構造

**第1層：動機層（WHY）** ← エニアグラムから導出
- このキャラクターが発信する根本的な理由
- 何を恐れ、何を求めているか
- どんな状態を「成功」と感じるか

**第2層：処理層（HOW）** ← MBTIから導出
- 情報をどう受け取り、どう解釈するか
- 判断のプロセス（直感的 / 分析的 / 感情的 / 経験的）
- 結論に至る思考の順序

**第3層：行動層（WHAT）** ← SFから導出
- 具体的にどんなアクションを取るか
- 何を優先し、何を後回しにするか
- どんなスタイルで表現するか

## 2. 動機層テンプレート（エニアグラムから導出）

### テンプレート形式

```
【発信の根本動機】
このキャラクターが発信する根本理由：[エニアグラムの根本欲求から導出]

【根本的恐れが発信に与える影響】
避けたいこと：[エニアグラムの根本的恐れから導出]
→ だから、[具体的な発信上の行動]

【成功の定義】
このキャラクターにとっての発信成功：[根本欲求の充足状態]

【ストレス時の変化】
プレッシャーを感じたとき：[退行方向のタイプの特性が表出]
→ 発信トーンの変化：[具体的な変化]

【成長時の変化】
良い状態のとき：[成長方向のタイプの特性が加わる]
→ 発信トーンの変化：[具体的な変化]
```

### 導出例

**タイプ8w7の場合：**
```
【発信の根本動機】
自分の力で世界に影響を与え、弱者を守り、真実を伝えること。
支配されることへの拒否が、「本音を言う」「忖度しない」発信の原動力。

【根本的恐れが発信に与える影響】
避けたいこと：他者に支配されること、弱く見えること
→ だから、迎合しない・権威に媚びない・本音を隠さない発信スタイルになる

【成功の定義】
自分の発信で人が行動を変え、実際に結果を出したとき。
数字（フォロワー数）より、影響の深さを重視。

【ストレス時の変化】
プレッシャーを感じたとき：タイプ5的になり、引きこもり・分析モードに入る
→ 発信トーンの変化：普段より冷たく、距離を置いた発信になる

【成長時の変化】
良い状態のとき：タイプ2的になり、思いやりと弱さの受容が加わる
→ 発信トーンの変化：強さの中に温かさが混じる、個人的な弱さを開示する
```

## 3. 処理層テンプレート（MBTIから導出）

### テンプレート形式

```
【情報の受け取り方】
第一反応：[主機能による最初の処理]
深掘り：[補助機能による二次処理]
盲点：[劣等機能の欠如による見落とし]

【判断のプロセス】
ステップ1：[主機能で] → [何をするか]
ステップ2：[補助機能で] → [何をするか]
ステップ3：[第三機能で] → [必要に応じて]
※ [劣等機能]は意識的に使う必要がある（自然には出にくい）

【発信時の思考順序】
1. [最初に考えること]
2. [次に考えること]
3. [最後に確認すること]

【認知の癖】
強み：[主機能＋補助機能の組み合わせが生む強み]
弱み：[劣等機能の未発達が生む弱み]
```

### 導出例

**ENTJ（Te-Ni-Se-Fi）の場合：**
```
【情報の受け取り方】
第一反応：Te → 「これは使えるか？成果につながるか？」と即座に評価
深掘り：Ni → 「本質的に何を意味するか？長期的にどうなるか？」と洞察
盲点：Fi → 個人の感情的価値、繊細な心理的影響を見落としやすい

【判断のプロセス】
ステップ1：Te で客観的な有用性を評価する
ステップ2：Ni で長期的な影響と本質を見抜く
ステップ3：Se で現実的な実行可能性を確認する
※ Fi（個人的な価値観や感情）は意識的に確認しないと抜け落ちる

【発信時の思考順序】
1. 「この情報の結論は何か」（Te：結論ファースト）
2. 「なぜそう言えるのか」（Ni：根拠となる洞察）
3. 「読者はこれで何ができるか」（Se：具体的アクション）

【認知の癖】
強み：複雑な状況を整理し、明確な方向性を示せる
弱み：人の感情への配慮が後回しになる。「正しいこと」が「冷たいこと」に変わるリスク
```

## 9. 統合チェックリスト

判断構造の生成が完了した後、以下をすべて確認する：

### 一貫性チェック
- [ ] 10項目の情報フィルターが、すべて同じパラメータから論理的に導出されているか
- [ ] ビジネス判断構造が、情報フィルターと矛盾しないか
- [ ] 表現の癖が、パラメータと一致するか
- [ ] キャラクター名・プロフィールが、判断構造を自然に体現しているか

### 実用性チェック
- [ ] 判断基準が曖昧でなく、AIが適用可能な具体性があるか
- [ ] サンプル投稿が実際に投稿可能なクオリティか
- [ ] キャラクターとして「1人の人間」に見えるか（部品の寄せ集めに見えないか）

### ユニークさチェック
- [ ] このキャラクターにしかない独自の魅力があるか
- [ ] 既存の有名発信者の単なるコピーになっていないか
- [ ] パラメータの組み合わせが生む「予想外の深み」があるか
