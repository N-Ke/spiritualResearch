# ナレーション・台本ナレッジ統合参照ガイド

## このファイルの役割

フェイスレス動画台本生成時に、汎用テンプレート（`記事生成テンプレート.md`）ではなく、
**実戦で検証済みの固有ナレッジを優先的に適用する**ためのガイド。

**重要：このファイルにすべての台本ナレッジが記載されている。外部ファイルの読み込みは不要。**

### 大原則

```
台本生成の優先順位：

1. 固有ナレッジ（このファイルで参照するもの） ← 最優先
2. キャラクターの判断構造（設計書）
3. 汎用テンプレート（記事生成テンプレート.md） ← 補助的に使用

固有ナレッジが存在する媒体 → 固有ナレッジに従う
固有ナレッジが存在しない媒体 → 汎用テンプレートを使用
```

## 1. ショート動画（TikTok/IG Reels/FB Reels/YT Shorts） → 高密度フックナレッジを適用

### ショート動画台本の必須適用ルール

#### ルール1：3レイヤー構造を必ず適用する

```
レイヤー1：認知レイヤー（一瞬で「自分ごと」にさせる）
  - 入口は「目の前の行動」で切る
  - 大きい主語の抽象（「人は〜」「世の中は〜」）は禁止
  - 損失回避で引き込む（「得する」より「損する」訴求）
  - 次の文が読みたくなる「バトン文」を使う

レイヤー2：感情レイヤー（モヤモヤに"名前"を与える）
  - 言葉の癖（概念のラベリング）を使う
  - セリフを入れる（地の文より臨場感）
  - 時間と感情を移動させる

レイヤー3：行動レイヤー（次の一手が"勝手に見える"）
  - 「確認作業」が生まれる設計にする
  - 「見ればわかる特徴」まで具体的に語る
  - 抽象ワードで終わらない
```

#### ルール2：フック設計の具体テクニック

```
【最初の1〜3秒で勝負を決めるルール】

NG: 前置きから始める
「今日は○○について話します」「こんにちは」「いきなりですが」

OK: 結論・衝撃・共感から始める
「99%の人が間違えている○○」
「○○で悩んでる人、これだけ覚えて」
「知らないと損する○○の真実」

【フックの設計パターン】

パターン1：衝撃の数字
ナレーション：「○○を続けた結果、月収が3倍になった」
テロップ：「月収3倍」（大きなフォント）

パターン2：常識の否定
ナレーション：「みんながやってる○○、実は逆効果」
テロップ：「逆効果」（赤字強調）

パターン3：悩みの代弁
ナレーション：「○○で悩んでない？それ、実はこれだけで解決する」
テロップ：「解決策は1つだけ」

パターン4：好奇心の喚起
ナレーション：「○○の本当の意味、知ってますか？」
テロップ：「本当の意味」

パターン5：Before-After
ナレーション：「3ヶ月前：○○だった → 今：△△になった」
テロップ：「Before → After」（ビジュアル対比）
```

#### ルール3：ナレーションの具体テクニック

**損失回避（ショート動画で特に有効）：**
```
NG: 得する情報アピール
「知っ得情報！」「これを知ればうまくいく」

OK: 損の匂いをさせる
「これに気づいてないと損する…」
「気づかないうちに○○になっている」
「知らないだけで、周りは○○している」
```

**ラベリング（言葉の癖）：**
```
作り方：
① 日常でよくある行動を1つ選ぶ
② その心理を単語で書く（造語OK）
③ 比喩で言い換える
④ ナレーションに組み込む

実例：
・考えすぎて動けない → 「思考ループ」→「頭の中のぐるぐる洗濯機」
・情報ばかり集める → 「インプット中毒」→「学びのジャンクフード」
・完璧を目指して止まる → 「100点病」→「永遠の下書きマン」
```

**テンポ設計：**
```
【AI音声で自然に聞こえるためのテンポルール】

1. 一文は20文字以内を推奨
2. 句読点で間を取る（「。」=0.5秒、「、」=0.3秒）
3. 3文ごとに話題のミニ転換を入れる
4. 体言止めを効果的に使う
   例：「これが答え。○○する。それだけ。」
5. 強調したい単語は大きくゆっくり読む指示を入れる
   例：「これが【最大の間違い】なんです」
```

#### ルール4：視覚設計の具体テクニック

```
【テロップの鉄則】
1. 音なしでも100%内容が伝わるテロップ設計
2. 1画面に表示するテキストは5語以下
3. キーワードは色変え＋サイズアップ
4. ネガティブワードは赤系、ポジティブワードは緑/青系
5. 数字は常に大きく表示

【カット切り替えの鉄則】
1. 3〜5秒に1カット切り替え
2. 同じ映像が5秒以上続かない
3. テロップ切り替え = カット切り替えのタイミング
4. フック部分は1〜2秒に1カット（高速切り替え）
```

#### ルール5：型を消す

```
NG: チェックリストを埋める作業にしない
  「フックから始めよう」→ はい、衝撃的なこと言おう
  「テロップ入れよう」→ はい、大きい字にしよう
  → 「テンプレ感」が見える動画になる

OK: 3つのレイヤーが自然に融合している
  「この人の実体験」として聞こえる
  感情が自然に揺れる
  見終わった後に「やってみよう」と思う
```

### ショート動画のキャラクター統合方法

```
Step 1: キャラクターの判断構造でテーマを咀嚼する（従来通り）
Step 2: 咀嚼結果を「3レイヤー構造」に載せる
  - 認知レイヤー → キャラクターの「切り口」を「目の前の行動」形式で表現
  - 感情レイヤー → キャラクター固有のラベリングとセリフを使用
  - 行動レイヤー → キャラクターの解決策を「確認作業」形式に変換
Step 3: 型が見えないかチェック
Step 4: 媒体別にフックとCTAを微調整
  - TikTok: トレンド連動、ループ構造、「Part 2で」
  - IG Reels: 保存したくなる実用性、キャプション充実
  - FB Reels: 共感→シェアの流れ、テキスト大きめ
  - YT Shorts: ロング動画への導線、チャンネル登録CTA
```

## 2. YouTube ロング動画 → セミナー構成フレームワーク＋視聴維持率ナレッジを適用

### YouTube ロング台本の必須適用ルール

#### ルール1：視聴維持率を最優先した構成

```
【YouTube台本の基本フレームワーク】

1. フック（0:00~0:15）
   - 最初の15秒で視聴者を掴む
   - 「この動画を見ないと損する」と思わせる一言
   - 結論の一部を先に見せる（ネタバレではなく予告）
   - 動画全体の価値を1文で宣言する

2. 問題提起（0:15~2:00）
   - 視聴者の悩みを言語化する
   - 「あるある」で共感を得る
   - 「実はその考え方が間違い」で引きつける

3. 原因の深掘り（2:00~5:00）
   - なぜその問題が起きるのか
   - 一般的な解決策がなぜ効かないのか
   - 本当の原因はここにある（再定義）
   → 中間フック：「ここからが本当に大事な話です」

4. 解決策の提示（5:00~10:00）
   - 3つの具体的なステップで解説
   - 各ステップに具体例やエピソードを添える
   - 「これならできそう」と思わせる粒度

5. まとめとCTA（10:00~12:00）
   - 核心メッセージを1文で再確認
   - 「まず今日これをやってみてください」
   - チャンネル登録・関連動画への誘導
```

#### ルール2：感情操作の6段階を構造に組み込む

```
第1段階：衝撃（フック）
  - 常識を覆す事実
  - 信じられない数字
  - 予想外の展開

第2段階：共感（導入）
  - 視聴者の悩みの代弁
  - 「自分もそうだった」体験

第3段階：不安（問題の深掘り）
  - 「このままでは取り返しがつかない」
  - 現状維持の危険性

第4段階：欲求（解決策の導入部）
  - 理想の未来への憧れ
  - 具体的な成功イメージ

第5段階：確信（解決策の本編）
  - 他の方法との明確な差別化
  - 成功事例の詳細

第6段階：決断（まとめ＋CTA）
  - 今すぐ行動する理由
  - 具体的な第一歩の提示
```

#### ルール3：ナレーション品質の具体テクニック

```
【口語体のルール】
- 「〜なんですよね」「〜ですよね」を適度に使う
- 「えーっと」「あのー」は入れない
- 問いかけを各チャプターに1つ以上入れる
  例：「みなさんはどう思いますか？」
- 自然な口語体でありながら、論理構成は崩さない

【データとエピソードのサイクル】
- データ/数字 → エピソード → 分析 のサイクルで展開
- データだけでは退屈、エピソードだけでは浅い
- 3分以上同じトーンが続かないようにする

【AI音声ナレーション特有のルール】
- 一文は30文字以内（AI音声で自然に聞こえるため）
- 段落と段落の間に間を取る指示を入れる
- 10分動画の場合、ナレーション量は約3000〜4000文字
- 強調ワードは【】で指示
- 句読点の位置で間のタイミングを制御する
```

#### ルール4：SEO最適化の具体テクニック

```
【タイトルのルール】
- メインキーワードを含む
- 30文字以内
- 数字を入れる
- クリックしたくなる構成

【サムネイルテキストのルール】
- タイトルと重複しない
- 3〜5文字で衝撃を与える
- フェイスレスでもインパクトのあるビジュアル設計

【説明文のルール】
- 最初の2行で動画の価値を伝える
- チャプター（タイムスタンプ）を入れる
- 関連動画へのリンクを入れる
- キーワードを自然に散りばめる
```

#### ルール5：台本作成の禁止事項

```
■ AI的表現の禁止
  - 過度に整った箇条書き風のナレーション
  - 機械的な構成が透けて見える展開
  - 「でしょう」「しましょう」等のAI特有の語尾
  - 「ではないでしょうか」「していきましょう」等の定型表現
  - 同じフレーズの繰り返し

■ 棒読み感の排除
  - 感情の起伏がないフラットなナレーション
  - 解説文をそのまま読み上げるような構成
  - 強調ポイントが不明確な原稿

■ 二人称の過度な使用回避
  - 「あなた」の連発は避ける
  - 主語を省略するか、別の表現を使用
```

### YouTube ロング動画のキャラクター統合方法

```
Step 1: キャラクターの判断構造でテーマを咀嚼する（従来通り）
Step 2: 咀嚼結果を「視聴維持率最適化フレームワーク」に載せる
  - フック → キャラクターの「衝撃的な主張」を使用
  - 問題提起 → キャラクターの「独自分析」を使用
  - 解決策 → キャラクターの「判断基準に基づく解決アプローチ」
  - まとめ → キャラクターの「信念」を1文にまとめる
Step 3: 「感情操作の6段階」をチャプター展開に埋め込む
  - 衝撃 → フック
  - 共感 → 導入
  - 不安 → 問題の深掘り
  - 欲求 + 確信 → 解決策
  - 決断 → CTA
Step 4: ナレーションをキャラクターの「声」で仕上げる
  - トーン設定を維持
  - キャラクター固有の口癖を適度に散りばめる
  - 判断構造に矛盾する表現を使わない
Step 5: 禁止事項チェック（AI的表現、棒読み感の除去）
```

## 3. フェイスレス動画共通ナレッジ

### 3-1. ナレーションとテロップの連携ルール

```
【ナレーション ≠ テロップ】
- ナレーションの全文をそのままテロップにしない
- テロップは「キーワード抽出」が基本
- ナレーションが語る → テロップが要点を強調 → 映像が視覚化
- この3要素が同時に機能して初めて「伝わる」動画になる

【例】
ナレーション：「実は99%の人が間違えているんですが、○○の本当の意味は△△なんです」
テロップ：「99%が間違い」（大きく）→「本当の意味は△△」（次のカットで）
映像：「99%」の数字アニメーション → 「○○ → △△」のビジュアル変換
```

### 3-2. BGM・効果音の演出ルール

```
【BGMのルール】
- BGM音量はナレーションの20〜30%
- チャプターごとにBGMのムードを変える
- テンションの山場ではBGM音量を少し上げる
- フック直前の「一瞬の無音」は最強の演出

【効果音のルール】
- 使いすぎない（ショートなら2〜3箇所、ロングなら5〜8箇所）
- キーポイントの強調に使用
- 驚き効果音（「ドン！」）は最初の1回が最もインパクト大
- しつこい効果音は視聴者離脱の原因

【フェイスレス特有の演出】
- 映像の切り替えでリズムを作る（顔の表情変化の代わり）
- テロップのアニメーションで「感情」を表現
- BGMの変化で「場面転換」を演出
```

### 3-3. 映像素材の選定ルール

```
【B-roll（映像素材）の選び方】
1. ナレーション内容を「比喩的に」表現する素材
   例：「成長」→ 植物が成長する映像
   例：「分析」→ グラフやデータの映像
   
2. ナレーション内容を「直接的に」表現する素材
   例：「パソコン作業」→ パソコンを操作する手のアップ
   例：「チームワーク」→ 複数人が協力している映像

3. 「感情」を表現する素材
   例：「不安」→ 暗い色調の映像
   例：「希望」→ 明るい光、朝日の映像

【NGな素材選び】
- 素材からAI制作が透けて見える不自然な映像
- ナレーションと全く無関係な映像
- 同じ素材の使い回し（30秒以上）
```

