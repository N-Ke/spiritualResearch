# ターゲット設計テンプレート

ターゲット（読者・フォロワー）の悩み・不安をデータベース化し、コンテンツ生成時に参照するためのテンプレート。

---

## 1. ターゲットの悩みの3層モデル

ターゲットの悩みは、深さによって3つの層に分かれる。
コンテンツの効果は、**どの層を狙うか**で大きく変わる。

```
┌─────────────────────────────────────┐
│  レベル1：表層の悩み                    │
│  （本人が自覚し、言語化できている）        │
│  → 検索キーワードに出る、直接質問される    │
│                                       │
│  ┌─────────────────────────────────┐ │
│  │  レベル2：中層の悩み               │ │
│  │  （漠然と感じているが言語化できない）  │ │
│  │  → 「そうそう、それ！」と共感を得る  │ │
│  │                                   │ │
│  │  ┌─────────────────────────────┐│ │
│  │  │  レベル3：深層の悩み          ││ │
│  │  │  （本人も気づいていない）      ││ │
│  │  │  → 気づきを与え信頼を獲得    ││ │
│  │  └─────────────────────────────┘│ │
│  └─────────────────────────────────┘ │
└─────────────────────────────────────┘
```

### 各レベルの特徴とコンテンツ戦略

| レベル | 特徴 | コンテンツ効果 | 最適媒体 |
|--------|------|-------------|---------|
| レベル1 | 「〇〇のやり方が分からない」 | 即効性あり、ただし差別化が難しい | X、YouTube（ハウツー） |
| レベル2 | 「なんかうまくいかない」 | 共感→信頼構築に最適 | Threads、note |
| レベル3 | 本人は別の悩みだと思っている | 強い信頼と権威性を獲得 | note（長文）、YouTube |

### コンテンツ設計の基本戦略

**最も効果的なパターン：レベル1で引きつけ → レベル2で共感 → レベル3で気づきを与える**

```
例：マーケティング領域

レベル1（フック）：「SNSのフォロワーが増えない」
    ↓
レベル2（共感）：「頑張っているのに成果が出ない焦り」
    ↓
レベル3（気づき）：「フォロワー数を追うこと自体が間違い。
                    本当の問題は"誰に届けたいか"が不明確なこと」
```

## 2. ターゲット悩みデータベース

### 2-1. データベースの作成方法

以下のフォーマットで、専門領域ごとの悩みを蓄積する。

#### 新規作成テンプレート

```
---
領域: [専門領域]
ターゲット属性: [年齢層、経験レベル、状況]
作成日: [YYYY-MM-DD]
最終更新: [YYYY-MM-DD]
---
```

#### 悩みの登録フォーマット（1エントリ＝1つの悩み）

```
### 悩み ID: [連番]

レベル: [1 / 2 / 3]
カテゴリ: [悩みのジャンル]
悩みの要約: [一言で]
悩みの詳細: [具体的にどういう状態か]
ターゲットの言葉: [本人がどう表現するか（検索ワード、質問文）]
裏にある感情: [本人が感じている感情]
よくある誤解: [この悩みに対してターゲットが持っている間違った前提]
関連する深層悩み: [この悩みの裏にあるレベル3の悩み ID]
効果的な切り口: [この悩みをコンテンツにするときのベストな角度]
```

### 2-2. 事前調査の方法

ターゲットの悩みを調査するための5つの情報源：

#### 情報源1：検索キーワード（レベル1の悩みを発見）

```
【調査方法】
1. 専門領域の主要キーワードでGoogle検索
2. 「関連する質問」「他の人はこちらも検索」を収集
3. サジェストキーワード（a〜zまで）を確認

【収集フォーマット】
キーワード: [検索語]
検索意図: [何を知りたいか]
悩みレベル: [1]
データベースへの登録: [悩みの要約として登録]
```

#### 情報源2：SNSのコメント・リプライ（レベル2の悩みを発見）

```
【調査方法】
1. 同じ領域の発信者のコメント欄を読む
2. 「分かります」「自分もそうです」が多いコメントに注目
3. 質問形式のコメントを収集

【収集フォーマット】
コメント原文: [そのまま記録]
発見場所: [どの投稿のコメント欄か]
抽出した悩み: [言い換え]
悩みレベル: [2]
裏にある感情: [推定]
```

#### 情報源3：質問サイト（レベル2〜3の悩みを発見）

```
【調査方法】
1. Yahoo!知恵袋、Quora、Reddit等で領域名を検索
2. 長文の質問ほど深い悩みの可能性が高い
3. 「誰にも聞けない」「恥ずかしいのですが」がついた質問に注目

【収集フォーマット】
質問原文: [要約]
発見場所: [URL]
抽出した悩み: [レベル分けして記録]
本人の誤解: [質問文から推定]
```

#### 情報源4：自分のフォロワーの反応（レベル1〜3の発見）

```
【調査方法】
1. 過去の投稿でエンゲージメントが高かったものを分析
2. どの投稿のどの部分に反応があったかを特定
3. DM・リプライでの質問パターンを分析

【収集フォーマット】
元投稿: [どの投稿か]
反応パターン: [何に反応したか]
抽出した悩み: [推定される悩み]
悩みレベル: [推定]
```

#### 情報源5：AIによる推定（レベル3の仮説生成）

```
【調査方法】
1. レベル1とレベル2の悩みをAIに入力
2. 「これらの悩みの根本原因は何か」を問う
3. 生成された仮説を検証材料として使用

【注意】
AIの推定はあくまで仮説。実際のフォロワー反応で検証が必要。
```

### 3-2. 悩みクラスターマップ

関連する悩み同士をクラスター化し、コンテンツシリーズの設計に使う。

```
【クラスター名】[テーマ]

┌─ 悩みA（レベル1）
│   └─ 悩みB（レベル2）
│       └─ 悩みC（レベル3）← 根本課題
│
├─ 悩みD（レベル1）
│   └─ 悩みE（レベル2）
│       └─ 悩みC（レベル3）← 同じ根本課題
│
└─ 悩みF（レベル1）
    └─ 悩みG（レベル2）
        └─ 悩みH（レベル3）← 別の根本課題

【コンテンツシリーズ設計】
第1回：悩みA/D → レベル1の解決（入口コンテンツ）
第2回：悩みB/E → レベル2の共感と気づき
第3回：悩みC → レベル3の根本解決（信頼構築コンテンツ）
第4回：悩みF/G/H → 別角度からの展開
```

## 4. ターゲットペルソナ設計

### 4-1. メインターゲットの定義

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
ターゲットペルソナ
━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 基本属性
年齢層：[20代前半 / 20代後半 / 30代 / 40代 / 50代以上]
性別：[男性 / 女性 / 指定なし]
職業・状況：[会社員 / フリーランス / 経営者 / 主婦 / 学生 / 副業中]
経験レベル：[完全初心者 / 初級 / 中級 / 上級]
年収帯：[参考として]

■ 心理属性
情報収集スタイル：[検索型 / SNS流し見型 / 信頼者フォロー型]
意思決定スタイル：[即断型 / 慎重型 / 誰かに相談型]
モチベーション源：[恐怖回避 / 利益追求 / 承認欲求 / 自己実現]

■ このキャラクターとの関係
どうやってこのキャラクターを見つけるか：[発見経路]

最終的にどうなりたいか：[ターゲットのゴール]

■ 日常の中での悩みの出現パターン
いつ悩むか：[時間帯、シチュエーション]
何がきっかけで悩むか：[トリガーイベント]

何で検索するか：[検索キーワードパターン]

━━━━━━━━━━━━━━━━━━━━━━━━━━

```

## 5. 定期更新の仕組み

### 5-1. 悩みデータベースの更新トリガー

| トリガー | アクション |
|---------|-----------|
| 新しいコメント・質問が来た | 悩みとして登録可能か検討 |
| エンゲージメントが高い投稿があった | どの悩みに響いたか分析 |
| 業界のトレンドが変わった | 新しい悩みが生まれていないか確認 |
| 月次振り返り | データベース全体のレビュー |

### 5-2. 更新フォーマット

```
【データベース更新ログ】
日付：[YYYY-MM-DD]
更新内容：[追加 / 修正 / 削除]
対象悩みID：[ID]
変更理由：[きっかけ]
```

## 6. サンプルデータベース（マーケティング領域の例）

### レベル1の悩み

```
### 悩み ID: 001
レベル: 1
カテゴリ: SNS運用
悩みの要約: フォロワーが増えない
悩みの詳細: 毎日投稿しているのにフォロワーが増えない。何が悪いのか分からない
ターゲットの言葉: 「フォロワー 増やし方」「SNS 伸びない」
裏にある感情: 焦り、自己否定感
よくある誤解: 投稿量を増やせば自然に伸びると思っている
関連する深層悩み: 005
効果的な切り口: 「量」ではなく「誰に」を問う
```

```
### 悩み ID: 002
レベル: 1
カテゴリ: コンテンツ制作
悩みの要約: 何を発信すれば良いか分からない
悩みの詳細: ネタが思いつかない、自分の専門性に自信がない
ターゲットの言葉: 「発信 ネタ切れ」「何を書けばいい」
裏にある感情: 不安、自信のなさ
よくある誤解: 特別な知識や経験がないと発信できないと思っている
関連する深層悩み: 006
効果的な切り口: 「特別な知識」ではなく「あなたの視点」に価値がある
```

### レベル2の悩み

```
### 悩み ID: 003
レベル: 2
カテゴリ: 成果と焦り
悩みの要約: 頑張っているのに成果が出ない焦り
悩みの詳細: 周りは成功しているように見えるのに、自分だけ取り残されている感覚
ターゲットの言葉: （明確な検索ワードにならない。SNSで「分かる…」とリプライする）
裏にある感情: 焦燥感、比較による劣等感
よくある誤解: 他人の成功は見えるが、自分の進歩は見えていない
関連する深層悩み: 005
効果的な切り口: 「成果の定義」を問い直す
```

```
### 悩み ID: 004
レベル: 2
カテゴリ: 一貫性
悩みの要約: 発信に軸がなく、ブレている感覚
悩みの詳細: 何を発信しても中途半端で、自分らしさが分からない
ターゲットの言葉: （言語化できないが、フォロー解除や反応の低さで感じる）
裏にある感情: 迷い、アイデンティティの揺らぎ
よくある誤解: 表現（文体）を統一すれば軸ができると思っている
関連する深層悩み: 006
効果的な切り口: 軸は「表現」ではなく「判断基準」にある
```

### レベル3の悩み

```
### 悩み ID: 005
レベル: 3
カテゴリ: 自己価値
悩みの要約: 自分の存在価値への根本的な不安
悩みの詳細: 発信を通じて自分の価値を証明したいが、何をしても足りない気がする
ターゲットの言葉: （本人は「フォロワー数」「売上」の問題だと思っている）
裏にある感情: 深い不安、自己肯定感の欠如
よくある誤解: 外部の数字が自分の価値を決めると思っている
効果的な切り口: 価値は「数字」ではなく「影響」で測る
```

```
### 悩み ID: 006
レベル: 3
カテゴリ: アイデンティティ
悩みの要約: 「自分は何者か」が分からない
悩みの詳細: 発信者としての自分の立ち位置が不明確で、常に迷いがある
ターゲットの言葉: （本人は「差別化」「ポジショニング」の問題だと思っている）
裏にある感情: 存在意義への問い
よくある誤解: 差別化は「他人との違い」ではなく「自分の判断基準の明確化」
効果的な切り口: 「何者か」は外から定義するのではなく、判断の積み重ねで形成される
```
