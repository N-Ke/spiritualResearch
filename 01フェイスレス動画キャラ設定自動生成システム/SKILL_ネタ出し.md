# ネタ出し・コンテンツ計画スキル

## このスキルの目的

キャラクター設計書 + ターゲットの悩みデータベースを元に、「今日（今週）何の動画を作るか」のコンテンツアイデアを自動生成し、計画的な動画発信を支援する。

このスキルが **「何の動画を作るか」を決め**、コンテンツ生成スキル（SKILL_コンテンツ生成.md）が **「どう台本にするか」を実行する**。

---

## 前提条件

### 必須ファイル

| ファイル | 用途 | 必須度 |
|---------|------|--------|
| キャラクター設計書（完全版） | 判断構造・専門領域の参照 | 必須 |
| ターゲット悩みデータベース | 悩みクラスター・ペルソナの参照 | 推奨 |
| `04_SNS出力フォーマット.md` | 媒体特性の参照 | 参照 |

### ターゲット悩みデータベースが未作成の場合

「SKILL_ターゲットリサーチ.md」を先に実行するか、本スキル内で簡易的にターゲットの悩みを推定して進行する。

---

## 起動条件

ユーザーが以下のいずれかの意図を示したとき：
- 「ネタを出して」「ネタが欲しい」
- 「今週何の動画を作ればいい？」
- 「コンテンツ計画を立てて」
- 「動画のアイデアが欲しい」
- 「ネタ切れした」
- テーマの方向性を相談してきた場合

---

## 実行フロー全体像

```
Step 1: 現状の確認（キャラ・悩みDB・過去投稿）
    |
Step 2: ネタの種を生成（5つのソース）
    |
Step 3: ネタの選別・優先順位付け
    |
Step 4: コンテンツ計画の設計
    |
Step 5: ユーザー確認・選択
    |
Step 6: コンテンツ生成スキルへの引き渡し
```

## Step 1: 現状の確認

### ユーザーに聞くこと

```
コンテンツのネタ出しを開始します。

【必須】
1. どのくらいの期間分のネタが必要ですか？
   □ 今日の1本だけ
   □ 今週分（5～7本）
   □ 今月分（20～30本）
   □ お任せ

【任意（あれば精度が上がります）】
2. 最近の動画で反応が良かったテーマはありますか？
3. 最近気になっているニュースやトレンドはありますか？
4. 避けたいテーマはありますか？
5. 特にプッシュしたい商品やサービスはありますか？
```

### 内部で確認すること

```
【自動読み込み】
1. キャラクター設計書から：
   - 専門領域
   - 判断構造（フィルター）
   - ビジネスポジショニング
   - 媒体別表現設計

2. ターゲット悩みデータベースから（あれば）：
   - 悩みクラスターマップ
   - 各レベルの悩み一覧
   - まだコンテンツ化していない悩み

3. 過去のコンテンツ生成ログから（あれば）：
   - 直近で扱ったテーマ（重複回避）
   - 反応が良かったテーマ（強化候補）
```

## Step 2: ネタの種を生成（5つのソース）

5つの異なるソースからネタの種を生成する。
各ソースから3～5個、合計15～25個の種を出す。

### ソース1: 悩み起点（ターゲットの悩みから逆算）

```
【生成ロジック】
悩みデータベースの各エントリに対して：
  「この悩みを持つ人が、今日見て"これだ"と思うコンテンツは？」

生成パターン：
  悩み → 問題提起型ネタ（「まだ○○していませんか？」）
  悩み → 解決策提示型ネタ（「○○を解決する3つの方法」）
  悩み → 共感型ネタ（「○○で悩んでいた頃の話」）
  悩み → 気づき型ネタ（「○○の本当の原因は実は△△」）

※ 各ネタに推奨フォーマット（ショート動画 or ロング動画）を付与
```

### ソース2: キャラクター起点（判断構造から生成）

```
【生成ロジック】
キャラクターの判断構造（情報フィルター）を起点に：
  「このキャラクターが"黙っていられない"テーマは何か？」

生成パターン：
  判断基準 → 意見表明型ネタ（「○○について本音を言う」）
  判断基準 → 業界批判型ネタ（「○○は間違っている」）
  判断基準 → 自分語り型ネタ（「自分が○○を選んだ理由」）
  判断基準 → 予測型ネタ（「○○は今後こうなる」）
```

### ソース3: トレンド起点（最近の話題を咀嚼）

```
【生成ロジック】
ユーザーが提供したニュース/トレンド、または一般的なトレンドを元に：
  「このキャラクターがこの話題を見たら、何を言うか？」

生成パターン：
  トレンド × キャラクターの判断 → 独自の切り口
  トレンド × ターゲットの悩み → 「今だから知っておくべき○○」
  トレンド × 逆張り → 「みんなが○○と言っているけど実は...」

【トレンドが不明な場合】
ユーザーに聞く：「最近、業界で話題になっていることはありますか？」
なければ、一般的なトレンド（AI、働き方、SNSの変化など）を仮置きして生成。
```

### ソース4: シリーズ起点（悩みクラスターから連続企画）

```
【生成ロジック】
悩みクラスターマップを元に、コンテンツシリーズを設計：
  「このクラスターを3～5回の連載で解決するなら？」

生成パターン：
  クラスター → 第1回: 入口（レベル1の悩みで引きつける）
  クラスター → 第2回: 深掘り（レベル2の悩みに共感）
  クラスター → 第3回: 核心（レベル3に切り込む）
  クラスター → 第4回: 実践（具体的なアクションプラン）
  クラスター → 第5回: まとめ（ビフォーアフターを見せる）
```

### ソース5: 反復起点（過去の人気テーマの切り口変え）

```
【生成ロジック】
過去に反応が良かったテーマ（あれば）を、別の切り口で再生成：
  「同じテーマを、違う角度から語れないか？」

生成パターン：
  過去テーマ → 別の悩みレベルから語る
  過去テーマ → 別の媒体向けに再構成
  過去テーマ → 時間軸を変える（過去→現在→未来）
  過去テーマ → ターゲットを変える（初心者向け→中級者向け）

【過去データがない場合】
このソースはスキップし、他の4ソースで補う。
```

## Step 3: ネタの選別・優先順位付け

### 選別の基準（4軸評価）

生成した15～25個のネタを、以下の4軸で評価する：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
ネタ評価マトリクス
━━━━━━━━━━━━━━━━━━━━━━━━━━

評価軸1: ターゲット共感度（この悩みを持つ人が多いか）
  高 ←――――――→ 低

評価軸2: キャラクター適合度（このキャラクターが語る必然性があるか）
  高 ←――――――→ 低

評価軸3: 差別化度（他の発信者とは違う切り口か）
  高 ←――――――→ 低

評価軸4: 緊急性（今このタイミングで出す意味があるか）
  高 ←――――――→ 低

総合評価 = (共感度 + 適合度 + 差別化度 + 緊急性) / 4
```

### 優先順位の決定

```
A: 必ず出すべき（4軸すべてが高い）
B: 出す価値が高い（3軸が高い）
C: ストック候補（2軸が高い）
D: 見送り（1軸以下が高い）
```

### コンテンツミックスの設計

同じタイプのコンテンツが続かないよう、バランスを設計する：

```
【推奨コンテンツミックス（週7本の場合）】

問題提起型: 2本（「まだ○○していませんか？」）
ハウツー型: 2本（「○○の方法」）
体験談型: 1本（「○○した結果」）
意見表明型: 1本（「○○について本音」）
トレンド反応型: 1本（「○○が話題だけど...」）

【避けるべきパターン】
- 問題提起型が3本以上連続（批判的な印象になる）
- ハウツー型だけ（教科書的で面白みがない）
- 自分語りだけ（内向き過ぎる）
```

```

1. このまま進める → 「OK」

OKの場合、どの日のコンテンツから生成しますか？

```

## Step 6: コンテンツ生成スキルへの引き渡し

### 引き渡し情報

ユーザーが特定のネタを選択したら、以下の情報をSKILL_コンテンツ生成.md に引き渡す：

```
【コンテンツ生成スキルへの入力情報】

テーマ: [選択されたテーマ]
切り口: [決定した角度]
狙う悩み: [悩み番号と詳細]
推奨媒体: [媒体リスト]
コンテンツタイプ: [問題提起/ハウツー/体験談/意見表明]
コンテンツ計画の位置: [週の何日目か、月間計画のどこか]

→ SKILL_コンテンツ生成.md の Step 1 に上記をそのまま入力として渡す
→ ユーザーは追加の入力なしで生成に進める
```

### 連続生成モード

```
ユーザー：「今週分を全部作って」
    ↓
ネタ出しスキル：Week計画を確定
    ↓
Day 1のテーマ → コンテンツ生成スキル → 生成 → ユーザー確認
    ↓
Day 2のテーマ → コンテンツ生成スキル → 生成 → ユーザー確認
    ↓
...（繰り返し）
```

## ネタのストック管理

### ストックの仕組み

生成したネタのうち、すぐに使わないもの（C評価）をストックとして保存する。

```
【ネタストック】

ID: [連番]
テーマ: [テーマ]
切り口: [角度]
評価: [A/B/C]
ソース: [悩み起点/キャラクター起点/トレンド起点/シリーズ起点/反復起点]
対応する悩み: [悩み番号]
推奨媒体: [媒体]
保存日: [YYYY-MM-DD]
使用状態: [未使用/使用済み/期限切れ]
備考: [いつ使うと効果的か等]
```

### ストックの鮮度管理

```
- トレンド起点のネタ: 2週間で期限切れ（鮮度が落ちる）
- 悩み起点のネタ: 3ヶ月有効（悩みは変わりにくい）
- キャラクター起点のネタ: 6ヶ月有効（判断構造が変わらない限り有効）
- シリーズ起点のネタ: 使い始めたら完走するまで有効
```

## エラーハンドリング

### ネタが出しにくい場合

```
ネタが少ない場合、以下を試みます：
1. 悩みデータベースが不足 → ターゲットリサーチスキルの実行を提案
2. キャラクターの専門領域が狭い → 隣接領域への拡張を提案
3. 同じテーマの繰り返し → 切り口変えや対象変更を提案
```

### ユーザーが「もう全部書いた」と感じた場合

```
以下のアプローチで新しいネタを探します：
1. 同じ悩みを「別のレベル」で語り直す（レベル1→3に深掘り）
2. 同じテーマを「別のターゲット層」に向けて書き直す
3. 過去の人気コンテンツの「その後」を書く
4. 読者からの質問や反応をネタにする
5. 業界の最新動向と既存テーマを掛け合わせる
```
