# YouTube長尺台本生成スキル

## このスキルの目的

キャラクター設計書を元に、**YouTube ロング動画（8〜30分）** のフェイスレス動画台本を生成する。

SEO・視聴維持率を意識した構成で、タイトル・サムネイルコンセプト・チャプター構成・ナレーション全文を含む完全版台本を出力する。

---

## 前提条件

### 必須ファイル

| ファイル | 用途 | 必須度 |
|---------|------|--------|
| キャラクター設計書（完全版） | ナレーショントーン・判断構造の参照 | 必須 |
| `04_SNS出力フォーマット.md` | YouTube ロング動画の特性 | 必須 |
| `記事生成テンプレート.md` | 台本テンプレート | 参照 |
| `ライティングナレッジ統合参照ガイド.md` | 固有ナレッジの参照 | 最優先 |
| ターゲット悩みデータベース | 悩みの参照 | 推奨 |

### 呼び出し元

- `SKILL_コンテンツ生成.md`（Step 5からの委譲）
- ユーザーの直接指示

---

## 起動条件

ユーザーが以下のいずれかの意図を示したとき：
- 「YouTube動画の台本を作って」
- 「YouTubeロング動画の台本が欲しい」
- 「メイン動画の台本」
- YouTube ロング動画の台本生成を求めた場合

## 実行フロー

```
Step 1: 入力の確認（テーマ・核心メッセージ・推奨尺）
    ↓
Step 2: タイトル・サムネイルコンセプトの設計
    ↓
Step 3: チャプター構成の設計
    ↓
Step 4: フック構築（最初の15秒）
    ↓
Step 5: 全編ナレーション原稿の生成
    ↓
Step 6: テロップ・映像素材・BGM指示の生成
    ↓
Step 7: 品質チェック → ユーザー確認
```

## Step 1: 入力の確認

### SKILL_コンテンツ生成.mdからの委譲の場合

以下が既に決まっている：
- テーマ、核心メッセージ、解決策の構造、ターゲットの悩み
→ そのまま Step 2 に進む。

### ユーザーの直接指示の場合

```
YouTube ロング動画の台本を生成します。

【必須】
1. テーマ：何についての動画ですか？

【任意】
2. 推奨尺（デフォルト：10〜15分）
3. 伝えたい核心メッセージ（あれば）
4. 参考にしたい動画や構成（あれば）
5. 動画の方向性
   □ ハウツー解説型
   □ 問題提起・分析型
   □ ストーリーテリング型
   □ リスト・ランキング型
   □ Before-After型
   □ お任せ
```

## Step 2: タイトル・サムネイルコンセプトの設計

### タイトル設計

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
タイトル設計
━━━━━━━━━━━━━━━━━━━━━━━━━━

【SEO対策キーワード】
- メインKW：[検索ボリュームの大きいキーワード]
- サブKW：[関連キーワード2〜3個]

【タイトル案】（3パターン）

案1（衝撃型）：「[常識を覆すフレーズ] + [キーワード]」
  例：「99%の人が間違えている○○の正しい方法」

案2（メリット提示型）：「[具体的メリット] + [数字] + [キーワード]」
  例：「○○で月10万円稼ぐ5つのステップ」

案3（好奇心型）：「[なぜ?] + [結論の一部を隠す]」
  例：「なぜ成功する人は○○をしないのか」

→ キャラクターのエニアグラム × ビジネストーンから最適な型を推奨

━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Step 3: チャプター構成の設計

### 基本チャプター構成（10〜15分動画の場合）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
チャプター構成設計
━━━━━━━━━━━━━━━━━━━━━━━━━━

Chapter 0 [0:00〜0:15] フック
  → 最初の15秒で離脱を防ぐ
  → 動画全体の価値を予告

Chapter 1 [0:15〜2:00] 導入・問題提起
  → テーマの提示
  → ターゲットの悩みへの共感
  → この動画を見る価値の提示

Chapter 2 [2:00〜5:00] 問題の本質
  → なぜうまくいかないのか（構造的原因）
  → 常識の否定 or 再定義
  → キャラクターの独自分析

Chapter 3 [5:00〜9:00] 解決策
  → 解決策1の詳細（例＋根拠）
  → 解決策2の詳細（例＋根拠）
  → 解決策3の詳細（例＋根拠）

Chapter 4 [9:00〜11:00] 実践のポイント
  → よくある失敗パターン
  → 最初の一歩の具体化
  → キャラクターの経験からの裏付け

Chapter 5 [11:00〜12:30] まとめ + CTA
  → 核心メッセージの再確認
  → 今日からできるアクション
  → チャンネル登録 + 関連動画誘導

━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 視聴維持率を意識した構成ルール

```
【離脱防止のルール】

1. 20秒ルール：最初の20秒で「この動画を見る価値」を明示
2. 3分ルール：3分以内に「最初の気づき/発見」を提供
3. 中間フック：動画の中間地点（5〜6分）で新たなフックを入れる
   例：「ここからが本当に大事な話です」
4. 予告ルール：次のチャプターの内容を予告して離脱を防ぐ
   例：「次に解説する方法が、実は一番効果的なんですが…」
5. 変化ルール：3分以上同じトーン/テンポが続かないようにする

【エンゲージメント誘発ポイント】
- [2分] 質問：「みなさんはどう思いますか？コメントで教えてください」
- [6分] 確認：「ここまでの内容、役に立ちましたか？いいねで教えてください」
- [10分] CTA予告：「最後にまとめとプレゼントがあるので、最後まで見てくださいね」
```

## Step 5: 全編ナレーション原稿の生成

### 各チャプターのナレーション原稿

チャプターごとに以下の形式で生成する：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
[Chapter N] [チャプター名] [タイムコード]
━━━━━━━━━━━━━━━━━━━━━━━━━━

【このチャプターの目的】
[視聴者がこのチャプターで得る価値]

【ナレーション原稿】

「[ナレーション全文]」

--- 間 ---

「[ナレーション全文]」

※ 各段落の間に「--- 間 ---」を入れて、AI音声の読み上げ時のポーズを指示

【このチャプターのキーポイント】
- [要点1]
- [要点2]
- [要点3]

【次チャプターへの接続文】
「[次の内容を予告する一文]」
```

### ナレーション品質ルール（ロング動画用）

```
【全体の品質ルール】
1. 一文は30文字以内（AI音声で自然に聞こえるため）
2. 段落と段落の間に「--- 間 ---」を入れて間を取る
3. 10分動画の場合、ナレーション量は約3000〜4000文字
4. 導入部は話し言葉（「〜なんですよね」「〜ですよね」）
5. 本編は説明調＋話し言葉のバランス
6. まとめは感情を込めた語りかけ

【キャラクターの声の反映】
1. トーン設定（断定度・親近感・専門性）を各チャプターで維持
2. キャラクター固有の口癖を適度に散りばめる
3. 判断構造に基づく独自の分析視点を必ず含める
4. 「AIが書いた」と分かるような定型表現を避ける

【変化をつけるルール】
1. 3分以上同じトーンが続かないよう調整
2. データ/数字 → エピソード → 分析 のサイクルで展開
3. 問いかけを各チャプターに1つ以上入れる
```

## Step 7: 品質チェック

```
【タイトル・サムネイル品質】
- [ ] タイトルにメインキーワードが含まれているか
- [ ] タイトルがクリックしたくなる構成か（30文字以内推奨）
- [ ] サムネイルテキストがタイトルと重複していないか

【フック品質】
- [ ] 最初の5秒で視聴者の注意を引けるか
- [ ] 15秒以内に「この動画を見る価値」が伝わるか

【構成品質】
- [ ] 視聴維持率を意識した構成になっているか
- [ ] 中間フック（5〜6分地点）があるか
- [ ] チャプター間の接続が自然か

【ナレーション品質】
- [ ] キャラクターの「声」として一貫しているか
- [ ] AI音声で読み上げたとき不自然にならないか
- [ ] テンポの変化があり単調でないか

【フェイスレス品質】
- [ ] 映像素材の指示が具体的で調達可能か
- [ ] 10秒以上同じ映像が続いていないか
- [ ] グラフィックと映像のバランスが取れているか

【SEO品質】
- [ ] タイトルにキーワードが含まれているか
- [ ] 説明文の構成が適切か
- [ ] チャプター名が検索に有利か

【エンゲージメント設計】
- [ ] コメント誘導の問いかけが含まれているか
- [ ] いいね誘導が自然に挿入されているか
- [ ] チャンネル登録CTAが最後に含まれているか
- [ ] エンドスクリーン用の関連動画提案があるか
```

## ショートカットコマンド

| コマンド | 動作 |
|---------|------|
| 「YouTube動画の台本を作って」 | フル台本を生成 |
| 「タイトルだけ作って」 | タイトル＋サムネコンセプトのみ |
| 「フックを変えて」 | フック部分のみ再生成 |
| 「構成を変えて」 | チャプター構成のみ再設計 |
| 「もっと短くして」 | 尺を短縮（チャプター削除/圧縮） |
| 「もっと長くして」 | 尺を延長（チャプター追加/拡張） |
| 「ショート動画に切り出して」 | ロング動画からショート用フックを抽出 |
