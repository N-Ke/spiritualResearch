**重要**: 下表は**本教材が採用している並びの例**。**D を早めに置く**配列（`19` のタイプA例など）もあり得る。**正本はあなたの戦略シート + `05`**。

### フェーズ別フロー（教材）

- **Phase 1（1–2話）**: K → **N**（敵発見）  
- **Phase 2（3–5話）**: **I → C → B**（信頼・未来）  
- **Phase 3（6–7話）**: **E → D**（証拠・選別）  
- **Phase 4（8–10話）**: 販売 → FAQ → **U**

## AI 本文生成の絶対ルール（教材）

**一気に10話を書かせない**（[`13_本文執筆のコツ.md`](13_本文執筆のコツ.md) の1章ずつと同型）。

| 悪い | 良い |
|------|------|
| 10話まとめて生成 | **第X話だけ**と指定し、出力→チェック→次 |

### 生成プロンプト（骨子）

```
D-BE_UNICK に沿った LINE コピーライターとして、第[X]話のみ執筆。

■ 基本情報：キャラ名 / ターゲット / 商品概要
■ 本通：配信タイミング / メイン要素 / 目的 / 目標文字数
■ 戦略シート抜粋：[該当8要素]

■ ルール
1. 1行は短め（目安20文字以内）、1文ごと改行、2〜3文ごと空行（LINE可読性）
2. 末尾に「明日（次）の予告＝クリフハンガー」
3. 読者を責めない。敵は外部概念
4. 曖昧語を減らし具体へ。五感描写を2つ以上

■ 出力：件名案3 / 本文 / クリフハンガー一文
```

## LINE 特有の3つの罠（教材）

### 1. 文字の壁

長文1ブロックは読まれにくい。**改行・空行**を多めに（[`15_品質チェックと公開.md`](15_品質チェックと公開.md) の UX 観点と共通）。

**修正指示例**: 「スマホ1画面を想定し、1文改行、2〜3文で空行」

### 2. クリフハンガー不足

「また明日」だけでは弱い。**次に何が起きるか**を具体に。

### 3. 話し言葉・キャラの欠落

メルマガ調の堅さは LINE で浮く。**キャラ設計書**（`05`・Phase 1 キャラ）に寄せる。

---

## つまずき（教材）

1. **序盤から売り** → 本テンプレでは**第8話まで**商品詳細を抑える例  
2. **全通同じトーン** → 通ごとに**感情の主役**を変える  
3. **弱いクリフハンガー** → 次回予告を**ドラマの予告**のように具体化  
4. **読者責め** → 外部の敵・仲間トーン（`18`）  

---

## 迷ったとき（教材）

- **悩みが深い** → N・C を厚く  
- **悩みが浅い** → B・I を厚く  
- **高額** → E・D を厚く（**分割通も可**）  
- **低価格** → K・U（**倫理ある U**）を明確に  
- **文字不足** → 五感・数値（**真実**）・理由の深掘り  
- **文字過多** → 重複削除、次話へ移送、具体に置換

## FAQ（要約）

| Q | A（教材） |
|---|-----------|
| 10通は長い？ | **つまらない10通**が長く感じられる。配列と引きで「続きが読みたい」に寄せる |
| 配列変更は？ | 慣れるまま**テンプレ通し**→結果見て調整。熱いリストは前半圧縮等もあり |
| 1通の文字数 | 上表目安。第4話は長め可、第1・10は短く力強くも可 |

## まとめ（極意・教材）

1. **短すぎるだけのオファー**はリスクが高い、という主張（設計次第）

3. **LINE は改行と引き**  
4. **責めない・仲間**
