## なぜ Remotion か（教材）

手作業タイムライン（画像貼り・Ken Burns・テロップ刻み）は**尺が伸びるほど負荷が増えやすい**。仕組み化した Phase 1〜3 と並べると、**編集を「再現可能なパイプライン」に落とす**価値が出やすい、という整理。

**投下したい時間**: 企画（[`24`](24_ネタ出しの方法.md)）、改善（[`23`](23_D-BE_UNICKの微調整・稼働開始.md)・[`30`](30_データ分析と改善サイクル.md)）など。  
**減らしたい時間**: 毎回同型のカット編集・テロップ打ち。

## Remotion とは（教材）

**React（TypeScript）**でコンポジションを書き、**CLI でレンダリング**する動画生成。**JSON レイアウト**を差し替えて**同テンプレで量産**しやすい。

---

## 本ボルトでのフロー（`03_Remotion_Mac`）

```text
public/images/     … シーン画像
public/audio/      … narration.mp3（例）＋ timestamps.json
        ↓
npm run layout     … python3 generate_layout.py
        ↓
src/data/demo_seminar_layout.json  … パネル・テロップ・Ken Burns 等
        ↓
npm run studio     … プレビュー（ポートは Remotion 既定）
        ↓
npm run render     … MP4 出力（out/ 以下・スクリプト定義どおり）
```

**自動化の範囲（設計意図）**: パネル長・テロップ区切り・Ken Burns パラメータの**機械生成**（`generate_layout.py` のロジック）。**実装に無い機能**（例: 特定 BGM ミックス）は**別途**。

## 稼働ステップ（教材）

### 1. 素材配置

- `public/images/` … 表示順に `scene_01.png` 等（**実際のファイル名はレイアウトと一致**させる）。  
- `public/audio/narration.mp3`（例）と **`public/audio/timestamps.json`**。  
- `timestamps.json` は **generate_layout が期待する構造**（`alignment.characters` / `start_times` / `end_times`）に合わせる。

### 2. レイアウト生成

```bash
cd 04動画生成システム/03_Remotion_Mac
npm run layout
```

内部は **`python3 generate_layout.py`**。**句点（。）**等で文分割し、画像割当・テロップフレーム・Ken Burns バリエーションを組み立て、**`demo_seminar_layout.json` に書き出し**。

### 3. プレビュー

```bash
npm run studio
```

ブラウザでプレビュー。**テロップ同期・読みやすさ・Ken Burns・パネル切替**を確認。

### 4. 書き出し

```bash
npm run render
```

**出力パス・コンポジション名**は **`package.json` の `scripts.render`** を見る（本ボルト例: `DemoVideo` → `out/adsense_scene1.mp4`）。**本番用にファイル名を変える場合はスクリプトを編集**。

## レイアウト JSON のイメージ（教材）

実際のキーは **`src/types.ts` と生成スクリプト**が正。概念として:

- `fps` / `width` / `height`  
- `panels[]`: `image`, `startFrame`, `durationFrames`, `kenBurns`, `telopSegments[]`, `audio`（パネルごとに有無は実装による）

ユーザ原稿のサンプル JSON は**理解用**。**本番は生成物を参照**。

## 品質のコツ（教材）

| 領域 | 悪い例 | 良い例 |
|------|--------|--------|
| テロップ | 小さすぎ・背景と溶ける | 十分なサイズ・帯・フェード |
| Ken Burns | 静止・過激なスケール | **1.03〜1.08 程度**のゆるい変化（推奨レンジの例） |
| 同期 | 別音声と別 JSON | **同一ナレーションに対応した** timestamps |

**フォント・色・ハイライト**: `TelopSegmentView.tsx` 等。**キーワード強調**は `generate_layout.py` の `highlight_keywords` で `{…}` マーク等（実装参照）。

## Remotion と手動編集（教材）

| 観点 | Remotion | Premiere / CapCut |
|------|----------|-------------------|
| 学習 | コード・JSON | UI 操作 |
| 量産の再現性 | 高め | 手作業差が出やすい |
| 高度な VFX | 実装コスト | 向きやすい |

**週複数本・同型テンプレ**なら Remotion 寄り。**月 1 本・凝った演出**なら従来ツールも選択肢。

**代替**: Vrew・CapCut 等（**緊急・少数本**）。長期は**パイプライン整備**を推奨、という整理。
