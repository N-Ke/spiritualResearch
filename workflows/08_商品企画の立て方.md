## このレポートで分かること

売れる商品に必要な **「5つの大黒柱」** を理解し、AI を使って **設計図（企画書）** を作る流れを掴む。テーマだけ決めて第1章から書き始める **「9割の挫折パターン」** を避けやすくする（教材の主張）。

## Phase 2 に入る

Phase 1 で「誰が・誰に・どう発信するか」の基盤ができた。Phase 2 はそれを **売れるテキスト商品というパッケージ** に変換する。

本ボルトでは **`02オンラインテキスト商品化`** のパイプラインを使う。**最初のステップ（Step 0）が商品企画**（教材・フォルダ内の手順名は `AGENTS.md` 等で確認）。

### 方向性（Phase 1）と企画（Phase 2）の違い（比喩）

| | 内容 |
|---|------|
| **方向性** | 「だいたいこんな魚を釣りに行く」（例：台本に悩む副業層向けに数千円帯） |
| **企画** | 「竿・エサ・釣り方」の設計図（例：プロンプト3つ・1日30分で台本完成のロードマップ・1,980円） |

## なぜ「企画書」を先に作るのか

設計図なしに家を建てないのと同様、**企画書なしで執筆**すると次の **3軸** がブレやすい（教材）。

| ブレ | 症状 |
|------|------|
| **ゴール** | 「これも教えたい」で情報が増え、読者の「どうなれるか」が曖昧な辞書になる |
| **ターゲット** | 序盤は初心者、後半は専門用語だらけで離脱 |
| **キャラ** | AI 丸投げで個性が抜け、検索で足りる無個性な説明になる |

**企画書**＝この3つを固定する土台。

### 失敗の具体例（教材）

**1. 約束（ゴール）**  
- ❌ 「YouTube について網羅的に」→ 読者は「で、何ができる？」  
- ✅ 「週末2時間で動画1本を AI で完成」→ 未来がイメージできる  

**2. ターゲット**  
- ❌ 序章は初心者、第5章で CTR 相関の専門話 → 脱落  
- ✅ 最後まで「見てもらえる動画のチェック3つ」など初心者線を維持  

**3. キャラ**  
- ❌ 教科書調の一般論  
- ✅ 失敗談・苦手の開示から入る、この人にしか書けない語り  

## 企画を構成する5つの大黒柱

売れる商品には**構造**があり、その柱が5本（教材）。**1本でも弱いと傾く**という比喩。

**土台**＝ Phase 1（ジャンル・ターゲット・キャラ・方向性）

| # | 要素 | 読者が感じる価値 |
|---|------|------------------|
| ❶ | **深い悩み** | 「これ、自分のことだ」 |
| ❷ | **究極の約束（ゴール）** | 「このしんどさから抜けられそう」 |
| ❸ | **解決策（How）** | 「そのやり方ならできそう」 |
| ❹ | **コンテンツの形式** | 「迷わず最後まで実行できそう」 |
| ❺ | **独自性（Who）** | 「この人の言うなら試そう」 |

### 各柱の深掘り（要約）

**1. 深い悩み**  
Lv.1 暇つぶし → 売りにくい／Lv.2 困っている → 無料で足りうる／**Lv.3 切実**を狙う／Lv.4 今すぐ系 → 高単価も検討されうる  

**2. 究極の約束**  
❌「学べます」→ その先が見えない／✅「翌週から週1投稿が習慣になる」など**具体的な未来**  

**3. How**  
❌「頑張って書きましょう」／✅「プロンプト3つをコピペで30分」など**手順を売る**  

**4. 形式のパターン（例）**

| パターン | 向く悩み | 例 |
|----------|----------|-----|
| ステップ型 | やり方が分からない | 初心者が迷わない3ステップ |
| 失敗回避型 | 失敗が怖い | やってはいけない5つ |
| 比較型 | 選べない | A vs B、初心者はどっち |
| チェックリスト型 | 漏れが怖い | 公開前10項目 |
| ビフォーアフター型 | 変化への不安 | 月0→5万の話（事実に基づく） |
| テンプレ型 | 考えるのが苦手 | コピペテンプレ |

**5. 独自性（Who）**  
「専門家です」より、**経験・立場が言語化された肩書き**の方が信頼につながりやすい（教材）。**虚偽はNG**。

## 商品グレード（教材）

**数値は一例。** `06`・`00` と違う場合は Phase 1 の仮決定を優先。

| グレード | 価格帯（例） | 内容 | 文字数目安 | 付属物 | 制作目安 |
|----------|--------------|------|------------|--------|----------|
| **Lite** | 100〜980円（別表では500〜） | 単一悩み | 3,000〜6,000字 | なし | 1〜2日 |
| **Standard** | 980〜5,000円 | 単一悩みを深掘り | 8,000〜15,000字 | テンプレ1〜2 | 3〜5日 |
| **Premium** | 5,000〜30,000円 | 複数悩みを包括 | 20,000〜50,000字+ | ツール+テンプレ+WS 等 | 7〜14日 |

### グレード判定フロー（教材）

1. **単独で解決できる悩みか？** NO → Premium 候補  
2. **手順は何ステップ？** 3〜5 → Lite 寄り／6〜10 → Standard 寄り  
3. **「使える道具」は？** テキストのみ → Lite／テンプレ → Standard／ツール+WS → Premium  

**迷ったら Lite から**（成功体験を先に）。

## 【実践】AI を使った商品企画

### プロセス全体像

```
入力：キャラ設計書 ＋ 悩みクラスター（＋任意の追加悩み）
  → パイプライン Step 0：AI が5柱を揃えた企画案
  → ディレクター：全部盛りでないか／Howか／キャラの色か
  → 企画書確定 → Step 1 へ
```

### 1. 入力準備

- **キャラクター設計書**（MBTI・判断構造・口癖・AI用プロンプトを含む）  
- **商品化対象の悩みクラスター**（テキストで OK）  
- （任意）追加のターゲット情報  

### 2. 企画生成の指示（Step 0・例）

`02オンラインテキスト商品化` を開き、Phase 1 データを読み込ませたうえで、教材では例えば次のような依頼をする：

```
入力データをもとに、Step 0 の『悩み抽出と商品企画』を実行してください。

以下を含む企画書を生成してください：
1. 悩みリスト（5-10個）
2. 悩みクラスター分析
3. 無料記事の企画（1-3本）
4. Lite / Standard 商品の企画
5. Premium パッケージの企画（該当する場合）
6. 制作優先順位
```

（実際のプロンプトは **`02オンラインテキスト商品化`** 内の Step 0 手順が正本。）

### 3. ディレクターチェック

出力をそのまま採用しない。

- **全部盛り**（幕の内）になっていないか？  
- **What（知識）** ではなく **How（手順）** を売っているか？  
- **キャラの色**が出ているか？  
- 悩みは **Lv.3 以上**か？  
- **約束**は具体的か？  

## 初心者が陥る「3つの企画ミス」（教材）

### 1. 全部盛り

❌「YouTube 副業完全攻略」で章が無限に増える  
✅「AI で台本を30分で完成」に**1事象に特化**  

### 2. What を売る

❌「アルゴリズムの徹底解説」→「で、私は何を？」  
✅「アルゴリズムに好かれやすい動画の5ステップ」  

### 3. キャラが消える

❌ 教科書タイトル・中立説明のみ  
✅ 経験年数・失敗・立場がタイトルや約束に入る  

## つまずきポイント集

1. **最初から Premium** → 未完成・モチベ低下。**順序**: 無料記事 → Lite で「売れた」体験 → Standard → 実績後に Premium（教材）。  
2. **AI 出力を鵜呑み** → ディレクターが修正ループ。  
3. **悩みが浅い** → 「眠れない」「払ってでも聞きたい」レベルか確認。  
4. **競合コピー** → **競合が薄い穴**（例：編集が苦手な人専用の最小ガイド）。  

## 迷ったときのガイド（教材）

**Lite か Standard か**  
テンプレ要否・悩みの単純/複雑で分岐。**迷ったら Lite**。  

**ステップ型か失敗回避型か**  
「やり方が分からない」→ ステップ／「失敗が怖い」→ NG 回避型。  

**抽象か具体か**  
「読んだあと**今すぐ手を動かせるか**」を基準に具体へ。  

## 実務リスク（教材）

- **Inauthentic**：テンプレ量産・変化の乏しい商品もリスク。**企画段階でキャラ独自性を必ず入れる**。  
- **医療・健康**：断定は避け、体験・一般情報に。  
- **金融**：「必ず儲かる」等は避け、必要なら専門家相談の誘導。  
- **稼ぐ系**：「誰でも簡単」等の景表リスク → **再現条件・根拠**。  

## FAQ

| Q | A（要約） |
|---|-----------|
| 企画に何時間？ | 初回 **2〜3時間** 目安。生成30分→寝かせ→修正→決断（教材）。 |
| 最初の価格？ | **500〜980円の Lite** 推奨が多い。実績を積んで段階的に上げる（教材）。 |

## まとめ

1. **いきなり執筆しない** — 企画書でゴール・ターゲット・キャラを固定  
2. **5つの大黒柱** — 悩み・約束・How・形式・Who  
3. **AI は下書き** — 全部盛り・What 売りをディレクターが弾く  
4. **Lite から** — 成功体験を先に  
5. **キャラの色** — 「この人にしか書けない」を1つ以上
