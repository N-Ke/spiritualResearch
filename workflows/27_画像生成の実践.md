## このレポートで分かること

本ボルト同梱の **Google 画像生成パイプライン**（Nano Banana 2 → 失敗時 Pro）で、**16:9・harumi 系スタイル**を前提にシーン画像を量産する流れ、プロンプトの型、つまずき、**ポリシー・権利の観点**（教材）。

---

## なぜ AI 画像か（教材）

| 手段 | メリット | デメリット |
|------|----------|------------|
| フリー素材 | 無料 | 被り・世界観の統一が難しいことが多い |
| 有料ストック | 品質 | 枚数でコストが積み上がる |
| **AI 生成（本ツール）** | シーン指定・画風の統一・オリジナルになりやすい | API 料金・生成のブレ・規約理解が必要 |

**コスト・秒数・枚数**の例（100円以下、10〜30秒/枚等）は**目安**。

---

## 本ボルトのツール概要（正本はコード）

- **API**: Google（`google.genai`）。**Midjourney / DALL·E / SD 等には切り替えない**（`google_image_client.py` 冒頭の方針どおり）。  
- **モデル優先順**（実装例）: **Nano Banana 2**（`gemini-3.1-flash-image-preview`）→ 失敗時 **Nano Banana Pro**（`gemini-3-pro-image-preview`）。  
- **画風**: `generate_manga.py` 内の **`HARUMI_STYLE`**（柔らかい日本風アニメ調、**Ghibli inspired** の**一般的スタイル表現**。**商標・キャラの再現は避ける**）。  
- **16:9**: `HARUMI_STYLE` に含まれる。`scene_data.json` の各 `prompt` に **すでに `LANDSCAPE 16:9` が含まれる場合**は二重付与を避けるため**そのまま**使われる（`load_scenes` のロジック）。

## 初心者がハマりやすい点（教材）

1. **画風がバラつく** → 本パイプラインでは **HARUMI_STYLE をスクリプト側で前置**しやすい。  
2. **主人公の顔が変わる** → **全シーン同じキャラ記述**をコピペで固定。  
3. **構図がブレる** → **カメラ・構図・光源**を英語で明示。

## 作業フロー（教材）

```text
① 台本から「場面転換」をリスト化
② scene_data.json に id / filename / title / prompt を並べる
③ 01_画像生成_Mac で python generate_manga.py（または generate_manga.sh）
```

**枚数目安**: 約 **30 秒に 1 枚** → 10 分で **15〜20 枚**程度（調整可）。

## scene_data.json の例（教材）

各シーンは **シーン固有の英語プロンプト**のみでよい（**HARUMI が無い場合は `generate_manga.py` が前置**）。

```json
{
  "scenes": [
    {
      "id": "A-1",
      "filename": "scene1_intro.png",
      "title": "導入",
      "prompt": "Scene: A 68-year-old Japanese man with gray hair, gentle but worried expression, sitting at his kitchen table in the morning, looking down at documents. Soft morning light from the window."
    }
  ]
}
```

## 実行コマンド（教材）

```bash
cd 04動画生成システム/01_画像生成_Mac
python3 generate_manga.py
# 特定シーンのみ
python3 generate_manga.py --scene A-1 A-2
```

**個別再生成**: `regen_split_image.py` は **`--prompt` / `--prompt-file` / `--preset`** が主（**`--scene` は `generate_manga.py` 側**）。詳細はスクリプト先頭の docstring と `マニュアル_画像生成.md`。

## プロンプトの型（教材）

**品質の多くはプロンプトで決まる**、という整理（「9割」は**強い言い方の例**）。

1. **キャラ**: 年齢・髪・服装・表情を**具体的**に（英語推奨）。  
2. **シーン・構図**: 場所・カメラ・被写体の位置。  
3. **雰囲気・光**: morning light、melancholic 等。

**共通で付与されがちなもの**（`HARUMI_STYLE`）: 16:9、スタイル断片、**No text / no speech bubbles** 等 → JSON 側では**書かなくてよい**（重複回避ロジックあり）。

[`25`](25_視聴者を引き込む台本システム.md) のブロックに寄せた**作業用ラベル**（**D-BE_UNICK の N/I/K そのものではない**）。

## 画風を変えたいとき（教材）

`generate_manga.py`（および `regen_split_image.py`）の **`HARUMI_STYLE` 文字列**を編集する。**リアル寄り**等に差し替える例はユーザ原稿どおり可能。**権利・トーンがジャンルに合うか**は自己責任で検証。

## つまずき（教材）

| 現象 | 原因の例 | 対策 |
|------|----------|------|
| 正方形に近い | プロンプトだけ API に投げて 16:9 指示がない | **`GoogleImageClient` ＋ 本スクリプト**で HARUMI 前置 |
| 顔が変わる | キャラ記述がシーンごとに違う | **同一ブロックを全プロンプトに** |
| 文字が入る | モデルのブレ | **No text** を強める・再生成 |
| AI っぽい | 指・目など | 再生成、Pro 側に回る、**natural / subtle** 等を少量追加 |
| 遅い・エラー | レート制限・長文プロンプト | **短文に分割**・**待機**・公式の上限確認 |

## 権利・YouTube（観点・非法律意見）

- **有名キャラ・実在人物のそっくり生成**は避ける。  
- **他人著作物の丸パクリ**は不可。  
- **テンプレ量産のみ**は「信頼性・オリジナリティ」の観点で不利になりうる → **台本・キャラ・編集で付加価値**を。  
- 詳細は **YouTube ヘルプ**と**必要なら専門家**へ。
