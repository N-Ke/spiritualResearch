## このレポートで分かること

Faceless で収益化するために重要な考え方 ― **「動画を出す前に、売る仕組みを先に作る」** ― を理解し、行動計画の軸をブレさせにくくする。

「最初に動画を出さないの？」という感覚は普通。多くの人がそう考える。しかしそれは**よくある失敗パターン**でもある。

## なぜ「順番」を最初に学ぶのか

多くの挫折は「やり方が分からなかった」ではなく、**「やる順番」を間違えただけ**であることが多い。

料理に例えるなら、「カレーの作り方を知らなかった」のではなく、「鍋に水を入れる前にルーを入れた」ようなもの。正しい手順を知れば初めてでも再現しやすい。

このレポートは Faceless 収益化の**レシピの最初のページ**に相当する。

## 一般的な失敗パターン

### よくある失敗の流れ（例）

1. まず動画を投稿し始める  
2. 再生が伸びない…  
3. それでも投稿を続ける（数ヶ月消耗）  
4. 少し視聴者がつく（例：チャンネル登録100人前後）  
5. 「何か売ろう」と考え始める  
6. 急いで商品を作る（さらに時間消耗）  
7. セールスの仕組みがない  
8. 売れない  
9. 挫折  

**問題は動画の質だけではなく、順番が間違っていること。**

## なぜ「動画先行」が不利になりやすいのか

本質は **リード（見込み客）の損失**。

- 動画を見て「面白い」「役に立つ」と思っても、**LINE 等の受け皿がない**と離脱しやすい。  
- 初期に表示が付きやすい時期に、**受け皿がないと機会損失**になりやすい。  

比喩：**穴の空いたバケツに水を汲む**のに近い。

- **仕組みなし**: 視聴 → 受け皿なし → 流出  
- **仕組みあり**: 視聴 → LINE 等に滞留 → ステップで説明・販売  

## 「仕組み先行」が推奨される理由（要約）

### 1. 最初の接触が貴重

初期に動画を見てくれる層は関心が高いことが多い。**受け皿があれば**、少人数でも次の行動につなげやすい。

### 2. 事例（教材上の数字）

| 項目 | 例（教材） |
|------|------------|
| 再生 | 約600回 |
| LINE登録 | 15人 |
| 商品単価 | 1万円 |
| 成約 | 2名 |
| 売上 | 2万円 |

※個人差・単価・導線で変動。**再現を約束するものではない。**

### 3. 精神的な負担の差

仕組みが先に「完成している」イメージがあると、投稿の意味付けがしやすい。**何も売れない不安だけが続く状態**と比べ、改善の対象も見えやすい。

### 4. データで改善しやすい

受け皿・配信・商品ページが揃うと、**どの段階で落ちているか**を指標で見やすくなる。

| 段階 | 計測の例 |
|------|----------|
| 動画 | インプレッション、CTR |
| 概要欄 | クリック率 |
| LINE | 登録率 |
| ステップ | 開封・精読 |
| 商品ページ | 訪問率 |
| 成約 | 成約率 |

仕組みがなければ、上記を一貫して追いにくい。

## 正しい順番：Faceless 収益化ワークフロー

1. **Phase 1：企画・設計（1〜3日）** — ジャンル、ターゲット、キャラ、商品の方向性、LINE 準備（詳細は `phase1_line…`）  
2. **Phase 2：商品を作る（2〜4週間）** — 企画、無料記事、有料商品、品質、公開  
3. **Phase 3：セールスシステム（1〜2週間）** — D-BE UNICK、ステップ配信、LP・特典、テスト  
4. **Phase 4：集客動画（継続）** — ネタ、台本、制作、投稿、LINE 誘導、改善  

**Phase 2 と 3 が「受け皿」、Phase 4 が「人を流すホース」。**

## 「動画先行」と「仕組み先行」の比較

| 項目 | 動画先行になりがち | 仕組み先行 |
|------|-------------------|------------|
| 最初にやること | 投稿優先 | 商品とセールス設計 |
| 初期の収益 | 売るものがなければ0に近い | 仕組み次第で早期に可能性 |
| リード | 流出しやすい | 受け皿に誘導しやすい |
| 精神的負荷 | 結果が見えにくいと高め | マイルストーンが見えやすい |
| データ | 取りにくい | 段階ごとに取りやすい |

### 「動画を先の方が早くない？」

**「やっている感」は早いが、受け皿がなければ収益は立ちにくい。**  
仕組み構築に要する期間（目安 1〜2ヶ月）を投資として扱うか、成果の見えにくい投稿に使うか、は設計の問題。

### 「動画で練習は？」

Phase 1〜3 の間にも**軽い素振り**は可能（競合視聴、ネタストック、音声・画像・Remotion の試し等）。Phase 4 で本番に近い品質を出しやすくする、という位置づけ。

## 「仕組み」のレイヤー構造（概念）

1. **集客**: 動画（Phase 4）… 概要欄等で LINE へ  
2. **誘導**: LP（Phase 3）… 特典・登録  
3. **信頼**: LINE ステップ（Phase 3）… フレームワークに沿った配信  
4. **販売**: テキスト商品（Phase 2）… Lite からでも可  

## 成長シミュレーション（教材・控えめ想定の例）

前提例：商品 1,000円、LINE 登録率 2%、成約率 10%、週2本投稿 等。

| 月 | 内容のイメージ |
|----|----------------|
| 1-2 | 仕組み構築、売上0円期間 |
| 3〜 | 投稿増、登録・成約が積み上がる想定 |

※**あくまでシミュレーション例**。単価変更・商品追加で伸び方は変わる。

## 初心者が陥りやすい罠と回避

1. **「みんな動画から」** — 広告収益・既に商品がある人向けの助言と、**商品販売型フェイスレス**は別。自分のモデルを混同しない。  
2. **「仕組み作りで飽きる」** — チェックリスト・毎日1成果の記録・並行で軽い素振り。  
3. **「商品が売れるか不安」** — Phase 1 のターゲットに立ち返る。Lite で試す。  
4. **「完璧にしてから公開」** — 60点で動かし、データで改善。Phase 3 はテスト送信が通ることを完了基準にしやすい。

## AI プロンプト集（要約）

教材では次の3系統が紹介されている（詳細は講義原本でも可）。

1. **現状診断** — チャンネル有無、商品有無、LINE有無、時間、目標月収から「どちらの順番にいるか」「最優先行動」「スケジュール案」を出させる。  
2. **仕組み構築スケジュール** — 利用可能時間・得意苦手から Phase 1〜3 の日別・週別タスクと所要時間。  
3. **挫折防止** — 今の Phase・詰まり・不安から、意味の再確認・進捗率・残り日数・「今日1つだけ」を提案させる。  

## FAQ（要約）

| Q | A（要約） |
|---|-----------|
| チャンネルは先に作る？ | 作成だけは早めでも可。**本番投稿は Phase 4 まで待つ**のが無難、という整理。 |
| もう動画を出している | 削除不要。Phase 1〜3 を進め、完成後に概要欄へ LINE 等を足すと過去動画も資産になりうる。 |
| お金は？ | 最小は LINE 無料 + note 無料 等。拡張ツールは規模に応じて。 |
| Phase 順を入れ替え？ | **推奨しない**。1→2 は土台と商品、2→3 は売る物と販売設計、3→4 は受け皿と集客の順。 |
| 1人でできる？ | ワークフローは1人運用を想定した設計、という位置づけ（AI・テンプレ活用前提）。 |

## 実務リスク（教材より）

1. **作り込みすぎ** — Lite や通数を小さく始め、データ後に増やす。  
2. **市場不安** — 一時トレンドではなく**持続的な悩み**か Phase 1 に立ち返る。  
3. **ツール変更** — 手順は概念でも残す。Phase 4 前に仕様確認。

## まとめ

1. **受け皿を先に** — 商品 + セールスシステム（Phase 1→2→3）  
2. **人を流す** — 集客動画（Phase 4）  
3. **仕組みが動く** — 改善は数字で  

**適切なターゲットに、適切な商品を、適切なタイミングで届ける仕組み**が、少ない数字でも収益の芽を出しやすくする、という考え方の土台。
