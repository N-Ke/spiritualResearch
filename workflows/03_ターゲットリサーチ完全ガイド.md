## このレポートで分かること

ターゲットリサーチは Faceless 収益化の**土台**。AI を使って**短時間で「悩みデータベース」**を揃える考え方と手順が身につく。

Report 02 でジャンルを「土地」に例えたなら、ターゲットリサーチは **その土地に住む人の本音の悩みを掘る作業**。金鉱のたとえでは、**金脈の位置を当てる地質調査**に近い。

## なぜターゲットリサーチが土台なのか

ここで作る **悩みデータベース（目安：100〜150個）** は、以降すべてに効く。

| 活用先 | 内容 |
|--------|------|
| **Phase 1** | コンセプト・キャラ・商品の方向性 |
| **Phase 2** | 商品テーマ・章立て |
| **Phase 3〜4** | ステップの訴求順・動画ネタ・シリーズ企画 |

**リサーチの質 ≒ 全体の成果の質**（個人差・実行あり）

## ターゲットリサーチとは

一言でいうと、**「ターゲットが何に悩んでいるかを、できるだけ網羅的に把握する」**こと。

### なぜ「属性」ではなく「悩み」か

**属性リサーチ（不十分の例）**

「30代の会社員で副業に興味がある人」  
→ 何を売るか・動画で何を話すか・ステップで何を伝えるかがまだ曖昧。

**悩みリサーチ（十分に近い例）**

「副業を始めたいが何から手をつけていいか分からず、毎日 YouTube の副業動画を見ているだけで3ヶ月が経っている。家族には内緒。プログラミングは苦手。月3万円から。」

**悩みから生まれる成果物の例**

- 商品：プログラミング不要で月3万円を目指すロードマップ  
- 動画：副業動画を100本見ても行動できない理由  
- ステップ：家族に気づかれずに始める方法  

**悩みは具体的であるほど、設計の武器になりやすい。**

## 悩みの具体レベル（5段階）

| レベル | 例 | 使いやすさ |
|--------|-----|------------|
| Lv.1 | 「お金に困っている」 | 抽象的すぎる |
| Lv.2 | 「副業で稼ぎたい」 | まだ広い |
| Lv.3 | 「YouTube を始めたいがやり方が分からない」 | 使える。さらに深掘り可 |
| Lv.4 | 「台本が書けない。AI を使いたいがプロンプトが分からない」 | 商品テーマが見えやすい |
| Lv.5 | 「ElevenLabs で音声を作ったが日本語の発音がおかしい。どう設定すれば自然か」 | ピンポイント。成約につながりやすいことも |

**目安：Lv.4〜5 を 100〜150 個の束の中に十分含める**（教材の目標）

## リサーチの全体フロー

| Stage | 内容 | 担当 |
|-------|------|------|
| **1 入力** | ジャンル・ゴール・制約（数分） | あなた |
| **2 悩み棚卸し** | 悩みを大量生成 | AI（＋あなたが指示・検証） |
| **3 クラスター分類** | 例：12 カテゴリに整理 | AI |
| **4 ペルソナ設計** | 理想の視聴者像 | AI（＋あなたが調整） |

**作業イメージ（教材）**: 入力＋確認で **約30分〜数時間**（深掘り・修正で伸びる）。AI は生成・分類の下支え。

### Stage 1：入力

AI は文脈が具体的なほど出力が具体化しやすい。

**入力する3つ**

1. **ジャンル** — 何について発信するか（例：50代からの年金・老後資金）  
2. **ゴール** — ターゲットにどうなってほしいか（例：老後の資金不安が和らぎ安心して暮らせる）  
3. **制約** — 対象外（例：不動産投資は対象外）  

**悪い例**: ジャンル「お金」、ゴール「稼げるようになる」、制約なし → 広すぎて抽象化しがち。  

**良い例**: ジャンル「顔出しなしの YouTube 副業」、ゴール「月10万円の副収入」、制約「プログラミング不要」。

### Stage 2：悩み棚卸し

#### なぜ 100〜150 個なのか（教材の理屈）

| 悩みの数 | 出やすい質 | 活用 |
|----------|------------|------|
| 〜10 | 表面的 | 設計には不足しがち |
| 〜30 | 中間 | 動画ネタ程度 |
| 〜50 | 深い層が増える | 商品設計に使いやすい |
| 100〜 | 本人も自覚しにくい層 | 差別化・バズの種になりうる |

**生成例のイメージ（副業 YouTube 系）**

1. 何のジャンルで始めればいいか分からない  
47. AI の声が機械っぽくて不安  
98. 動画を出すのが怖い（批判されたくない）  
100. 自分が人に教えていいのか自信がない  

#### ディレクターの確認ポイント

- **具体性**: Lv.4〜5 が十分あるか。少なければ「シチュエーション付きで具体化」と指示。  
- **網羅性**: 技術・心理・時間・金銭・人間関係・将来 など、偏りがないか。  
- **重複**: 表現違いの同一悩みを統合するよう指示。  

### Stage 3：12 クラスターに分類

バラの悩みを**束ねる**と、「どの領域が厚いか」が見える。

**活用イメージ（例）**

- クラスター1 始め方系 → 無料記事のテーマ  
- クラスター3 台本系 → Lite 商品の候補  
- クラスター6 AI 活用系 → Standard 候補  
- クラスター10 収益拡大系 → Premium 候補  
- クラスター11 メンタル系 → ステップの共感パート  

**目安**

- **悩みの数が多い × 悩みが深い** → 商品化の最有力候補  
- **数は多いが浅め** → 集客動画のシリーズ向き  

### Stage 4：ペルソナ設計

**ペルソナなし**  
「副業に興味のある方へ」→ 誰か特定しにくく刺さりにくい。  

**ペルソナあり**  
「毎日23時帰宅で子どもの寝顔しか見られない35歳のあなたへ。月3万円あれば月1回は外食できるのに」→ 自分ごと化しやすい。  

#### ペルソナ設計書の構成（項目）

- 基本情報（名前・年齢・性別・職業・年収・家族 等）  
- **現状（Before）** — 一日の流れ、悩みトップ5、すでに試したこと  
- **理想（After）** — 具体的な状態、何が変わるか  
- **最大の恐怖・購買** — 行動を止める不安、お金を払う基準、「買わない理由」  

#### 具体例（教材：架空の合成例）

**田中太郎（35歳・会社員）**

| 項目 | 内容 |
|------|------|
| 職業 | IT 系中間管理職、年収550万円目安 |
| 家族 | 妻1、子2人（5歳・3歳） |
| 現状 | 毎日22時帰宅、ローンあり、副業動画を3ヶ月見ただけで行動できていない |
| 理想 | 月10万円副収入、顔出しなし、仕組み化 |
| 恐怖 | 稼げず時間だけ溶かす、家族にバレる |
| 購買 | 1000円以下は衝動買いしやすい、5000円以下が心理的に安心 等 |

## 【実践】AI への指示方法

### 方法①：一番ラクな流れ（本ボルトのシステム向け）

「**初回セットアップを開始して**」等、**フェイスレス動画キャラ設定自動生成システム**の手順に従う。  
ターゲットリサーチ → コンセプト → キャラまで一気通貫になる設計が多い。各ステップの確認ポイントで OK を出す。

### 方法②：ターゲットリサーチだけ実行する場合

以下を AI に渡す（要約プロンプト）。

```
【ジャンル】〇〇
【ゴール】ターゲットに〇〇できるようになってほしい
【制約】〇〇は対象外

出力：
1. 悩みリスト（100-150個）。各悩みに深さタグ（浅い/中/深い）。シチュエーションを含める。
2. 12クラスター分類。クラスター名、件数、平均的な深さ。
3. ペルソナ1〜2名。基本情報・現状・理想・恐怖・購買行動。
```

### 方法③：既存クラスターの深掘り

```
【深掘りしたいクラスター名】〇〇
【現在の悩みの数】〇個

1. このクラスター内に悩みを20-30個追加
2. 追加はすべて Lv.4-5 の具体性
3. 「口には出せない本音」を含める
4. 各悩みに「刺さる一言」（動画タイトル案）を1つ
```

## リサーチ結果の使い道

### Phase 1

- クラスター → **コンセプト**：どの束の悩みを自分が解決するか  
- クラスター → **キャラ**：その悩みに響く人物像  
- クラスター → **商品方向**：深い束をテーマ候補に  

### Phase 2

- クラスター → **商品テーマ**  
- 個別悩み → **章立て**（悩み1→第1章 等）  

### Phase 3〜4

- 深さ順 → **ステップの訴求順**（例：序盤は共感、中盤で深い悩み、後半で解決策・商品）  
- 個別悩み → **1本1ネタ**  
- クラスター → **シリーズ企画**  

## 初心者が陥りやすい失敗と回避

### 1. 自分の頭だけで終わらせる

- まず自分で 10〜20 個書く  
- AI に「これに加えて100個」と依頼  
- AI 出力で「それもある」と気づく  
→ **経験 × 網羅性** で偏りを減らす  

### 2. 悩みが浅いまま進む

- ❌ 「台本が書けない」だけ  
- ✅ 時間・場所・感情まで含めた **Lv.5 シチュエーション** まで落とす  

### 3. リサーチに時間をかけすぎる

**教材の目安：最大1日（2〜3時間）で一区切り**  

- 30分：入力＋初回出力確認  
- 60分：修正指示＋再出力  
- 60分：最終確認＋ペルソナ  

**80% で進み、Phase 4 で実データを足す**方が前に進みやすい。

### 4. 結果を保存しない

- `悩みリスト.md` / `悩みクラスター.md` / `ペルソナ設計書.md` 等にコピー保存  
- プロジェクトフォルダに固定  
- 以降の Phase では **このファイルを AI に添付**して文脈を共有

## AI プロンプト集（要約）

1. **クラスター商品化判定** — 12 クラスターを深さ・件数・商品化しやすさ・競合等でスコアし、優先順位を付ける。  
2. **ペルソナ深掘り** — 1日のスケジュール、見そうなチャンネル、過去の購入推定、「買わない理由」3つ、口癖、タイトル案10本。  
3. **リサーチ検証** — 網羅性、Lv.4-5 の比率、重複、追加悩み10、総合評価 A〜D。  

## FAQ

| Q | A（要約） |
|---|-----------|
| 本当に100〜150出る？ | AI と指示次第で**多く出せる**。ジャンルにより幅あり（教材の目安表あり）。 |
| 外れが多い？ | 修正指示で何度でも。**足りない束を20個追加**等。 |
| 一度きり？ | 基本構造は残しつつ、**月1や四半期**でコメント・反応を反映して更新推奨。 |
| ペルソナは複数？ | **最初は1人**で十分。増やすとブレやすい。 |
| 自分と悩みが違う？ | AI ベース＋**知恵袋・レビュー・コメント**等の生の声で補完。 |

## まとめ

- **属性ではなく悩み**  
- **10個ではなく 100〜150 を目安に**  
- **表面的ではなく Lv.4〜5 を厚く**  

**悩みデータベースは全 Phase で使う。深い悩みは後半の生成で出やすい、という整理も教材にある。**

**成果物（目安）**: 悩みリスト ＋ クラスター ＋ ペルソナ（所要 2〜3時間のイメージ）
