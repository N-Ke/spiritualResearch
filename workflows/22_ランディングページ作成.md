## このレポートで分かること

**LINE 登録用 LP** の役割、5パーツ構成、ツール選択、**AI 原稿プロンプト**、失敗パターン、チェックリスト（教材）。

## LP とは（教材）

動画等のリンクから**最初に着地する1枚**。**商品の長いセールスレター**ではなく、**LINE 登録までの短い導線**が主目的（本稿の型）。

## なぜ LP か（教材）

1. **意図のフィルタ** — 「なんとなく登録」と**本気度**の差を、コピーと構成で寄せる（数値効果は**計測**）。  
2. **教育の第0歩** — 悩みの言語化・期待で、**第1話**の受け取りを良くしうる。  
3. **媒体の役割分担** — 概要欄は短く、**詳細・オファー表現は LP** で整理（**YouTube 等の規約は公式確認**）。

---

## 5パーツ構成（教材）

| 順 | パーツ | 役割 |
|----|--------|------|
| 1 | **ヘッダー** | 誰向け・何がもらえる・無料の認知 |
| 2 | **フック** | 悩み・痛みの代弁 |
| → | **CTA 1** | 登録ボタン |
| 3 | **ベネフィット** | 登録後の**未来**（機能羅列だけにしない） |
| 4 | **特典紹介** | 中身・画像・具体数（**盛らない**） |
| → | **CTA 2** | 同上 |
| 5 | **フッター** | 最後の一押し |
| → | **CTA 3** | 同上 |

**CTA は最低3回**（上・中・下）。文言は**行動が分かる**もの（「今すぐ無料で受け取る」等）。

### 各パーツの要点（教材）

**ヘッダー** — ターゲットラベル、特典名、無料の明示。  
**フック** — 疑問形・あるある・断定のいずれかで**短く**（3〜5文目安）。  
**ベネフィット** — After の生活・感情（[`17`](17_8要素の詳細解説_前編.md) の B に相当）。  
**特典** — モックアップ・目次・ページ数等（[`21`](21_LINE登録特典の作り方.md) と整合）。  
**CTA** — 色は**背景と差**、スマホで**押しやすいサイズ**。

## D-BE_UNICK と LP の対応（**正本は `16`**）

| 要素 | LP での使い方（例） |
|------|---------------------|
| **D** Declaration | ヘッダー・「誰のためのページか」の線引き |
| **N** Nameless Enemy | フック（常識・構造が足を止めていた等）**個人攻撃はしない** |
| **B** Benefit | 理想の未来・箇条書き |
| **E** Evidence | 実績・数値を**短く**入れるなら**真実のみ**（任意） |
| **I** Innovation | 差別化の一言（任意・短く） |
| **C** Case Study | 一文の体験談（任意・捏造禁止） |
| **U** Urgency | **真実の**期限・定員のみ（[`17`](17_8要素の詳細解説_前編.md)） |
| **K** Kickstart | **CTA**・登録のしやすさ |

※「C＝CTA」「K＝キャラ」等の**別解釈は本ボルトでは採用しない**。

## 良い LP / 悪い LP（教材）

| 項目 | 悪い方向 | 良い方向 |
|------|----------|----------|
| 長さ | セールスレター級 | **スクロール少数回**で登録へ |
| 目的 | 商品を語りすぎ | **LINE 登録に集中** |
| フック | 長い自己紹介から | **読者の悩み**から |
| CTA | 下に1つ・曖昧文言 | **3回以上・明確動詞** |
| リンク | SNS 等で分散 | **登録以外の逃げ道を減らす**（ポリシー上必要な表記は残す） |
| 表示 | PC のみ確認 | **スマホ実機最優先** |

---

## つまずき5（教材）

1. **長文化** — LP は登録用。**ストーリー全文はステップへ**。  
2. **ボタンが沈む** — コントラスト・サイズ・文言。  
3. **他リンクだらけ** — 目的を**登録1本**に。  
4. **スマホ未確認** — 文字・画像・ボタンを実機で。  
5. **特典の実体が見えない** — 画像・具体項目・**嘘のボリュームは禁止**。

## ツール例（教材・仕様は公式確認）

| ツール | メモ |
|--------|------|
| ペライチ | ノーコード・テンプレ、素早い完成向け（**無料範囲は変動**） |
| Canva Web | ビジュアルと簡易ページ |
| Notion 公開 | 手軽だが「LP らしさ」は弱めになりうる |
| Studio 等 | 自由度大・学習コスト高 |

**WordPress 初手**は運用コストが重くなりやすい、という注意（教材）。

**選択の目安**: 最速ならノーコードテンプレ、デザイン一体なら Canva、完全無料重視なら Canva/Notion 等。

## AI で LP 原稿（教材）

戦略シート・キャラシート・特典概要を貼り、**1回で完璧を求めず**人間が削る。

```
【戦略設計シート・キャラ・特典の要約を貼る】

LINE登録用LPの原稿パーツを出力してください。

1. ヘッダーキャッチ（20字以内目安）
2. サブキャッチ（ターゲット＋特典＋無料）
3. フック（N寄り：痛み・共感）3〜5文、短行・改行多め
4. ベネフィット（B寄り）箇条書き3〜5
5. 特典の具体列挙（21と一致させる）
6. CTA文言 3パターン（命令ではなく明確な行動）

【禁止】読者責め、根拠なし「絶対・100%」、虚偽の価格・限定
【トーン】キャラ設計に合わせ、中学生でも読める語彙目安
```

### 調整チェック（教材）

- キャラと語彙  
- ターゲットの言葉  
- 誇大・断定の削除  
- 各ブロック**短く**  
- CTA の明確さ

## FAQ（要約）

| Q | A（教材） |
|---|-----------|
| スマホ vs PC | **スマホ優先**（流入の多くを想定） |
| 自己紹介 | 基本**不要**。**短い実績**なら数行までなら可（**検証可能**） |
| 特典未完成で公開？ | **原稿だけ**は可。**特典が無い状態で募集は不可** |
| 登録率目安 | **30〜50%**はあくまで**一例**。計測して改善 |
| デザイン | **テンプレ信頼**し、時間は文章に回す |

## まとめ（極意・教材）

1. LP は**濃い登録**と**第0歩の教育**を狙う箱  
2. **`16` の8要素**で部品を取る（略称混同に注意）  
3. **5パーツ＋CTA3回**  
4. **逃げ道を減らす**（法的表記は残す）  
5. **スマホファースト**
