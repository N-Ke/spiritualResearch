## このレポートで分かること

無料記事が「おまけ」ではなく、有料商品へ読者を運ぶための**教育コンテンツ（キラーコンテンツ）**になりうる理由、および **Why / What と How の切り分け**・**黄金構成（5ステップ）**・**AI 原稿のチェック**（教材）。

---

素晴らしい商品企画が完成した。**有料商品の執筆（教材では Step 3）に入る前に**、教材では **「Step 0.5：無料記事の作成」** を挟む、という流れ（本ボルトの Step 名は `02` 内ドキュメントで確認）。

「早く商品を作って売りたいのに、なぜ無料記事なの？」と思うかもしれない。次のような考え方は、**購買までの心理の階段**を見落としやすい（教材）。

- 「無料記事はおまけでしょ？」  
- 「時間がないから、いきなり有料を書きたい」  
- 「無料で出したら有料が売れなくなる」  
- 「集客は YouTube だけで十分」  

教材の主張として、**無料記事は「最後に売上を左右しうる鍵の一つ」** と位置づけられる（効果は個人差・導線・商品による）。

## なぜ「いきなり有料商品」だけに寄せにくいのか（教材）

無料記事を置く理由は、教材では次の **3つ** に整理される。

### 1. 読者を「次の一歩」に近づける教育ファネル

商品を置いて「買ってください」だけでは、**まだ階段の下にいる読者**には届きにくい。教材では次のような**心理的な階段**のイメージが使われる。

| 段階 | 読者の心理（例） | 主な担い手の例（教材） |
|------|------------------|------------------------|
| 1. 認知 | 「こんな動画がある」 | YouTube 視聴 |
| 2. 共感 | 「この人、分かってくれてる」 | 無料記事が担いうる |
| 3. 信頼 | 「原因が分かった。この人すごい」 | 無料記事が担いうる |
| 4. 欲求 | 「具体的にどうすればいいか知りたい」 | 無料記事が担いうる |
| 5. 購買 | 「この手順ならこの商品か」 | 有料商品 |

**対比の例（教材・数値はイメージであり保証ではない）**

- **無料記事なし**: 動画 → いきなり有料セールス → 読者は「急に売り込み」と感じやすい  
- **無料記事あり**: 動画 → 無料記事で整理 → 有料へ。教材では購入率の差を **0.1% 以下 vs 3〜10%** などと**例示**することがある（**再現を約束するものではない**）

### 2. LINE 登録の「特典（リードマグネット）」になりうる

人はメリットが見えにくいと動きにくい。**無料記事を特典として渡す**と、別途特典だけ作る手間を抑えつつ、「続きは有料で」と**導線を一本化**しやすい（教材）。詳細は [`21_LINE登録特典の作り方.md`](21_LINE登録特典の作り方.md)。

```
YouTube → 「詳しくは LINE」→ 登録 → 特典として無料記事 → 記事末尾で有料へ
```

### 3. 作者の頭の整理・軸の固定

**1,500〜3,000 字程度**の無料記事を先にまとめると、教材では次が得られやすいとされる。

- 全体メッセージが明確になる  
- いちばん伝えたい点が整理される  
- 本編執筆でブレにくい  
- 品質の土台ができる  

**映画の予告編の先に本編**、という比喩（教材）。

## 読者を惹きつける無料記事の「黄金構成」（5ステップ）

無料記事は「好きなように書く」だけでは、購買までの導線が弱くなりうる。教材では次の **5 ステップ** が推奨される。

| STEP | 役割 | 読者の心理（例） |
|------|------|------------------|
| **1. 共感と代弁** | 痛みを言語化 | 「まさに今の自分だ」 |
| **2. 原因の究明** | なぜ失敗しやすかったか | 「だからだったのか」 |
| **3. 新しい解決策（What）** | 方向性を示す | 「そんな方法があるのか」 |
| **4. 小さな成功体験** | 一部のチラ見せ | 「私にもできそう」 |
| **5. CTA** | 本編・有料への誘導 | 「続きを知りたい」 |

### ステップ別の記述例（教材）

| ステップ | 記述例の方向性 |
|----------|----------------|
| 1. 共感と代弁 | 「毎日〇〇しているのに、△△で心が折れそう、と感じていませんか？」 |
| 2. 原因の究明 | 「あなたのせいではなく、『××という前提』がズレていた、という側面がある」 |
| 3. 解決策（What） | 「解決の方向性は、『〇〇を先に整える』ことです」 |
| 4. 小さな成功体験 | 「例えば、△△を1つ変えるだけで…」（How の核心は出さない） |
| 5. CTA | 「具体的な N ステップは、本編（有料）でまとめています」 |

### 文字数配分の目安（教材）

合計 **約2,000字** を基準にした例。ジャンル・媒体で調整する。

| ステップ | 文字数目安 | 割合の例 |
|----------|------------|----------|
| 1. 共感と代弁 | 400字前後 | 20% |
| 2. 原因の究明 | 500字前後 | 25% |
| 3. 解決策（What） | 500字前後 | 25% |
| 4. 小さな成功体験 | 400字前後 | 20% |
| 5. CTA | 200字前後 | 10% |
| **合計** | **約2,000字** | — |

**全体の文字数**: 教材では **1,500〜3,000字** がよく言及される（短すぎると薄く、長すぎると読了が重い、という整理）。

## 「出しすぎ」と「出し惜しみすぎ」のバランス（Why / What / How）

| 層 | 内容 | 無料記事での扱い（教材） |
|----|------|--------------------------|
| **Why** | なぜそれが必要か | 丁寧に語ってよい。問題意識を上げる |
| **What** | 何をすべきか | 方向性は示す。**手順の細部までは出しすぎない** |
| **How** | どうやるか | **有料の核**（手順・テンプレ・プロンプト全文等）は出しすぎない |

**出しすぎの例（教材）**: プロンプト全文と全手順を無料で載せる → 「もう十分」と感じられ、有料の価値が相対的に下がりうる。

**バランスの例（教材）**: 「コツは〇〇です。具体的なプロンプトと手順は本編で」と**焦らす**。

## 「チラ見せ」の4つのテクニック（教材）

1. **一部だけ公開** — 「3ステップのうち1つ目だけ…」と残りを気にさせる  
2. **モザイク的に隠す** — 構成は見せるが、中身のキーワードは伏せる  
3. **結果だけ見せる** — ビフォーアフターや出力例で「どうやったか」を知りたくさせる  
4. **数字で具体化** — 「作業時間が約1/10になった」等（**事実と一致**させ、誇大にしない）  

## 【実践】AI 原稿の「5つの罠」（ディレクター視点）

AI に任せっきりにせず、次をチェックする（教材）。

1. **How を喋りすぎ** — 手順・プロンプトまで無料に出ていないか。**Why / What に寄せ、How は本編へ** と指示し直す。  
2. **キャラの熱量がない** — 行儀だけの文章になっていないか。[`05_キャラクター設計.md`](05_キャラクター設計.md) の**失敗談・口調**を共感パートに足す、等。  
3. **共感が弱い** — いきなり解決策から入っていないか。  
4. **CTA が弱い** — 「詳しくは有料」一行で終わっていないか。**最後の一押し**を設計する。  
5. **長すぎる** — 5,000字超などで読了が重く、次の行動までのエネルギーが残りにくい。  

### AI 原稿チェックリスト（教材）

- [ ] 1,500〜3,000字に収まっているか  
- [ ] 黄金構成（5ステップ）の順で読めるか  
- [ ] How（具体手順）を出しすぎていないか  
- [ ] 共感で痛みを言語化できているか  
- [ ] キャラの個性・熱量が出ているか  
- [ ] CTA は十分か  
- [ ] 有料への導線が自然か  
- [ ] 「続きを知りたい」で終われるか  

note 投稿前の体裁・規約観点は、必要に応じて [`../../02オンラインテキスト商品化/ナレッジ/note記事チェックリスト.md`](../../02オンラインテキスト商品化/ナレッジ/note記事チェックリスト.md) も参照。

## 無料記事の公開場所（教材の整理）

| プラットフォーム | 難易度の印象 | SEO・発見 | 備考（教材） |
|------------------|--------------|-----------|----------------|
| note | 低め | 検索流入を狙いやすい**ことがある** | 初心者向けに推されやすい |
| Brain | 低め | プラットフォーム内中心になりうる | 販売機能との連携イメージ |
| 自社サイト | 高め | 初期は弱くなりうる | 運用・技術コスト |
| Google ドキュメント等 | 低め | なしに近い | 特典配布・限定共有向け |

**規約・表現の注意（教材・一般論）**

- **note**: アフィリエイトリンクの扱い等は**公式ガイドライン**で確認。「必ず稼げる」等の**断定的収益表現**は規約・景表法の両面でリスク。  
- **全プラットフォーム共通**: **景品表示法**（誇大・優良誤認・有利誤認）／ **薬機法**（健康・効能の表現）／ **著作権**／ **特定商取引法**（販売表示）など、該当するものは**最新法令・各公式**を確認。  

詳細は [`../../02オンラインテキスト商品化/ナレッジ/プラットフォーム仕様.md`](../../02オンラインテキスト商品化/ナレッジ/プラットフォーム仕様.md)。

## よくあるつまずき（教材）

1. **完璧主義で未公開** — 60点でも公開し、反応を見て改善する、という姿勢の例。  
2. **無料記事を「本編」にしすぎる** — 前菜と本編の役割分担。**「もっと知りたい」**で切る。  
3. **YouTube・無料記事・有料のテーマがバラバラ** — 一本の線で統一する。  
4. **CTA 忘れ** — 記事末は必須。可能なら中盤にも導線を置く、と教材で言われることがある。  

## FAQ

| Q | A（教材） |
|---|-----------|
| 文字数のベストは？ | **1,500〜3,000字** がよく推奨される。短すぎると薄く、長すぎると読了負荷。 |
| 複数本作る？ | **最初は1本**に集中。成果と学びが出たら切り口を変えた2本目を検討。 |
| 有料と被る？ | **Why/What** を無料、**How** を有料、で切り分ける。 |
| YouTube 概要欄に貼る？ | **積極的に**貼る、と教材で言われることが多い。動画内誘導とセット。 |
| AI は使っていい？ | **使ってよい**が、上記 **5つの罠** を必ずチェックし、人間がディレクションする。 |

## まとめ

1. 無料記事は「おまけ」に限定されない。**教育・特典・執筆前の設計**に使える（教材）。  
2. **黄金構成** — 共感 → 原因 → What → 小さな成功 → CTA。  
3. **How は有料の核** — 焦らしと価値の両立。  
4. **1,500〜3,000字** を目安に、読了負荷を意識。  
5. **AI は下書き** — 罠をチェックし、修正を重ねる。  
6. **完璧より公開** — 改善は後からでよい、という整理（教材）。
