## D — Declaration（違いの宣言）

### Why（教材）

読者は過去の体験から**防衛フィルター**（「また同じパターン」）をかけやすい。**D** はそのフィルターを早い段階で緩め、「話を聞く態度」を作る、という整理。

### How

**業界の常識**と**自分のスタンス**の線引き。Phase 1 の**キャラ・独自性**を前面に。**批判だけで終わらず**、自分の立場と**代案**までセット。

| 項目 | 弱い D | 強い D（方向性） |
|------|--------|------------------|
| 表現 | 「初心者に優しく教えます」 | 常識に NO を突きつけ、**自分の教え方**をはっきり言う |
| 反応 | 他と同じに見える | コントラストで記憶に残りやすい |
| リスク | 差別化ゼロ | 攻撃的すぎると信頼損失（倫理原則） |

### 配信例（教材・サンプルキャラ）

**件名例**: 「僕はアドセンスを否定しません。でも──」

**骨子**: アドセンス否定しない意外性 → 長年の経験 → **非属人のまま**別柱の実験 → **具体数字1つ**（例：短期のリスト件数）で興味。

※数字・実名は**架空可**。実運用は**検証済みの事実のみ**。

### つまずき（教材）

1. **スタンスがない** — 「色々お伝えします」だけ  
2. **批判で終わる** — 代案のない悪口  
3. **誰でも言えること** — 差別化にならない正論だけ  
4. **煽りすぎ** — 倫理・信頼を損なう攻撃性  

### 迷ったとき（教材）

- 業界の「常識」で**自分が違和感**を持っていることは？  
- 過去の自分が**言ってほしかった**ことは？  
- 競合が**言いにくいが、あなたは言える**真実は？（**個人・企業の名指し**は避ける）

```
・ペルソナ：{ }

・本音の違和感：{ }

【あなたの立場】
・スタンス：{ }
・実績・経験（事実のみ）：{ }

【条件】
・冒頭で掴める分量（目安 200〜400字）

・検証可能な具体（数字等）を1つ以上
・締め「だから私は○○を伝える」型の宣言

【出力】件名案3つ、本文
```

## B — Benefit（究極の恩恵）

### Why（教材）

読者が欲しいのは**商品そのもの**より**結果**、さらには**結果の先の生活・自己像**（変容）に近い、という整理。

### 3層（教材）

| 層 | 内容（例の方向性） |
|----|---------------------|
| 第1層 機能的 | 数字・時間・成果の事実 |
| 第2層 体験的 | 日常の具体的な変化 |
| 第3層 変容的 | アイデンティティ・在り方の変化（**最も感情に触れやすい**） |

### 弱い B vs 強い B（教材）

| 弱い B | 強い B |
|--------|--------|
| 機能説明だけ（台本10分） | 地獄からの解放、家族時間、通知のイメージなど**場面**が見える |

### 配信例の骨子（教材）

**件名例**: 「アドセンスの"横"にもうひとつ柱があると、夜が変わる」

**展開**: まず感情（眠れる等）→ **箇条書きで3つの変化**（コントロール感・作業の楽しさ・資産の捉え方）→ 本人の**体感一文**で締め。

### つまずき（教材）

1. 機能で止まる → **だから人生でどう変わるか**まで  
2. 「人生が変わる」だけ → **五感・場面**まで落とす  
3. 自分語り偏り → **読者の視点**「あなたは？」を挟む  
4. 非現実な約束 → **景表法・信頼**を壊す  

### AI プロンプト（B）

```
「B（恩恵）」を作成してください。

【ターゲット】
・ペルソナ：{ }
・最大の悩み：{ }
・憧れの生活：{ }

【提供価値】
・機能：{ }
・直接的メリット：{ }

【条件】
・機能的・体験的・変容の3層
・五感に触れる描写
・「想像してみてください」等の問い1つ
・非現実な約束は禁止

【出力】件名案3つ、本文 400〜600字
```

## E — Evidence（システムの証拠）

### Why（教材）

D・B で熱が出たあと、「本当に？自分にも？」という**疑念**。**E** は論理・データ・事例で**納得**に寄せる。

### 核心（教材）

**才能アピール**より**システム・仕組みの再現性**。「あの人だから」で処理されないようにする。

### 弱い E vs 強い E（教材）

| 弱い E | 強い E |
|--------|--------|
| 自分が頑張ったから | **仕組み**が属人性を減らす、という説明＋根拠 |

### 5つの証拠カテゴリ（教材）

1. **実績データ**（**捏造禁止**・**出典または母数明記**）  
2. **具体的事例**（ストーリー・**事実確認**）  
3. **第三者の声**（**許諾・表現の適法**）  
4. **権威**（**論文・公式・発言の確認**）  
5. **自己体験**（日付・数値・文脈を**誠実**に）  

※「327名中87%」等は**教材ダミー**。

### 配信例の骨子（教材）

**件名例**: 「概要欄の4文字を変えたら、クリック率が1.8倍になった話」

**展開**: 条件（非属人等）→ **やったこと1つ** → **なぜそう動いたか（構造）** → もう1数字（開封率等。**実測のみ掲載**）。

### つまずき（教材）

1. 「多くの人が」→ **母数・条件**  
2. 才能自慢 → **仕組み語り**へ  
3. 証拠が単一 → **複数カテゴリ**で組み合わせ  
4. **嘘のデータ** → 最悪ケース  

### AI プロンプト（E）

```
「E（証拠）」を作成してください。

【実績・データ】※すべて事実ベースで入力
・具体数字：{ }
・比較：{ }
・事例：{ }

【システム】
・再現性の理由：{ }
・従来との違い：{ }

【条件】
・5カテゴリのうち少なくとも3
・「多くの人」禁止、母数具体的
・才能ではなくシステム
・嘘データ禁止

【出力】件名案3つ、本文 400〜600字
```

## U — Urgency（必然性）

### Why（教材）

「いつかやる」は先延ばしに落ちやすい。**今動く理由**を、倫理を守りつつ渡す。

### D-BE_UNICK 流の U（教材）

| 使わない例 | 使う方向性 |
|------------|------------|
| 嘘の残り枠・根拠なき50%オフ | **内的**・**状況的**・**構造的**な必然 |

- **内的**: 登録・開封している事実＝変化への関心、等（**押し付けない**）  
- **状況的**: 市場・技術の変化（**断定は公式・データと照合**）  
- **構造的**: 少人数・サポート上限など**本当の制約**  

### 弱い U vs 強い U（教材）

| 弱い U | 強い U |
|--------|--------|
| いつでも安そうな値引き | **構造・機会**の説明＋読者の**選択**に委ねる |

### 配信例の骨子（教材）

**件名例**: 「あなたが動けない理由は、あなたのせいじゃない」

**展開**: 意志の弱さを否定 → **染みついた前提**の可視化 → 外部変化は**脅しではなく構造**と明示 → 「今持つ意味」→ **同条件が将来も保証されない**は**一般論としての注意**（断定バブル等は避け、検証した事実だけ）。

### つまずき（教材）

1. 嘘の限定  
2. 恐怖一辺倒（反発）  
3. 理由のない値引き  
4. D・B・E が弱いまま U だけ強い  

### AI プロンプト（U）

```
「U（必然性）」を作成してください。

【状況】
・市場・技術の変化（事実ベース）：{ }
・タイミング：{ }

【機会損失・維持コスト】
・先延ばしで失うもの：{ }
・現状のコスト：{ }

【条件】
・煽りではなく気づき
・嘘の限定禁止
・読者を責めない
・論理で外部変化を説明
・最後は選択を委ねる

【出力】件名案3つ、本文 400〜600字
```

## D → B → E → U の配信の流れ（教材）

| 通目（例） | 役割 |
|------------|------|
| 1 | **D** — 防衛フィルターに挑む宣言 |
| 2 | **B** — 五感・変容までの未来 |
| 3 | **E** — 疑念への証拠・再現性 |
| 4 | **U** — 今を選ぶ理由（気づき・構造） |

**組み合わせ例**: 1通に **D+B**、次に **E+U** など（`16` の通し設計と整合）。
