# Block Ledger Template — スピ系自チャンネル

> 成果物単位（YouTube動画 1本／note有料記事 1本／LINE特典／横断資料）の進行台帳。
> 新規 Block は本ファイルをコピーして `blocks/works/{ID}.md`（作品単位）または `blocks/assets/{名前}.md`（横断資料）に置く。
> `/handoff-update {block}` で 8 セクションの更新を促す。

## プレースホルダの使い分け（厳格）

> **本ルールのスコープは Block Ledger 内（`memory/blocks/` 配下）のみ**。
> AGENTS.md・decision_log.md・open_questions.md など他のファイルでの「未確定」運用語はそのまま維持してよい。

Block Ledger の埋まっていない欄には次の 2 種類だけを使う。**Block Ledger 内では「未確定」「未定」など他の表現は使わない**。

- **`<TBD: 理由>`** — **着手前**（これから決める）。例：`<TBD: 構成書 v1 着手時に決定>`
  - `grep "<TBD" memory/blocks/` で着手前 Block の一覧化が可能
- **`N/A: 理由`** — **構造的に該当しない**（決まらなくて正しい）。例：`N/A: 単発完結のため伏線なし`
  - 理由なしの `N/A` 単独は禁止（形式主義化を防ぐため）
