# D-BE UNICK ステップメールシステム v2 実行手順

## 概要
D-BE UNICK フレームワーク（8要素：D-B-E-U-N-I-C-K）に基づき、
高品質なステップメールシリーズを自動生成するシステム。

v2では、以下を大幅に強化：
- **8要素×個別ナレッジファイル** — 各要素200-350行超の完全ガイド（テンプレート・フレーズ・チェックリスト付き）
- マーケティング戦略設計（キャラクター設計・ビジネスカルテ）
- コピーライティング強化ツール（件名テンプレート・感覚言語・コントラスト表現）
- メールフォーマット・読みやすさの具体的ガイドライン
- 先行予約・リアルタイム感などのテクニック集

## 前提条件
- `入力/` フォルダにクライアント情報（商品概要、ターゲット、USPなど）が格納されていること
- `ナレッジ/` フォルダの全ファイルを参照可能であること

## Phase 1：戦略設計

**目的**: ステップメール全体の戦略を設計する

**参照ナレッジ**:
- `D-BE_UNICK_統合ナレッジ.md` → 8要素の概要・索引・変換マップ
- `D-BE_UNICK_戦略設計シートテンプレート.md` → 戦略設計シート
- **8要素個別ファイル** → 各要素の詳細テンプレートとフレーズ集
  - `D_違いの宣言.md` — D要素の全知見（感覚言語・コントラスト・Yes階段・言語パターン・社会的証明）
  - `B_恩恵.md` — B要素の全知見（3層ベネフィット・五感描写・質問フレーム・60フレーズ）
  - `E_裏づけ.md` — E要素の全知見（5証拠カテゴリ・コントラスト・投資認識転換・社会的証明）
  - `U_必然性.md` — U要素の全知見（内的必然性・損失回避・見えないコスト・FOMO）
  - `N_名もなき敵.md` — N要素の全知見（敵命名・5階層モデル・勘違い暴露・解放型パターン）
  - `I_新しい勝ち方.md` — I要素の全知見（選択肢消去法・再現性証明・30フレーズ）
  - `C_体感事例.md` — C要素の全知見（8構造テンプレート・10フレーム・具体化5要素）
  - `K_最初の一歩.md` — K要素の全知見（CTA設計・5障壁突破法・マイクロステップ・60フレーズ）

**実行内容**:
1. `入力/` のクライアント情報を読み込む
2. 戦略設計シートの全セクションを埋める
   - セクション1：基本情報
   - セクション2：マーケティング戦略設計
     - キャラクター設計（5大要素スコアリング）
     - ビジネスカルテ（価値の3階層・悩みの立体把握）
     - キャラクターとビジネスの統合
   - セクション3：D-BE UNICK 8要素カードマッピング（**各要素ファイルを参照して深い内容を設計**）
   - セクション4：シーケンス設計（タイプ選択・各通の概要）
   - セクション5：トーン&マナー設定
3. 完成した戦略設計シートを `実行結果/` に出力

**出力**: `実行結果/01_戦略設計シート.md`

## Phase 2：配列設計

**目的**: 各通のD-BE UNICK要素配置と送信タイミングを設計する

**参照ナレッジ**:
- `D-BE_UNICK_ステップメール設計図.md` → シーケンスタイプ別の詳細設計
- Phase 1で作成した戦略設計シート
- `K_最初の一歩.md` → CTA段階設計・強度マップ

**実行内容**:
1. 選択されたシーケンスタイプ（A or B）に基づき、各通を設計
2. 各通に以下を割り当てる：
   - 送信タイミング（登録後〇時間/〇日目の〇時）
   - 件名案（3つ以上）→ コピーライティング強化ツールの件名テンプレートを活用
   - 主要D-BE UNICK要素（2-3個/通）
   - メールの目的（1文で）
   - 使用テクニック（サンドイッチ構造、選別姿勢、リアルタイム感、等）
   - 文字数目安
3. 全体の要素分布バランスをチェック
4. 通間の流れの自然さを確認

**出力**: `実行結果/02_配列設計表.md`

## Phase 3：本文生成

**目的**: 各通のメール本文を生成する

**参照ナレッジ**:
- **8要素個別ファイル** → 各要素のテンプレート・フレーズ集・チェックリスト（**最重要**）
- `D-BE_UNICK_コピーライティング強化ツール.md` → ワード集・テンプレート・チェックリスト
- `D-BE_UNICK_ステップメール設計図.md` → 1通の基本構造・各通の構成
- Phase 1-2の出力

**生成ルール**:

### 必須ルール
1. 各通にD-BE UNICK 8要素を最低2つ織り込む
2. 読者を責める・恥ずかしめる表現を絶対に使わない
3. CTAは「招待型」「自己決定型」「仲間型」のいずれかにする（→ `K_最初の一歩.md` 参照）
4. サンドイッチ構造を使用する（具体→抽象→具体→行動）
5. 抽象的主張には必ず具体的な数値・事例をつける

### 要素別参照（生成時に必ず該当ファイルを参照）
- D要素を使う通 → `D_違いの宣言.md` のテンプレート・40表現パターンから選択
- B要素を使う通 → `B_恩恵.md` の3層構造・60フレーズから選択
- E要素を使う通 → `E_裏づけ.md` の5証拠カテゴリ・コントラスト・投資フレームから選択
- U要素を使う通 → `U_必然性.md` の3アプローチ・見えないコスト計算から選択
- N要素を使う通 → `N_名もなき敵.md` の3カテゴリ・解放型パターンから選択
- I要素を使う通 → `I_新しい勝ち方.md` の消去法・再現性証明から選択
- C要素を使う通 → `C_体感事例.md` の8構造テンプレート・具体化5要素から選択
- K要素を使う通 → `K_最初の一歩.md` の段階設計・障壁突破法から選択

### 文体ルール
- 基本：「です・ます」調
- 強調部分：「だ・である」調で断定的に
- 曖昧表現（「〜と思います」「〜かもしれません」）は禁止
- 「皆さん」ではなく「あなた」に語りかける
- 機能説明ではなく感情・ストーリー・価値を伝える

### フォーマットルール
- 「。」で1行空ける
- 話題転換で2行空ける
- 1行20文字前後以内
- 1文は最大4行まで
- 重要キーワードは独立行にする
- 接続詞は独立行に置くと効果的

### 件名ルール
- 件名案を3つ以上提示する
- コピーライティング強化ツールの8カテゴリから選択
- 開封欲を刺激する4タイプ（衝撃型・具体数字型・緊急性型・好奇心型）

### 感覚言語ルール
- 各通で五感のうち2つ以上の感覚言語を使用
- コピーライティング強化ツールの感覚言語パレットを参照
- `D_違いの宣言.md` 第3章の五感別表現集も活用

### CTA設計ルール
- 1通1CTA原則
- 序盤：情報受取系CTA
- 中盤：小さな行動系CTA
- 終盤：決断促進系CTA
- → `K_最初の一歩.md` のCTA強度マップに従う

**出力**: `実行結果/03_メール本文/` 配下に通番ごとのファイル

## Phase 4：品質チェック

**目的**: 生成されたメール全体の品質を検証し、改善する

**参照ナレッジ**:
- **8要素個別ファイルの各チェックリスト**
- `D-BE_UNICK_コピーライティング強化ツール.md` → ライティングチェックリスト
- `D-BE_UNICK_ステップメール設計図.md` → 品質チェックリスト

**チェック項目**:

### D-BE UNICK要素チェック
- [ ] 各通にD-BE UNICK 8要素が2つ以上含まれているか
- [ ] 全体を通じて8要素が偏りなく分布しているか
- [ ] 1通目にD要素（違いの宣言）が含まれているか
- [ ] 各要素が対応するファイルのチェックリストを通過しているか

### 倫理原則チェック
- [ ] 読者を責める/恥ずかしめる表現はないか
- [ ] 敵は外部要因として定義されているか
- [ ] 嘘の限定性を使っていないか
- [ ] 根拠のない煽りを使っていないか

### コピーライティング品質チェック
- [ ] CTAは「招待型」「自己決定型」になっているか
- [ ] サンドイッチ構造が適切に使われているか
- [ ] 具体的な事例・数値・証拠が十分か
- [ ] 件名は開封欲を刺激するものになっているか
- [ ] 五感のうち2つ以上の感覚言語が使われているか

### フォーマットチェック
- [ ] 行間ルール（「。」で1行空け等）が守られているか
- [ ] 1行20文字以内が守られているか
- [ ] リズムの変化（デコボコパターン）があるか
- [ ] スマホで読んだ時のレイアウトは適切か

### 全体フローチェック
- [ ] 通と通の間の流れは自然か
- [ ] リアルタイム感は演出されているか
- [ ] 全体を通じて「仲間感」のトーンが維持されているか
- [ ] K要素のCTAが段階的に強度を上げているか

**出力**: `実行結果/04_品質チェック結果.md`
