# D-BE UNICK 配列設計 ── ウラアド・ラボ

---

## 選択した型：分岐後即セールス型

### 選択理由

ウラアド・ラボのターゲットは「アドセンスで月5〜50万稼いでいるYouTuber」。
すでにある程度の実績と行動力がある層であり、熱量が高い見込み客。
長期間の信頼構築よりも、短期間で「違い」を示し決断へ導く構成が適切。

## フェーズ構造

### Phase 1：比較停止（Day 0〜2）── D / B / E

最初の3要素で「この人は違う」と思わせる。
問題提起は一切しない。違いの宣言→恩恵→裏づけの順。

### Phase 2：意味づけ（Day 4〜6）── U / N / I

「なぜ今か」「なぜ動けないのか」「どう勝つのか」を構造化。
ここで初めてN（名もなき敵）として問題を扱う。

### Phase 3：決断前進（Day 7〜9）── C / K

体感事例で腹落ちさせ、セールスページ公開。
K（最初の一歩）を繰り返し提示して行動へ。

### Phase 4：フォロー（Day 10）

募集終了報告＋今後の関係性維持。

## 既存構成との差分

| 項目 | 旧（教育型） | 新（D-BE UNICK型） |
|------|------------|-------------------|
| 第0通の起点 | 自己紹介＋歓迎 | **Dから入る**（何者かの宣言） |
| 恩恵の描き方 | 実績紹介の中に混在 | **Bとして独立通**（生活変化で描写） |
| 裏づけの出し方 | ストーリーの中で自然に | **Eとして独立通**（数字と構造で明示） |
| 問題提起の位置 | 3通目（4日後）で現状認識 | **4通目以降**（D/B/Eの後にNとして構造化） |
| 投資マインド | 3通目で「投資の教育」として直接言及 | U（必然性）として市場文脈で語る |
| 読者の問題 | 「アドセンス依存のリスク」として読者の問題 | N（名もなき敵）として構造の問題に再定義 |

## 維持する要素

- ウラアドの温かく知的なトーン
- 「実験ノート」としてのリアルタイム感
- 動画3本シリーズの段階的配信
- 10日間13通の全体構造
- 3日間限定募集の枠組み
