# D-BE UNICK ステップメール設計図【完全版】

## 設計の基本思想

D-BE UNICKのステップメールは、従来の「教育→煽り→販売」ではなく、
**「違いの宣言→共鳴の構築→仲間としての招待」** というアーキテクチャで構成する。

---

## シーケンスタイプA：分岐後セールス型（10日間・10-13通）

メインメルマガからの分岐者＝すでに高い関心を持つ読者に対して、
短期間で信頼を構築し、自然な形で決断を促す設計。

### フェーズ構成

```
【Phase 1】共鳴の着火（Day 1-2 / 1-4通目）
 目的：強い第一印象 + 「この人は違う」の確立
 主要要素：D → B → E
 心理効果：興味の点火 + 選別感による価値向上

【Phase 2】共感と信頼の構築（Day 3-5 / 5-8通目）
 目的：人間性の共有 + 社会的証明 + 核心的価値の伝達
 主要要素：E → C → N → I
 心理効果：「この人の言うことは信頼できる」の確立

【Phase 3】決断の支援（Day 5-7 / 9-11通目）
 目的：最終的な決断の後押し + 残りの不安解消
 主要要素：U → I → K
 心理効果：「今、行動する必然性」の醸成

【Phase 4】再招待（Day 8-10 / 12-13通目）※オプション
 目的：決断できなかった人への再アプローチ
 主要要素：C → B → K
 心理効果：後悔感の活用 + 最後の招待
```

### 各通の詳細設計

#### 【1通目】登録直後：違いの宣言（D + K）
```
件名例：「【重要】あなたが今まで聞いたことのない話をします」
タイミング：登録直後
文字数目安：800-1200字
構成：
 ① 挨拶 + 登録への感謝（3行）
 ② 違いの宣言（D要素）：自分のスタンスを明確に（10行）
 ③ これから届く内容の概要（3行）
 ④ 第一のCTA：LPまたは動画への誘導（K要素）
 ⑤ 限定性の提示（構造的理由があるもの）

ポイント：
 - シンプルかつ力強く。余計な説明は入れない
 - 「招待状」のような特別感を演出
 - 件名で「他との違い」を暗示する
```

#### 【2通目】登録5分-1時間後：再送・補足（D + B）
```
件名例：「※補足※ 先ほどのメールで伝え忘れたことがあります」
タイミング：1通目の5分〜1時間後
文字数目安：600-800字
構成：
 ① 1通目の再送理由（自然な形で）
 ② あなたが得られるもの（B要素）の具体化
 ③ CTA再掲
 
ポイント：
 - 見落とし防止 + 重要性の倍増
 - 内容は1通目とほぼ同一でOK
```

#### 【3通目】Day1夜 or Day2朝：選別と価値観（D + E）
```
件名例：「お申し込みの中に、残念な方がいらっしゃいます」
タイミング：登録から12-24時間後
文字数目安：1500-2500字
構成：
 ① 申込み状況の報告（社会的証明）
 ② 選別姿勢の明確化（D要素の強化版）
   - 「本質的な変化を求める方」を歓迎する姿勢
   - 短絡的な期待との距離感
 ③ 自分が求める"仲間"の条件
 ④ 「99%の方は問題なく通過」の安心材料（E要素）
 ⑤ CTA

ポイント：
 - 「選ばれたい」と思わせる心理的転換
 - 読者を責めるのではなく、価値基準を示す
 - D-BE UNICK倫理原則：「ゴキブリは排除」等の攻撃的表現は使わない
```

#### 【4通目】Day2朝：人間性とストーリー（C + B + N）
```
件名例：「〇〇だった頃の話を、初めてします」
タイミング：登録から24-36時間後
文字数目安：2000-3500字
構成：
 ① 自己開示ストーリー（C要素）
   - どん底 → きっかけ → 発見 → 行動 → 現在
   - 五感の描写 + 具体的な日時・場所
 ② サンドイッチ構造で価値観教育（B要素の深化版）
   - 「知識より行動」「出発点より方向性」等
 ③ 読者の共通の壁を名指し（N要素）
   - 壁の正体 = 外部要因として定義
 ④ CTA

ポイント：
 - このメールが最も「仲間感」を生む通
 - ストーリーは10のフレームから適切なものを選択
 - サンドイッチ構造を必ず使う
```

#### 【5通目】Day2昼-夕：核心的価値の提示（I + E）
```
件名例：「最小限の労力で最大の成果を生む"仕組み"の全貌」
タイミング：登録から30-42時間後
文字数目安：2000-3000字
構成：
 ① 核心的な哲学/原則の提示（I要素）
 ② なぜこの方法が機能するのか（3つのポイント）
 ③ 具体的な成果事例 x 3名（E要素）
   - 名前（仮名可）、年齢、職業、具体的数値
 ④ 得られるベネフィット一覧（B要素）
 ⑤ CTA

ポイント：
 - 「事例 + 数値」で信頼を固める通
 - メタファーを使って理解しやすく
```

#### 【6通目】Day3朝：決断力への挑戦（U + K）
```
件名例：「このままでは終わらせません」
タイミング：登録から48-60時間後
文字数目安：2000-3000字
構成：
 ① 「言葉だけの人」と「行動する人」の対比
 ② 自分自身のリアルなエピソード（体調不良でも書いている等）
 ③ 「お客様ではなくパートナーを求めている」（D要素の再確認）
 ④ 行動する/しないの分岐点を提示（U要素）
 ⑤ 力強いCTA（K要素）

ポイント：
 - 決断を迫るが、押し付けではない
 - あくまで「選択の尊重」を前提に
```

#### 【7通目】Day3昼-夕：参加者の声と共感（C + E）
```
件名例：「"手が震えるほど緊張しました" ある参加者の告白」
タイミング：登録から60-72時間後
文字数目安：2500-3500字
構成：
 ① 参加者の声（実例メッセージの引用）
 ② その声への共感と自分の類似体験
 ③ 不安解消：審査/面談/購入プロセスの安心材料
 ④ CTA

ポイント：
 - 最も「自分もやってみたい」を生む通
 - 参加者の声は感情的な共感を生むものを選ぶ
```

#### 【8通目】Day4：具体的価値と社会的証明（I + E + B）
```
件名例：「なぜこの仕組みは"持続的に"結果を出し続けるのか」
タイミング：登録から72-84時間後
文字数目安：2000-3000字
構成：
 ① 商品/サービスの3つの核心的価値
 ② 具体的成果事例 x 3名（数値付き）
 ③ 残り日数のカウントダウン
 ④ FAQ（よくある不安への回答 x 3-5個）
 ⑤ CTA
```

#### 【9-10通目】Day4-5：締め切りとラストチャンス（U + K + B）
```
件名例：「【残り24時間】あなたへの最後のメッセージ」
タイミング：締切24時間前 + 締切当日
文字数目安：1500-2500字
構成：
 ① これまでの要約（D-BE-U-N-I-C-Kの核心を凝縮）
 ② 行動した場合の未来像（B要素）
 ③ 行動しなかった場合の穏やかな対比（煽りすぎない）
 ④ 最後のCTA（仲間型）

ポイント：
 - 最後まで「仲間」のトーンを崩さない
 - 「買わない人は損」ではなく「一緒に始められたら嬉しい」
```

#### 【11-13通目】Day7-10：再招待フェーズ（オプション）
```
件名例：「先日の件について、もう一度だけお伝えさせてください」
タイミング：締切後2-5日
構成：
 ① 締切後の参加者の声（社会的証明）
 ② 「参加できなかった方への特別再招待」
 ③ 期間限定の再募集（3日間）
 ④ 最終CTA
```

## シーケンスタイプB：高額商品販売型（10日間・23通）

無料コンテンツから段階的に信頼を構築し、高額商品への心理的抵抗を最小化する設計。

### フェーズ構成

```
【Phase 1】価値提供と信頼構築（Day 1-3 / 1-6通目）
 目的：無料価値の提供 + 複数メディアへの誘導 + 初期信頼
 主要要素：D → E → B
 テクニック：プレゼント→動画→YouTube連携

【Phase 2】教育と動機付け（Day 4-7 / 7-14通目）
 目的：4話構成動画シリーズで深い教育 + N要素の浸透
 主要要素：N → I → E → C
 テクニック：動画+メール補足+ワーク提出の三層構造

【Phase 3】セールスと決断促進（Day 8-10 / 15-23通目）
 目的：最終決断への導線 + 残りの不安解消 + 限定性
 主要要素：U → K → B → E
 テクニック：カウントダウン+FAQ+参加者の声+最終CTA
```

### 高額商品型の追加テクニック

```
1. 価値積み上げ構造
   無料レポート → 動画シリーズ → ウェビナー → 個別相談 → 本商品
   各段階で価値を提供し、次への期待を高める

2. マルチメディア連携
   メール + YouTube + LINE + 無料PDF
   異なる感覚（視覚・聴覚・読解）への多角的アプローチ

3. 動画シリーズとの連動
   動画配信メール → 補足メール → 次の動画予告メール
   各動画に「行動ワーク」を設定し、能動的参加を促進

4. PS/PPS活用術
   PS：次回予告で期待感醸成
   PPS：YouTube等の他メディアへの誘導
```

## 先行予約テクニック（リアルタイム感の演出）

ステップメールの「自動配信感」を消すための技術。

```
1. 先行予約パターン
   「〇月〇日（〇曜日）から、特別な企画を開始します」
   → 未来日付を設定し、まるでリアルタイムイベントのように演出
   → 読者は「自分だけのために」送られている感覚を持つ

2. リアルタイム感パターン
   「今朝、一つ嬉しいメッセージが届きました」
   「現在、予想を超えるペースでお申込みをいただいており…」
   → 過去形・進行形を使い、ライブ感を出す

3. 反応型パターン
   「昨日のメールに対して、こんなご質問をいただきました」
   「多くの方から"もっと詳しく知りたい"というお声が…」
   → 読者の反応に応えている形にする

注意：
 - ステップメールでありながら、まるでリアルタイムのやり取りのように読める文体を使う
 - 具体的な日時の指定は、相対的な表現（「先日」「今朝」）を使う
 - 嘘はつかない。一般的に起こりうる状況描写に留める
```

## 品質チェックリスト（Phase 4で使用）

```
□ 各通にD-BE UNICK 8要素が2つ以上含まれているか
□ 全体を通じて8要素が偏りなく分布しているか
□ 1通目にD要素（違いの宣言）が含まれているか
□ 読者を責める/恥ずかしめる表現はないか
□ 敵は外部要因として定義されているか（読者自身は敵でない）
□ CTAは「招待型」「自己決定型」になっているか
□ 件名は開封欲を刺激するものになっているか（4タイプ）
□ サンドイッチ構造が適切に使われているか
□ 具体的な事例・数値・証拠が十分か
□ 通と通の間の流れは自然か
□ リアルタイム感は演出されているか
□ メールフォーマット（行間・文字数・リズム）は適切か
□ 全体を通じて「仲間感」のトーンが維持されているか
□ スマホで読んだ時のレイアウトは適切か
```
