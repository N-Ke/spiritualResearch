## 1. このシステムでできること

このシステムを使うと、**商品・サービスの情報を入力するだけ** で、
プロ品質のステップメールシリーズが自動生成されます。

### 活用例

| 用途 | 説明 |
|------|------|
| 新規メルマガの立ち上げ | ゼロから戦略〜本文まで一括生成 |
| 既存メルマガの改善 | 戦略設計を見直して本文をリライト |
| セールスファネルの構築 | 教育〜セールスまでの自動化シナリオ |
| 高額商品の販売メール | 信頼構築→提案→行動促進の設計 |
| 無料→有料への誘導 | フリーコンテンツからの自然な導線設計 |

### このシステムの特徴

- ✅ **煽りに頼らない** — 恐怖や不安で売るのではなく、「共鳴と仲間意識」で自然に行動を促します
- ✅ **8要素のフレームワーク** — 体系化された8つの要素を組み合わせるだけで高品質なメールに
- ✅ **350+のフレーズ集** — ゼロから考える必要なし。選ぶだけで強力な表現が使えます
- ✅ **200+のテンプレート** — 構造が決まっているので迷わず書けます
- ✅ **80+のチェック項目** — 品質を自動検証し、改善点を指摘します

## 2. D-BE UNICK とは？

D-BE UNICK は「ステップメールの8つの構成要素」を表す頭文字です。

### 従来のやり方との違い

```
❌ 従来（PASONA型）：
   「あなたにはこんな問題があります」→「不安ですよね」→「だから買ってください」
   
✅ D-BE UNICK：
   「私はこう考えています」→「共感しますよね」→「一緒に歩みましょう」
```

### 8要素の一覧

| 記号 | 英語 | 日本語 | かんたんな説明 |
|------|------|--------|--------------|
| **D** | Declaration | 違いの宣言 | 「この人は他と違う！」と思わせる |
| **B** | Benefit | 恩恵 | 読者が得られる未来を鮮明に描く |
| **E** | Evidence | 裏づけ | 数字や実績で信頼を得る |
| **U** | Urgency | 必然性 | 「今やるべき理由」を自然に感じさせる |
| **N** | Named Enemy | 名もなき敵 | 本当の敵を名指しし、読者を解放する |
| **I** | Innovation | 新しい勝ち方 | 「この方法ならできそう！」と希望を持たせる |
| **C** | Case Study | 体感事例 | 成功ストーリーを追体験させる |
| **K** | Kickstart | 最初の一歩 | 具体的な行動へ優しく招待する |

### 大切なルール（倫理原則）

> ⚠️ このシステムでは、以下のような表現は **絶対に使いません**：
> - 読者を責める言葉（「あなたが悪い」「怠けている」）
> - 嘘の限定性（「残り3名！」← 実際は無制限）
> - 過度な煽り（「知らないと人生終わります！」）
> - 読者を「お客様」扱いする（「仲間」として扱う）

├── 📁 ナレッジ/              ← AIが参照する知識ベース

│   ├── D_違いの宣言.md                     ← D要素の完全ガイド
│   ├── B_恩恵.md                           ← B要素の完全ガイド

│   ├── C_体感事例.md                       ← C要素の完全ガイド

│   ├── D-BE_UNICK_コピーライティング強化ツール.md ← 表現テクニック集

### 処理の流れ

```
あなたの商品情報を入力/ に配置

   Phase 1：戦略設計（全体の方針決め）
        ↓
   Phase 2：配列設計（各通の構成決め）
        ↓
   Phase 3：本文生成（メール文章の生成）
        ↓
   Phase 4：品質チェック（最終チェック）
        ↓
   完成！→ 実行結果/ フォルダに出力
```

## 4. 準備：入力情報を用意する

### 用意するもの

`入力/` フォルダに、以下の情報を含むテキストファイル（.md や .txt）を配置してください。

| 項目 | 記入内容 | 記入例 |
|------|---------|--------|
| **商品名** | あなたの商品・サービスの名前 | 「SNSマーケティング実践講座」 |
| **商品概要** | 何を提供するのかの説明 | 「SNS運用の基礎〜収益化までを学ぶ3ヶ月の講座」 |
| **ターゲット** | 誰に向けた商品か | 「副業で月5万円を目指す30代会社員」 |
| **USP** | 他との違い・独自の強み | 「AIツールを使って作業時間を1/3に削減」 |
| **価格帯** | 商品の価格 | 「29.8万円（税込）」 |
| **実績・事例** | これまでの成果 | 「受講者127名、平均3ヶ月で月5万円達成」 |
| **目指すゴール** | メルマガの最終目標 | 「無料登録者を有料講座へ誘導する」 |

### 記入例テンプレート

以下をコピーして、ご自身の情報に書き換えてください：

```markdown
# 入力情報

## 商品名
（あなたの商品名）

## 商品概要
（何を提供するか、どんな内容か）

## ターゲット
- 年齢層：
- 性別：
- 職業：
- 悩み：
- 理想の未来：

## USP（独自の売り）
（他の類似商品との違い）

## 価格帯
（商品の価格）

## 実績・事例
（これまでの成果、お客様の声など）

## 目指すゴール
（このステップメールで最終的に何をしたいか）
```

> 💡 **ポイント**: 情報は多ければ多いほど、AIの生成品質が上がります。
> 迷ったら「お客様からよく聞かれること」を書くと効果的です。

## 5. 実行：4つのフェーズで進める

AIに以下のように指示してください：

### 実行方法

```
このフォルダの SKILL.md に従って、ステップメールを生成してください。
入力情報は 入力/ フォルダに配置してあります。
```

AIが自動的に4つのフェーズを順番に実行します。

---

### Phase 1：戦略設計 ⏱ 約5-10分

**何をするか**: あなたの商品に合った全体戦略を設計します。

**生成されるもの**:
- ターゲットの深掘り分析
- キャラクター設計（発信者のキャラクター5要素）
- ビジネスカルテ（価値の3階層・悩みの立体把握）
- 8要素のカードマッピング（どの要素でどんな内容を伝えるか）
- シーケンスタイプの選択（Aタイプ or Bタイプ）

**出力先**: `実行結果/01_戦略設計シート.md`

> 💡 **確認ポイント**: 戦略設計シートが生成されたら、一度内容を確認してください。
> ターゲット設定やキャラクター設計が実態と合っているか確認し、修正があればAIに伝えてください。

---

### Phase 2：配列設計 ⏱ 約5分

**何をするか**: 各メール（各「通」）の設計図を作ります。

**生成されるもの**:
- 各通の送信タイミング
- 各通の件名案（3つ以上）
- 各通に含めるD-BE UNICK要素
- 各通の文字数目安

**出力先**: `実行結果/02_配列設計表.md`

---

### Phase 3：本文生成 ⏱ 約20-40分

**何をするか**: 各通のメール本文を実際に書きます。

**このフェーズでAIが参照するもの**:
- 8要素の個別ファイル（D/B/E/U/N/I/C/K）からテンプレート・フレーズを引用
- コピーライティング強化ツールから件名・感覚言語を適用
- ステップメール設計図から構造ルールを適用

**出力先**: `実行結果/03_メール本文/` 配下に通番ごとのファイル

> 💡 **ポイント**: このフェーズが最も時間がかかります。
> メールの通数が多い場合（10通以上）は、途中で区切って進めることも可能です。

---

### Phase 4：品質チェック ⏱ 約5-10分

**何をするか**: 生成されたメール全体を検証し、改善点を指摘します。

**チェック内容**:
- D-BE UNICK要素の分布バランス
- 倫理原則の遵守（煽り・読者攻撃がないか）
- コピーライティングの品質
- フォーマットの適切さ
- 全体フローの自然さ

**出力先**: `実行結果/04_品質チェック結果.md`

> 💡 **改善が必要な場合**: 品質チェックで指摘された項目をAIに伝えると、Phase 3 を再実行して修正されます。

## 6. 生成結果の確認と活用

### 出力されるファイル一覧

```
実行結果/
├── 01_戦略設計シート.md      ← 全体の戦略方針
├── 02_配列設計表.md          ← 各通の設計図
├── 03_メール本文/
│   ├── 01通目_XXX.md         ← 各メールの本文
│   ├── 02通目_XXX.md
│   ├── ...
│   └── 10通目_XXX.md
└── 04_品質チェック結果.md    ← 検証レポート
```

### メール本文の活用方法

1. `03_メール本文/` フォルダの各ファイルを開く
2. 件名案（3つ以上）から最適なものを選ぶ
3. 本文を自分の言葉に微調整する（固有名詞・数値・エピソードなど）
4. メール配信ツール（マイスピー、エキスパ、ConvertKit 等）に登録する
5. 送信タイミングを設定して配信開始！

> ⚠️ **重要**: AIが生成した文章は「たたき台」です。
> あなた自身の実体験、具体的な数値、実名のエピソードを追加すると、
> 信頼性とオリジナリティが大幅にアップします。

## 7. ナレッジファイルの読み方

### 8要素の個別ファイルの使い方

各要素ファイルは同じ構成で統一されています：

| セクション | 内容 | 活用シーン |
|-----------|------|-----------|
| 概要と役割 | その要素の定義と心理効果 | 「この要素は何のためにあるの？」を理解する |
| テンプレート集 | コピペして使える型 | メール本文を手動で書く時の土台にする |
| フレーズ集 | 選ぶだけで使える表現 | AIが生成した文章を手動で強化する時に参照 |
| チェックリスト | ✅チェック形式の品質基準 | 完成したメールの品質を自分でも確認する |
| AIによる生成時の鉄則 | AI用のルール | AIへの追加指示として活用する |

### 手動で文章を改善したい場合

1. 改善したい要素のファイルを開く（例：信頼性を上げたい → `E_裏づけ.md`）
2. テンプレート集からマッチするパターンを選ぶ
3. フレーズ集から強力な表現を選んで差し替える
4. チェックリストで最終確認する

## 8. よくある質問（FAQ）

### 🔵 「メール何通分作れますか？」

システムには2つのタイプがあります：
- **タイプA（分岐後セールス型）**: 10日間で10〜13通
- **タイプB（高額商品販売型）**: 10日間で23通

どちらが適切かは、Phase 1 で自動判定されます。

---

### 🔵 「商品がまだ完成していなくても使えますか？」

はい、使えます。商品のコンセプトとターゲットが決まっていれば、
「仮の情報」で戦略設計とメール構成を先に作成しておくことが可能です。
商品完成後に具体的な数値や事例を差し替えてください。

---

### 🔵 「生成されたメールをそのまま使っていいですか？」

そのまま使うことも可能ですが、以下の箇所だけは **必ずあなた自身の情報に差し替えて** ください：
- 実績の数字（受講者数、成果数値など）
- 具体的なエピソード（登場人物、日時、場所）
- 価格情報
- CTA（申し込みリンク）

---

### 🔵 「途中で方針を変えたくなったらどうすればいいですか？」

Phase 1（戦略設計）をやり直すだけでOKです。
AIに「Phase 1 を以下の方針で修正してください：〇〇」と伝えれば、
それ以降の Phase 2〜4 も新しい方針に沿って再生成されます。

---

### 🔵 「8要素を全部理解しないと使えませんか？」

不要です。AIがナレッジファイルを自動参照して適切に配置します。
ただし、8要素の概要 ([2. D-BE UNICK とは？](#2-d-be-unick-とは) セクション) だけは
読んでおくと、生成結果の良し悪しを判断しやすくなります。

---

### 🔵 「事例フォルダ（事例_ウラアド）には何が入っていますか？」

実際に本システムで生成したステップメールの完成例が入っています。
「こんな感じのものが出力される」というイメージの参考にしてください。

---

### 🔵 「品質チェックで問題が見つかったらどうすればいいですか？」

AIに以下のように伝えてください：
```
品質チェックの結果を踏まえて、Phase 3 の〇通目を修正してください。
```
AIが自動的に該当箇所を改善して再出力します。

---

## 📌 注意事項

1. **入力情報の正確性** — AIの生成品質は入力情報の質に比例します。できるだけ詳しい情報を提供してください
2. **実績の正直さ** — 嘘の実績や誇張した数字を入力しないでください。D-BE UNICK の倫理原則に反します
3. **最終確認** — 配信前に必ず人間の目で最終チェックしてください
4. **法令遵守** — 特定電子メール法などの法令を遵守し、配信停止リンクを必ず含めてください
