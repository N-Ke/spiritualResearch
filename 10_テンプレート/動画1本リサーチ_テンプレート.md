# 動画1本リサーチ（テンプレート）

旧名: 非属人スピchリサーチテンプレート。

このテンプレートは、**人が事実を入れる** → **AIが分析する** で回す前提のもの。  
**ジャンル全体の結論** は `20_ジャンル/…/_genre_sheet.md` に集約し、本テンプレは **動画1本の素材** 用（手順は [[../00_はじめに]]）。

このテンプレでやりたいことは、単なる要約ではなく、動画を `約束 / 信頼 / 参加 / 再視聴` の構造に分解して、**自分のチャンネルに転用できる勝ち筋** を抜くこと。
学習したいときは、AI に **「各フェーズの狙い」「なぜこの順番か」「私が答える問いも入れて」** と依頼してよい。

毎回、人が頑張って考察まで書かなくていい。  
基本は **基本情報 + 台本 or 文字起こし** があれば十分。
できれば **サムネ情報** も入れる。  
**概要欄（説明文）** は収益導線・CTA・免責の SoT に近いので、可能なら **全文を `description.md` に保存** し、§3 で分析する（取得方法は文末「YouTube Data API」参照）。

---

## 使い方

1. まず `channel_id / video_id / thumbnail_id` を振る
2. フォルダは `チャンネルごと` に分け、その中に `動画ごとのフォルダ` を作る
3. 人が `1. 人が入力するところ` だけ埋める
4. サムネの文字・構図・第一印象も入れる
5. 台本か文字起こしを `2. 台本 / 素材` に貼る（長文は `transcript.md` でも可）
6. **概要欄**を `description.md` に貼る（手入力 or API 取得。未取得なら status のみ）
7. その状態でAIに、`3以降を埋めて` と依頼する
8. 学習したい場合は、`各フェーズの狙いと、私が考えるための問いも入れて` と追加する
9. 最後に `5. 横断メモ3行` を、必要なら **対象ジャンルの `_genre_sheet.md` の §4〜6** へ転記する（チャンネル全体の分析は `チャンネル分析_ジャンルへ転記用_テンプレート` が向く）

### IDルール

- `channel_id`: `CH001` のようにチャンネル単位で固定
- `video_id`: `CH001-VID001` のようにチャンネル配下で連番
- `thumbnail_id`: `TH-CH001-VID001-01` のように動画ごとに付与

## 7. 概要欄の取得（YouTube Data API v3）

**結論**: **取得できる。** 動画の `snippet.description` に概要欄の本文が入る（公開動画で一般的に利用可能）。

| 項目 | 内容 |
|---|---|
| エンドポイント | `https://www.googleapis.com/youtube/v3/videos` |
| 主なパラメータ | `part=snippet`（必須に近い）、`id=動画ID`（カンマ区切りで複数可） |
| 認証 | **公開データのみ**なら **API キー**（`key=...`）で足りることが多い。OAuth は、ユーザー限定データが必要なとき。 |
| クォータ | `videos.list` は **1 回あたり 1 ユニット**（2024 時点の一般的説明。Google の最新ドキュメントで確認）。 |
| 得られるもの | `items[0].snippet.title`・`description`・`publishedAt`・`channelId` など。本文は **`description`**。 |
| 補足 | チャンネル全体の「固定の概要テンプレ」ではなく、**動画ごとの説明文**が取れる。YouTube 上で編集されていれば、その時点の文字列。 |

**リクエスト例（概念）**

`GET .../videos?part=snippet&id=動画ID&key=APIキー`

**このリポジトリでの運用**: 取得した本文は **`VID###/description.md`** に保存し、`research.md` の §2 と §3「概要欄分析」を埋める。API 実装は `tooling/` 側に置く場合は、Google Cloud で YouTube Data API v3 を有効化し、キー管理に注意する。

公式: [YouTube Data API 動画一覧](https://developers.google.com/youtube/v3/docs/videos/list)
