# C. 表情マッピング

Remotion で使い分ける綾の4表情を、**いつ／なぜ使うか** まで落とし込む。

引継ぎ書 §9「制作パイプライン情報」で表情指定は4種に固定済み：
**真顔／微笑み／やや驚き／思索中**

## 設計原則（3つ）

1. **微笑みを基調にする** — 綾は「ホッとできる人」。基本表情を引き締めて見せる必要はない。
2. **表情の振れ幅を抑える** — 引継ぎ書 §3 の語り口ルール「推量表現で押し付けない」と整合。大きな驚きや強い真顔は人格と矛盾する。
3. **言葉の意味と表情を一致させる** — 「気づきました」と言いながら微笑みを保つのは違和感。表情は言葉の意味を増幅する補助線。

## 4表情の役割

### 微笑み（基本表情・基調）
- **コア役割**: 安心感・親密度の維持。視聴者の警戒を解く。
- **使う場面**:
  - 動画冒頭の挨拶
  - 視聴者への呼びかけ（「もし〇〇な方は…」）
  - 肯定のフレーズ（承認／許可／救済のいずれか）
  - 締めのCTA（「よかったら受け取ってください」）
  - 押し付けない言い回し（「合わなければやめてください」）
- **想定比率**: 全表情の **45〜55%**

### 真顔（穏やかな真顔）
- **コア役割**: 情報伝達。微笑みから少し力を抜いた中立表情。**険しい真顔ではない**。
- **使う場面**:
  - 原理の説明（「こういう仕組みなんです」）
  - 数字・研究・データの提示
  - 注意事項・補足（「ただし〜」）
  - 反論処理の質問パート（「でも本当かな、と思いますよね」の "本当かな" 部分）
  - 喋り続ける情報密度の高い区間
- **想定比率**: 全表情の **25〜35%**

### やや驚き
- **コア役割**: 「気づき」「発見」。**外的ショックの驚きではなく、内的な「あ、そうか」の表情**。
- **使う場面**:
  - 転換点（「気づいたんです」「実は…」）
  - 反論処理の答えパート（「でも、本当にそれだけで変わったんですよ」）
  - 視聴者の心の声を代弁（「ありますよね、こういう瞬間」）
  - 実践結果の描写（「やめた翌朝、少しだけ体が軽かったんです」）
- **想定比率**: 全表情の **8〜12%**
- **重要**: 多用すると人格が落ち着かない印象になり、テーゼと矛盾する。動画1本で **3〜5回まで** が目安。

### 思索中
- **コア役割**: 言葉を選んでいる、自分の中で確認している。視線わずかに下〜遠く。
- **使う場面**:
  - 自己開示パート（綾の過去を思い出す場面、体験談型のみ）
  - 「正確に言うと…」「うまく言えないんですが…」「なんていうか…」
  - 結論前の余韻（次の言葉を出す前の間）
  - 心地よさの描写（紅茶・窓辺の描写を語るとき）
  - 視聴者と一緒に静かに考える瞬間
- **想定比率**: 全表情の **10〜15%**
- **長さ**: 思索中は **2〜4秒の短い差し込み** が基本。長すぎるとテンポが落ちる。

## 動画パターン別の比率目安

| パターン | 微笑み | 真顔 | やや驚き | 思索中 | 補足 |
|---|---:|---:|---:|---:|---|
| 体験談×原理解説型 | 45% | 25% | 10% | 20% | 思索中を多めに（自己開示の質感） |
| 実践メソッド型 | 50% | 35% | 8% | 7% | 真顔を多めに（手順説明の明瞭さ） |
| 反論処理型 | 50% | 30% | 12% | 8% | やや驚きを多めに（疑問→気づきの流れ） |

## 台本構造の位置別マッピング

| 位置 | 基本表情 | 補足 |
|---|---|---|
| 冒頭挨拶 | 微笑み | 安心の入り口 |
| テーマ宣言 | 微笑み→真顔 | 情報伝達に切り替え |
| 結論先出し | 真顔 | 内容を聴かせる |
| 対象視聴者の明示 | 微笑み | 「あなたに語っている」感 |
| 原理の提示 | 真顔 | 情報密度高め |
| 過去の失敗（多くの方が…） | 真顔 | 案内人視点 |
| 過去の失敗（綾自身） | 思索中 | 体験談型のみ。記憶を辿る感じ |
| 反論処理（疑問の提示） | 真顔→やや驚き | 「でも本当かな、と思いますよね」 |
| 転換点（気づき） | やや驚き | 「気づいたんです」 |
| 具体的実践法 | 微笑み→真顔 | 安心感+明瞭さ |
| 実践結果（変化の描写） | やや驚き→微笑み | 「翌朝、少しだけ軽かったんです」→余韻の微笑み |
| まとめ（テーゼ再提示） | 微笑み | 中核テーゼには微笑みを必ず合わせる |
| CTA | 微笑み | 押し付けない |

## 表情遷移のルール

### 推奨パターン
- 微笑み ⇄ 真顔（最も多い遷移。情報密度に応じて）
- 真顔 → やや驚き → 微笑み（反論処理〜転換点で頻出）
- 思索中 → 微笑み（自己開示の終わりに戻る）

### 禁止パターン
- 微笑み → やや驚き への急な切り替え（情緒の振れ幅が大きすぎる）
- 連続する「やや驚き」（人格が落ち着かなく見える）
- 真顔の連続多用（説教調・押し付けに見える）
- 締めCTA直前の「やや驚き」（押し売り感が出る）

## 表情切り替えのタイミング

- **言葉の意味の境目**で切り替える（句点・読点・接続詞の前）
- 1秒未満の高速切替は避ける（チラつきになる）
- 表情変化と同時に画面タイプ（キャラ正面→テロップ→Bロール）も切り替えると、キャラ画像生成数を抑えられる

## 画面タイプとの連動（Remotion）

引継ぎ書 §4「画面構成のリズム」より：
キャラ正面3〜8秒→テロップ要点画面→イメージBロール→キャラに戻る

| 画面タイプ | 表情の扱い |
|---|---|
| キャラ正面 | 4表情を使い分ける |
| テロップ要点画面 | 表情指定は **直前の表情を維持** （連続性のため）|
| イメージBロール | 表情指定は不要。Bロール明けで自然に戻す |

これにより、必要な表情画像の生成数を最小化できる。1動画あたり、4表情 × 服装1〜2パターン = **最大8枚** で足りる想定。

## 検証手順（次にやること）

1. テスト動画第1本「頑張るのをやめた瞬間、人生が動き出した話」（体験談×原理解説型）の台本ドラフトに、本マッピングを適用
2. 各セクションに表情タグ `[表情:微笑み]` などを書き込み
3. 比率を計測し、想定比率（45/25/10/20）から乖離があれば調整
4. Remotion の入力 JSON 形式に変換できるか確認

## NG（絶対に出さない表情）

- 怒り・困惑・嫌悪・蔑み（綾の人格と完全に矛盾）
- 強い驚き（目を大きく見開く・口を開ける系）
- 大笑い・歯を見せる笑顔
- 流し目・ウインク（媚びの方向）
- 涙目・泣き顔（劇的すぎる。テーゼ「穏やかな希望」と矛盾）
